# 多 GPU 训练

本指南介绍如何使用 [Hugging Face Accelerate](https://huggingface.co/docs/accelerate) 在多个 GPU 上训练策略。

## 安装

`accelerate` 包含在 `training` 可选依赖中。通过以下命令安装：

```bash
pip install 'lerobot[training]'
```

## 多 GPU 训练

您可以通过两种方式启动训练：

### 选项 1：不配置（直接指定参数）

您可以直接在命令中指定所有参数，无需运行 `accelerate config`：

```bash
accelerate launch \
  --multi_gpu \
  --num_processes=2 \
  $(which lerobot-train) \
  --dataset.repo_id=${HF_USER}/my_dataset \
  --policy.type=act \
  --policy.repo_id=${HF_USER}/my_trained_policy \
  --output_dir=outputs/train/act_multi_gpu \
  --job_name=act_multi_gpu \
  --wandb.enable=true
```

**主要的 accelerate 参数：**

- `--multi_gpu`：启用多 GPU 训练
- `--num_processes=2`：使用的 GPU 数量
- `--mixed_precision=fp16`：使用 fp16 混合精度（如果支持也可使用 `bf16`）

### 选项 2：使用 accelerate 配置

如果您希望保存配置，可以选择运行以下命令为您的硬件环境配置 accelerate：

```bash
accelerate config
```

此交互式设置将询问您关于训练环境的问题（GPU 数量、混合精度设置等），并将配置保存下来供以后使用。对于单台机器上的简单多 GPU 设置，您可以使用以下推荐设置：

- 计算环境：本机
- 机器数量：1
- 进程数量：（您想要使用的 GPU 数量）
- 要使用的 GPU ID：（留空则使用全部）
- 混合精度：fp16 或 bf16（推荐用于加速训练）

然后使用以下命令启动训练：

```bash
accelerate launch $(which lerobot-train) \
  --dataset.repo_id=${HF_USER}/my_dataset \
  --policy.type=act \
  --policy.repo_id=${HF_USER}/my_trained_policy \
  --output_dir=outputs/train/act_multi_gpu \
  --job_name=act_multi_gpu \
  --wandb.enable=true
```

## 工作原理

当您使用 accelerate 启动训练时：

1. **自动检测**：LeRobot 自动检测是否运行在 accelerate 下
2. **数据分发**：您的批次会自动分配到各个 GPU
3. **梯度同步**：在反向传播过程中梯度会在 GPU 之间同步
4. **单进程日志记录**：仅主进程向 wandb 记录日志和保存检查点

## 学习率和训练步数的缩放

**重要提示：** LeRobot **不会**根据 GPU 数量自动缩放学习率或训练步数。这使您能够完全控制训练超参数。

### 为什么不做自动缩放？

许多分布式训练框架会根据 GPU 数量自动缩放学习率（例如 `lr = base_lr × num_gpus`）。然而，LeRobot 会严格使用您指定的学习率。

### 何时以及如何缩放

如果您希望在使用多个 GPU 时缩放超参数，应手动操作：

**学习率缩放：**

```bash
# 示例：2 个 GPU，线性学习率缩放
# 基础学习率：1e-4，使用 2 个 GPU -> 2e-4
accelerate launch --num_processes=2 $(which lerobot-train) \
  --optimizer.lr=2e-4 \
  --dataset.repo_id=lerobot/pusht \
  --policy.type=act
```

**训练步数缩放：**

由于有效批量大小 `bs` 随着多 GPU 的增加而增大（batch_size × num_gpus），您可能需要按比例减少训练步数：

```bash
# 示例：2 个 GPU，有效批量大小为原来的 2 倍
# 原始：batch_size=8, steps=100000
# 使用 2 个 GPU：batch_size=8（总共 16），steps=50000
accelerate launch --num_processes=2 $(which lerobot-train) \
  --batch_size=8 \
  --steps=50000 \
  --dataset.repo_id=lerobot/pusht \
  --policy.type=act
```

## 使用 FSDP 训练大模型

DDP（Distributed Data Parallel）会在每个 GPU 上复制完整模型，因此单 GPU 无法容纳的模型在 DDP 下同样无法容纳。对于大模型，请使用 **FSDP**（Fully Sharded Data Parallel，全分片数据并行），它将参数、梯度和优化器状态分片到多个 GPU 上。背景知识请参见 [accelerate FSDP 指南](https://huggingface.co/docs/accelerate/usage_guides/fsdp)。

以下是在 4 个 GPU（1 台机器）上使用 FSDP 启动 LeRobot 训练的示例：

```bash
accelerate launch --config_file fsdp.yaml --num_processes=4 $(which lerobot-train) \
  --dataset.repo_id=${HF_USER}/my_dataset \
  --policy.type=<your_policy> \
  --output_dir=outputs/train/my_policy_fsdp
```

一个最小化的 `fsdp.yaml`（FSDP1；分片参数/梯度/优化器——相当于 ZeRO-3）：

```yaml
compute_environment: LOCAL_MACHINE
distributed_type: FSDP
mixed_precision: bf16
num_machines: 1
num_processes: 4
fsdp_config:
  fsdp_version: 1
  fsdp_sharding_strategy: FULL_SHARD # 参数 + 梯度 + 优化器 (ZeRO-3)
  fsdp_auto_wrap_policy: TRANSFORMER_BASED_WRAP
  fsdp_transformer_layer_cls_to_wrap: <YourTransformerBlock> # 需要分片的重复块类
  fsdp_use_orig_params: true # 必需：优化器在 prepare 之前已构建
  fsdp_state_dict_type: FULL_STATE_DICT
```

将 `fsdp_transformer_layer_cls_to_wrap` 设置为模型重复的 transformer 块类，以便每个块作为独立单元进行分片。`fsdp_use_orig_params: true` 是必需的，因为 LeRobot 在 `accelerator.prepare()` 之前构建优化器。

### FSDP 检查点

LeRobot 会收集所有 rank 的完整状态字典，由主进程将其写入单个 `model.safetensors` 文件，可像往常一样使用 `Policy.from_pretrained(...)` 加载。需要注意两点：

- **检查点存储 fp32 权重。** 在混合精度（`bf16`/`fp16`）下，FSDP 会保留一份 fp32 主副本，检查点保存的就是这份副本（磁盘上约为 bf16 大小的 2 倍），以便训练能够与 fp32 优化器状态一致地恢复；`from_pretrained` 在加载时会将其转换回策略的数据类型。FSDP 特有的问题：fp32 检查点会在目标设备上以完整精度实例化_然后_再进行类型转换，因此在显存紧张的 GPU 上加载它进行推理时，即使 bf16 模型可以容纳，也可能导致 OOM——请先在 CPU 上加载，或者将 `model.safetensors` 离线转换为部署时的数据类型。
- 分片的优化器状态会被收集为完整的（与世界大小无关的）状态字典，并以与单 GPU 训练相同的 `optimizer_state.safetensors` / `optimizer_param_groups.json` 格式与模型一起保存，因此支持**从检查点恢复训练**（使用 `--resume=true`）。恢复时会对模型和优化器状态重新分片以适应当前的 FSDP 拓扑，因此您可以在不同数量的 GPU 上恢复 FSDP 检查点。请注意，数据采样器只有在世界大小和批量大小与原始运行一致时才是样本精确的（否则会记录一条警告）；优化器/模型状态本身不受影响。

## 注意事项

- `lerobot-train` 中的 `--policy.use_amp` 标志仅在**未**使用 accelerate 运行时才有效。使用 accelerate 时，混合精度由 accelerate 的配置控制。
- 训练日志、检查点和 Hub 上传仅由主进程执行，以避免冲突。非主进程的控制台日志记录被禁用，以防止重复输出。
- 有效批量大小为 `batch_size × num_gpus`。如果您使用 4 个 GPU 和 `--batch_size=8`，则您的有效批量大小为 32。
- 学习率调度在多个进程间正确处理——LeRobot 设置 `step_scheduler_with_optimizer=False`，以防止 accelerate 根据进程数调整调度器步数。
- 在保存或推送模型时，LeRobot 会自动从 accelerate 的分布式包装器中解包模型，以确保兼容性。
- WandB 集成会自动仅在主进程上初始化，防止创建多个运行。

有关更高级的配置和故障排除，请参见 [Accelerate 文档](https://huggingface.co/docs/accelerate)。如果您想了解如何在大量 GPU 上训练，请查看这份优秀的指南：[Ultrascale Playbook](https://huggingface.co/spaces/nanotron/ultrascale-playbook)。
