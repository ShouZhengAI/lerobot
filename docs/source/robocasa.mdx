# RoboCasa365

[RoboCasa365](https://robocasa.ai) 是一个大规模仿真框架，用于在日常厨房任务中训练和基准测试**通用机器人**。它提供了涵盖 2,500 个厨房环境中的 365 种多样化操作任务、3,200 多个物体资产和 600 多小时的人类示教数据，运行在 PandaOmron 12 自由度移动操作臂（Franka 机械臂搭配全向底盘）上。

- 论文：[RoboCasa: Large-Scale Simulation of Everyday Tasks for Generalist Robots](https://arxiv.org/abs/2406.02523)
- GitHub：[robocasa/robocasa](https://github.com/robocasa/robocasa)
- 项目网站：[robocasa.ai](https://robocasa.ai)
- 预训练策略：[`lerobot/smolvla_robocasa`](https://huggingface.co/lerobot/smolvla_robocasa)
- 单任务数据集（CloseFridge）：[`pepijn223/robocasa_CloseFridge`](https://huggingface.co/datasets/pepijn223/robocasa_CloseFridge)

## 可用任务

RoboCasa365 将其 365 个任务组织为两个系列和三个上游基准测试组，LeRobot 将它们作为一级 `--env.task` 快捷方式提供：

| 系列       | 任务数 | 描述                                                                 |
| ---------- | ------ | -------------------------------------------------------------------- |
| 原子操作   | ~65    | 单一技能任务：抓取放置、门/抽屉操作、电器控制                          |
| 复合操作   | ~300   | 跨 60+ 类别的多步骤任务：烹饪、清洁、整理等                             |

**原子操作任务示例：** `CloseFridge`（关冰箱）、`OpenDrawer`（开抽屉）、`OpenCabinet`（开橱柜）、`TurnOnMicrowave`（开微波炉）、`TurnOffStove`（关炉灶）、`NavigateKitchen`（导航厨房）、`PickPlaceCounterToStove`（台面到炉灶抓放）。

**复合操作任务类别：** 烘焙、煮沸、冲泡、切菜、清理桌面、解冻食物、装洗碗机、泡茶、微波加热、洗碗等。

`--env.task` 接受三种形式：

- 单个任务名称（`CloseFridge`）
- 逗号分隔的任务列表（`CloseFridge,OpenBlenderLid,PickPlaceCoffee`）
- 基准测试组快捷方式——`atomic_seen`、`composite_seen`、`composite_unseen`、`pretrain50`、`pretrain100`、`pretrain200`、`pretrain300`——会自动展开为上游任务列表并自动设置数据集 `split`（`target` 或 `pretrain`）。

## 安装

RoboCasa 及其依赖 `robosuite` 未发布在 PyPI 上，且 RoboCasa 自身的 `setup.py` 硬编码了 `lerobot==0.3.3`，这与本仓库的 `lerobot` 冲突。因此 LeRobot **不**提供 `robocasa` 额外依赖项——需要以可编辑克隆方式手动安装这两个包（对 `robocasa` 使用 `--no-deps` 以跳过其屏蔽的 `lerobot` 版本锁定）：

```bash
# 在完成标准 LeRobot 安装说明后执行。

git clone https://github.com/robocasa/robocasa.git ~/robocasa
git clone https://github.com/ARISE-Initiative/robosuite.git ~/robosuite
pip install -e ~/robocasa --no-deps
pip install -e ~/robosuite

# Robocasa 的运行时依赖（即其 setup.py 本应安装的依赖，减去有问题的 lerobot 版本锁定）
pip install numpy numba scipy mujoco pygame Pillow opencv-python \
            pyyaml pynput tqdm termcolor imageio h5py lxml hidapi \
            tianshou gymnasium

python -m robocasa.scripts.setup_macros
# 轻量级资产（lightwheel 物体网格 + 纹理）。足以开箱即用地运行默认环境。
python -m robocasa.scripts.download_kitchen_assets \
  --type tex tex_generative fixtures_lw objs_lw
# 可选：完整的 objaverse/aigen 注册表（约 30GB），用于更丰富的物体种类。
# 通过 --env.obj_registries 在评估时启用（见下文）。
# python -m robocasa.scripts.download_kitchen_assets --type objs_objaverse
```

<Tip>
RoboCasa 需要 MuJoCo。在训练或评估前设置渲染后端：

```bash
export MUJOCO_GL=egl  # 用于无头服务器（HPC、云环境）
```

</Tip>

### 物体注册表

默认情况下，环境仅从 `lightwheel` 注册表（即 `--type objs_lw` 提供的资源）采样物体，这可以避免在 objaverse/aigen 包未下载到磁盘时出现 `Probabilities contain NaN` 崩溃。如果你已下载完整资产集，可以在运行时启用完整注册表：

```bash
--env.obj_registries='[objaverse,lightwheel]'
```

## 评估

以下所有评估代码片段与 CI 命令保持一致（参见 `.github/workflows/benchmark_tests.yml`）。`--rename_map` 参数将 RoboCasa 的原生摄像头键名（`robot0_agentview_left` / `robot0_eye_in_hand` / `robot0_agentview_right`）映射到已发布 `smolvla_robocasa` 策略训练时使用的三摄像头（`camera1` / `camera2` / `camera3`）输入布局。

### 单任务评估（推荐用于快速迭代）

```bash
lerobot-eval \
  --policy.path=lerobot/smolvla_robocasa \
  --env.type=robocasa \
  --env.task=CloseFridge \
  --eval.batch_size=1 \
  --eval.n_episodes=20 \
  --eval.use_async_envs=false \
  --policy.device=cuda \
  '--rename_map={"observation.images.robot0_agentview_left": "observation.images.camera1", "observation.images.robot0_eye_in_hand": "observation.images.camera2", "observation.images.robot0_agentview_right": "observation.images.camera3"}'
```

### 多任务评估

传入逗号分隔的任务列表：

```bash
lerobot-eval \
  --policy.path=lerobot/smolvla_robocasa \
  --env.type=robocasa \
  --env.task=CloseFridge,OpenCabinet,OpenDrawer,TurnOnMicrowave,TurnOffStove \
  --eval.batch_size=1 \
  --eval.n_episodes=20 \
  --eval.use_async_envs=false \
  --policy.device=cuda \
  '--rename_map={"observation.images.robot0_agentview_left": "observation.images.camera1", "observation.images.robot0_eye_in_hand": "observation.images.camera2", "observation.images.robot0_agentview_right": "observation.images.camera3"}'
```

### 基准测试组评估

运行整个上游组（例如所有 18 个 `atomic_seen` 任务，使用 `split=target`）：

```bash
lerobot-eval \
  --policy.path=lerobot/smolvla_robocasa \
  --env.type=robocasa \
  --env.task=atomic_seen \
  --eval.batch_size=1 \
  --eval.n_episodes=20 \
  --eval.use_async_envs=false \
  --policy.device=cuda \
  '--rename_map={"observation.images.robot0_agentview_left": "observation.images.camera1", "observation.images.robot0_eye_in_hand": "observation.images.camera2", "observation.images.robot0_agentview_right": "observation.images.camera3"}'
```

### 建议的评估回合数

**每个任务 20 个回合**以实现可重复的基准测试。这与已发表结果中使用的协议一致。

## 策略输入与输出

**观测数据**（原始的 RoboCasa 摄像头名称保持不变）：

- `observation.state` — 16 维本体感受状态（基座位置、基座四元数、相对末端执行器位置、相对末端执行器四元数、夹爪位置）
- `observation.images.robot0_agentview_left` — 左侧代理视角，256×256 HWC uint8
- `observation.images.robot0_eye_in_hand` — 腕部摄像头视角，256×256 HWC uint8
- `observation.images.robot0_agentview_right` — 右侧代理视角，256×256 HWC uint8

**动作：**

- 在 `Box(-1, 1, shape=(12,))` 中的连续控制——基座运动（4D）+ 控制模式（1D）+ 末端执行器位置（3D）+ 末端执行器旋转（3D）+ 夹爪（1D）。

## 训练

### 单任务示例

一个可直接使用的单任务数据集已在 Hub 上：
[`pepijn223/robocasa_CloseFridge`](https://huggingface.co/datasets/pepijn223/robocasa_CloseFridge)。

在 `CloseFridge` 上微调 SmolVLA 基础模型：

```bash
lerobot-train \
  --policy.type=smolvla \
  --policy.repo_id=${HF_USER}/smolvla_robocasa_CloseFridge \
  --policy.load_vlm_weights=true \
  --policy.push_to_hub=true \
  --dataset.repo_id=pepijn223/robocasa_CloseFridge \
  --env.type=robocasa \
  --env.task=CloseFridge \
  --output_dir=./outputs/smolvla_robocasa_CloseFridge \
  --steps=100000 \
  --batch_size=4 \
  --env_eval_freq=5000 \
  --eval.batch_size=1 \
  --eval.n_episodes=5 \
  --save_freq=10000
```

评估生成的检查点：

```bash
lerobot-eval \
  --policy.path=${HF_USER}/smolvla_robocasa_CloseFridge \
  --env.type=robocasa \
  --env.task=CloseFridge \
  --eval.batch_size=1 \
  --eval.n_episodes=20
```
