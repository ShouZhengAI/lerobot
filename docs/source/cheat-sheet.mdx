# 速查表

所有 LeRobot 命令汇总于此。如果你忘记了某个特定命令的用法，或者想了解一个新命令，可以在此查找。

> [!WARNING]
> 对于下面列出的所有命令，请记得将端口/名称/ID 替换为你自己的值！

> [!TIP]
> 查看所有命令并根据你的具体设置进行配置的另一个好方法是使用这个 [Jupyter Notebook](https://github.com/huggingface/lerobot/blob/main/examples/notebooks/quickstart.ipynb)。

### 安装与设置

有关安装，请查阅 [LeRobot 安装指南](https://huggingface.co/docs/lerobot/main/en/installation)。

### 实用工具

###### 查找端口

使用此命令识别你的机器人连接到哪些串行端口。按照终端中的提示操作：你需要拔掉 USB 线缆并按 Enter 键。脚本会检测并打印该机器人的正确串行端口。

```bash
lerobot-find-port
```

###### 查找摄像头

快速查找摄像头索引并验证其输出。此命令将摄像头信息打印到终端，并将每个检测到的摄像头的测试帧保存到 `lerobot/outputs/captured_images`

```bash
lerobot-find-cameras
```

### 标定

大多数情况下，你只需要对每个机器人和遥操作设备执行一次标定。在进行标定之前，请确保所有关节大致处于中间位置。

```bash
lerobot-calibrate \
    --robot.type=so101_follower \
    --robot.port=/dev/ttyACM0 \
    --robot.id=my_follower_arm
```

确保后续在其他脚本中使用与标定时相同的 ID。LeRobot 通过 ID 查找标定文件。

### 遥操作

使用两个摄像头进行遥操作并通过 Rerun 显示数据。

```bash
lerobot-teleoperate \
    --robot.type=so101_follower \
    --robot.port=/dev/ttyACM0 \
    --robot.id=my_follower_arm \
    --robot.cameras="{ top: {type: opencv, index_or_path: 1, width: 640, height: 480, fps: 30}, wrist: {type: opencv, index_or_path: 0, width: 640, height: 480, fps: 30} }" \
    --teleop.type=so101_leader \
    --teleop.port=/dev/ttyACM1 \
    --teleop.id=my_leader_arm \
    --display_data=true
```

### 录制数据集

数据集会自动上传到服务器并保存在 repo_id 下，请确保已通过 CLI 登录到你的 Hugging Face 账户：
`hf auth login`

可以从以下地址获取令牌： [https://huggingface.co/settings/tokens](https://huggingface.co/settings/tokens)

```bash
lerobot-record \
    --robot.type=so101_follower \
    --robot.port=/dev/ttyACM0 \
    --robot.id=my_follower_arm \
    --robot.cameras="{ top: {type: opencv, index_or_path: 1, width: 640, height: 480, fps: 30}, wrist: {type: opencv, index_or_path: 0, width: 640, height: 480, fps: 30} }" \
    --teleop.type=so101_leader \
    --teleop.port=/dev/ttyACM1 \
    --teleop.id=my_leader_arm \
    --dataset.repo_id=${HF_USER}/so101_dataset_test \
    --dataset.num_episodes=30 \
    --dataset.single_task="put the red brick in a bowl" \
    --dataset.streaming_encoding=true \
    --display_data=true
```

在采集数据集时，你可以使用键盘控制整个过程：
使用键盘快捷键控制数据录制流程：

- 按 **右箭头键（`→`）**：保存当前片段并进入下一个。
- 按 **左箭头键（`←`）**：删除当前片段并重试。
- 按 **退出键（`ESC`）**：停止、编码视频并上传。

### 录制深度数据

Intel RealSense 摄像头（`type: intelrealsense`）在设置 `use_depth: true` 时会录制深度流。深度数据被量化为 12 位编码并存储为单独的视频。

```bash
lerobot-record \
    ... \
    --robot.cameras="{ head: {type: intelrealsense, serial_number_or_name: \"0123456789\", width: 640, height: 480, fps: 30, use_depth: true} }" \
    --dataset.repo_id=${HF_USER}/so101_depth_test \
    --dataset.single_task="put the red brick in a bowl" \
    --dataset.depth_encoder.depth_min=0.01 \
    --dataset.depth_encoder.depth_max=10.0 \
    --dataset.depth_encoder.shift=0.0 \
    --dataset.depth_encoder.use_log=true
```

### 视频编码参数

RGB 和深度流通过 `--dataset.rgb_encoder.*` 和 `--dataset.depth_encoder.*` 参数独立编码。

```bash
lerobot-record \
    ... \
    --dataset.rgb_encoder.vcodec=h264 \
    --dataset.rgb_encoder.pix_fmt=yuv420p \
    --dataset.rgb_encoder.crf=23 \
    --dataset.depth_encoder.vcodec=hevc \
    --dataset.depth_encoder.extra_options='{"x265-params": "lossless=1"}'
```

### 训练

根据你的硬件配置，策略训练可能需要几个小时。以下是训练简单 `ACT` 策略的方法：

```bash
lerobot-train \
    --dataset.repo_id=${HF_USER}/so101_dataset_test \
    --policy.type=act \
    --output_dir=outputs/train/act_so101_test \
    --job_name=act_so101_test \
    --policy.device=cuda \
    --wandb.enable=true \
    --policy.repo_id=${HF_USER}/policy_test \
    --steps=20000
```

- 策略类型：`act`、`diffusion`、`smolvla`、`pi05`
- 设备：`cuda`（NVIDIA）、`mps`（Apple Silicon）、`cpu`

如果你想微调特定模型，可以提供模型路径。此时路径就足够了，可以省略类型。

```bash
lerobot-train \
    --dataset.repo_id=${HF_USER}/so101_dataset_test \
    --policy.path=username/the_policy_to_finetune \
    --policy.device=cuda \
    --policy.repo_id=${HF_USER}/policy_test \
    --output_dir=outputs/train/act_so101_test \
    --steps=20000
```

没有本地 GPU？任一条命令加上 `--job.target=<flavor>`（例如 `a10g-small`），`lerobot-train` 就会在 [Hugging Face Jobs](https://huggingface.co/docs/hub/jobs) 上运行 —— 它会自动上传本地数据集并推送训练好的模型。使用 `hf jobs hardware` 列出可用的 flavor。

要恢复训练，将 `--config_path` 指向一个检查点并添加 `--resume=true`。它可以接受本地路径或 Hub 仓库 ID（会获取最新的检查点），并且通过添加 `--job.target=<flavor>` 可以在本地或任务上工作：

```bash
lerobot-train --config_path=${HF_USER}/policy_test --resume=true --job.target=a10g-small
```

### 推理

推理是指在机器人上运行训练好的策略/模型。我们使用 `lerobot-rollout` 来实现此目的。你需要提供策略的路径。它可以是本地路径，也可以是 Hugging Face 上的路径，例如 "lerobot/folding_latest"。你的摄像头配置需要与采集数据集时使用的配置一致。duration 以秒为单位，如果未指定，将无限运行。

> [!TIP]
> 如果你使用的是之前的 V0.5.1 版本，需要使用 `lerobot-record` 而不是 `lerobot-rollout`。更多信息请参见[此处](https://huggingface.co/docs/lerobot/v0.5.1/en/il_robots#run-inference-and-evaluate-your-policy)。

```bash
lerobot-rollout \
    --strategy.type=base \
    --policy.path=${HF_USER}/my_policy \
    --robot.type=so101_follower \
    --robot.port=/dev/ttyACM1 \
    --robot.cameras="{ up: {type: opencv, index_or_path: /dev/video1, width: 640, height: 480, fps: 30}, side: {type: opencv, index_or_path: /dev/video5, width: 640, height: 480, fps: 30}}" \
    --task="Put lego brick into the transparent box" \
    --duration=60
```
