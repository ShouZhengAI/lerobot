# SARM：阶段感知奖励建模

SARM（Stage-Aware Reward Modeling，阶段感知奖励建模）是一种基于视频的奖励建模框架，适用于长程机器人操作任务。本指南介绍如何训练 SARM 奖励模型，以及可选地将其与基于奖励的行为克隆（RA-BC，Reward-Aligned Behavior Cloning）结合使用。

**论文**: [SARM: Stage-Aware Reward Modeling for Long Horizon Robot Manipulation](https://arxiv.org/abs/2509.25358)

## 为什么需要奖励模型？

标准的行为克隆（behavior cloning）将所有演示帧视为同等重要，但实际机器人数据集往往是杂乱无章的，包含犹豫、修正以及质量参差不齐的轨迹。奖励模型通过从演示中学习一种可泛化的**任务进度**概念来解决这一问题：给定视频帧和任务描述，模型预测机器人离完成任务还有多远（0→1）。这种学习到的"进度信号"可以多种方式使用，两个有前景的应用是：（1）**加权模仿学习**（RA-BC），即策略训练时高进度帧获得更高权重；（2）**强化学习**，即奖励模型为在线或离线策略改进提供密集奖励。

## 概览

SARM 具有以下特点：

1. **阶段感知架构**：联合预测高层级任务阶段和每个阶段内的细粒度进度
2. **子任务标注**：使用自然语言子任务标注来推导一致的进度标签
3. **时间比例**：为每个子任务计算数据集级别的先验（α̅\_k），以在不同长度的演示之间标准化进度

SARM 为每帧训练一个紧凑的 **stage+tau** 目标：

- **stage**：整数阶段索引 `k ∈ {0, ..., K-1}`
- **τ (tau)**：阶段内进度 `τ ∈ [0, 1]`
- **目标编码**：`y = k + τ`（这是数据集处理器生成的内容）

在推理时（以及下游 RA-BC 中），SARM 使用数据集级别的**时间比例** `α̅_k`（存储在 `meta/temporal_proportions_*.json` 中）将原始 `k + τ` 值转换为 `[0, 1]` 范围内的**标准化进度**。

这与论文中的**公式 (2)** 一致：

```
progress_t = P_{k-1} + α̅_k × τ_t
```

其中：

- `τ_t = (t - s_k) / (e_k - s_k)` 是子任务内标准化时间
- `P_{k-1}` 是累积先验（前几个子任务比例之和）
- `α̅_k` 是子任务 k 的时间比例

这确保了相同的任务状态映射到一致的进度值，即使是在不同长度的演示之间也是如此。

## 输入与目标（新代码的预期形式）

SARM 通过其处理器（`src/lerobot/rewards/sarm/processor_sarm.py`）进行训练，该处理器：

- **编码**：使用 CLIP（ViT-B/32）将图像和任务文本编码为 `video_features` 和 `text_features`
- **填充/截断**：将机器人状态填充或截断为 `state_features`（最多 `max_state_dim`）
- **构建目标**：使用 stage+tau 编码 `y = k + τ` 构建 `sparse_targets`（以及 `dense_only`/`dual` 模式下的 `dense_targets`）
- **掩码回退帧**：使用每个样本的 `lengths` 张量进行掩码（回退是一种训练时数据增强）

每个训练样本至少需要：

- `task`（字符串）：任务描述
- 数据集中 `policy.image_key` 对应的图像和 `policy.state_key` 对应的状态

## 标注模式

你可以选择 **3 种标注模式**，它们决定如何计算进度标签：

| 模式           | 所需标注          | 输出头                          | 使用场景                                                |
| -------------- | ----------------- | ------------------------------- | ------------------------------------------------------- |
| `single_stage` | 无                | 仅稀疏头                        | 简单任务、快速实验，无需 VLM                            |
| `dense_only`   | 密集标注 (VLM)    | 双头（稀疏头自动生成）          | 详细子任务追踪，无需定义高层级阶段                      |
| `dual`         | 稀疏 + 密集标注（VLM） | 双头                            | 完整 SARM 论文设置，兼具两种粒度                        |

### 模式详情

<hfoptions id="mode_explanation">
<hfoption id="single_stage">

**无需任何标注。** 整个回合被视为单个名为 `"task"` 的阶段，进度在回合持续时间内从 0 线性增加到 1。

- **稀疏头**：1 个阶段（"task"），线性进度
- **密集头**：不使用
- **最佳适用**：简单任务、快速实验，或无法使用 VLM 标注时

## 步骤 5（可选）：使用 RA-BC 训练策略

基于奖励的行为克隆（RA-BC，Reward-Aligned Behavior Cloning）使用训练好的 SARM 模型，根据预测的进度改善情况对训练样本进行加权。这需要两个步骤：

1. **预计算进度值**：使用训练好的 SARM 模型计算所有帧的进度值
2. **使用 RA-BC 加权训练策略**：使用预计算的进度值训练策略

### RA-BC 工作原理

对于每个训练样本，RA-BC 计算进度差异（delta）：

```
r_i = φ(o_{t+Δ}) - φ(o_t)
```

其中 `φ` 是 SARM 进度预测，`Δ` 是策略的 `chunk_size`。进度为正（良好演示）的样本获得更高权重，而进度为负或零的样本权重降低。

加权方式遵循论文中的**公式 8-9**：

- **软权重**：`w̃_i = clip((r_i − (μ − 2σ)) / (4σ + ε), 0, 1)`
- **最终权重**：`w_i = 𝟙{r_i > κ} + 𝟙{0 ≤ r_i ≤ κ} × w̃_i`

### 步骤 5a：计算 SARM 进度值

首先，对数据集中所有帧运行 SARM 模型以计算进度值：

```bash
python -m lerobot.rewards.sarm.compute_rabc_weights \
  --dataset-repo-id your-username/your-dataset \
  --reward-model-path your-username/sarm-model \
  --head-mode sparse \
  --num-visualizations 5 \
  --push-to-hub
```

该脚本：

- 处理所有帧并计算进度值
- 将进度值保存到磁盘上数据集旁边的 parquet 文件中（默认为 `<dataset_root>/sarm_progress.parquet`）
- 生成前 N 个回合的可视化结果（默认：5）

**参数：**

| 参数                     | 描述                                          | 默认值      |
| ------------------------ | --------------------------------------------- | ----------- |
| `--reward-model-path`    | 训练好的 SARM 模型路径                        | （必填）    |
| `--head-mode`            | 使用的 SARM 头：`sparse`、`dense` 或 `both`   | `sparse`    |
| `--device`               | 推理设备                                      | `cuda`      |
| `--visualize-only`       | 仅可视化预测（不进行 RA-BC 计算）             | `false`     |
| `--num-visualizations`   | 可视化的回合数（默认：5，设为 0 可跳过）      | `5`         |

**输出格式**（`sarm_progress.parquet`）：

| 列                  | 描述                              |
| ------------------- | --------------------------------- |
| `index`             | 数据集中的全局帧索引              |
| `episode_index`     | 回合编号                          |
| `frame_index`       | 回合内的本地帧索引                |
| `progress_sparse`   | 稀疏头进度值 [0, 1]              |
| `progress_dense`    | 密集头进度值 [0, 1]（如果计算了）|

### 步骤 5b：使用 RA-BC 训练策略

获得进度文件后，即可使用 RA-BC 加权训练策略。进度文件会从数据集路径（`sarm_progress.parquet`）自动检测，除非明确指定。目前 PI0、PI0.5 和 SmolVLA 支持 RA-BC：

```bash
lerobot-train \
  --dataset.repo_id=your-username/your-dataset \
  --policy.type=pi0 \
  --sample_weighting.type=rabc \
  --sample_weighting.head_mode=sparse \
  --sample_weighting.kappa=0.01 \
  --output_dir=outputs/train/policy_rabc \
  --batch_size=32 \
  --steps=40000
```

训练脚本会自动：

- 从 parquet 文件加载预计算的进度值
- 使用策略的 `chunk_size` 计算进度差异（Δ）
- 根据进度改善情况计算样本权重
- 在训练期间应用加权损失

**RA-BC 参数：**

| 参数                               | 描述                                        | 默认值                   |
| ---------------------------------- | ------------------------------------------- | ------------------------ |
| `--sample_weighting.type`          | 加权策略类型（`rabc` 或 `uniform`）         | `rabc`                   |
| `--sample_weighting.progress_path` | 进度 parquet 文件路径                       | `sarm_progress.parquet`  |
| `--sample_weighting.head_mode`     | 使用哪个 SARM 头的进度：`sparse` 或 `dense` | `sparse`                 |
| `--sample_weighting.kappa`         | 高质量样本的阈值 κ                          | `0.01`                   |
| `--sample_weighting.epsilon`       | 数值稳定性的小常数                          | `1e-6`                   |

### 调优 RA-BC Kappa

`kappa` 参数是决定哪些样本获得全权重（w=1）的阈值。理解如何调优它对于 RA-BC 的有效运行至关重要。

**加权方式：**

| 条件                | 权重                     |
| ------------------- | ------------------------ |
| `delta > kappa`     | 1.0（硬阈值）            |
| `0 ≤ delta ≤ kappa` | 公式 8 的软权重          |
| `delta < 0`         | 0.0（负向进度）          |

**诊断 kappa 问题：**

训练期间监控以下 WandB 指标：

| 指标                              | 健康范围    | 问题指示                    |
| --------------------------------- | ----------- | ---------------------------- |
| `sample_weight_mean_weight`       | 0.3 - 0.8   | ≈ 1.0 表示 kappa 过低        |
| `sample_weighting/delta_mean`     | > 0         | 应为正值                     |
| `sample_weighting/delta_std`      | > 0         | 数据质量存在差异             |

**如果 `sample_weight_mean_weight ≈ 1.0`：** 你的 kappa 太低了。大多数样本的 `delta > kappa`，完全绕过了软加权。RA-BC 等同于普通的 BC。

**根据数据设置 kappa：**

默认的 `kappa=0.01` 是针对论文中的 T 恤折叠任务（约 90 秒的回合，30fps）调优的。对于你自己的数据集，请检查记录的 `sample_weighting/delta_mean` 和 `sample_weighting/delta_std`：

```
# 如果 delta_mean ≈ 0.03 且 delta_std ≈ 0.02：
# 大多数 delta 落在 [0.01, 0.05] 范围内

# 选项 1：设置 kappa = delta_mean（中等选择性）
--sample_weighting.kappa=0.03

# 选项 2：设置 kappa = delta_mean + delta_std（高选择性）
--sample_weighting.kappa=0.05

# 选项 3：设置 kappa = delta_mean + 2*delta_std（极高选择性）
--sample_weighting.kappa=0.07
```

**RA-BC 可能无帮助的情况：**

如果你的数据集质量已经很高（所有演示的进度一致），RA-BC 不会带来太多提升，因为没有什么需要过滤的。

## 提示与最佳实践

### 选择模式

- **从 `single_stage` 开始**进行快速实验——无需标注开销
- 在需要详细进度追踪但任务没有明确高层级阶段时，使用 **`dense_only`**
- 对于粗粒度和细粒度进度都有意义的复杂任务，使用 **`dual`**

### 标注质量

1. **子任务名称要具体**：与其用"fold"，不如用"抓住靠近的一侧并向中心折叠"
2. **通过可视化验证**：训练前始终检查几个回合
3. **命名一致**：在所有回合中使用相同的子任务名称

### RA-BC

1. **先训练 SARM**：RA-BC 的质量完全取决于 SARM 的质量
2. **监控 `sample_weight_mean_weight`**：如果 ≈ 1.0，增大 kappa（参见[调优 RA-BC Kappa](#tuning-ra-bc-kappa)）
