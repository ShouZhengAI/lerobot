# 视频编码参数

当启用视频存储时，LeRobot 将每个相机流存储为 **MP4** 文件，而不是每个时间步保存一张图像文件。视频编码跨时间维度进行压缩，通常可以显著减少数据集大小和 I/O 开销，同时保持 MP4——这种格式每个播放器和加载器都支持。

将帧编码为 MP4 是一个完整的 FFmpeg 流水线：编码器选择、像素格式、GOP（图像组）/关键帧、质量与速度的权衡，以及可选的额外编码器标志。这些参数大部分都可以通过 `rgb_encoder`（一个嵌套的 `RGBEncoderConfig`（`lerobot.configs.video.RGBEncoderConfig`））进行用户调节，通过 PyAV 传递。

你可以在 CLI 上使用 `--dataset.rgb_encoder.<field>` 设置这些参数（例如使用 `lerobot-record` 或 `lerobot-rollout`）。同一组设置适用于该次运行中的所有相机视频流。

> [!TIP]
> 必须启用视频存储，`rgb_encoder` 才能生效——
> 在 Python API 中设置 `use_videos=True`，或在 CLI 上设置 `--dataset.video=true`（录制时的默认值）。
> 关闭视频时，输入将保持为图像，`rgb_encoder` 将被忽略。

关于**何时**写入帧与编码帧（流式编码 vs. 片段后编码）、队列以及其他顶层 `--dataset.*` 开关的详细信息，请参见[流式视频编码](./streaming_video_encoding)。有关编码参数对比和实验，请参见[视频基准测试 Space](https://huggingface.co/spaces/lerobot/video-benchmark)。

---

## 示例

```bash
lerobot-record \
    --robot.type=so100_follower \
    --robot.port=/dev/tty.usbmodem58760431541 \
    --robot.cameras="{laptop: {type: opencv, index_or_path: 0, width: 640, height: 480, fps: 30}}" \
    --robot.id=black \
    --teleop.type=so100_leader \
    --teleop.port=/dev/tty.usbmodem58760431551 \
    --teleop.id=blue \
    --dataset.repo_id=<my_username>/<my_dataset_name> \
    --dataset.num_episodes=2 \
    --dataset.single_task="Grab the cube" \
    --dataset.streaming_encoding=true \
    --dataset.encoder_threads=2 \
    --dataset.rgb_encoder.vcodec=h264 \
    --dataset.rgb_encoder.preset=fast \
    --dataset.rgb_encoder.extra_options={"tune": "film", "profile:v": "high", "bf": 2} \
    --display_data=true
```

## 调优参数

> [!WARNING]
> 默认值经过调优，旨在为典型机器人数据集平衡**压缩比**、**视觉质量**和**解码/查找速度**。更改这些参数可能会影响录制（CPU 负载、丢帧）和训练（解码吞吐量、图像质量）。
>
> 仅在你有特定理由时才重写这些参数，并在依赖新设置之前测量其对流水线的影响。

以下所有标志在 CLI 上均以 `--dataset.rgb_encoder.` 为前缀。

| 参数              | 类型               | 默认值             | 描述                                                                                                                                       |
| ----------------- | ------------------ | ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------ |
| `vcodec`          | `str`              | `"libsvtav1"`      | 视频编解码器名称。`"auto"` 从固定优先级列表中选取第一个可用的硬件编码器，若不可用则回退到 `libsvtav1`。                                     |
| `pix_fmt`         | `str`              | `"yuv420p"`        | 输出像素格式。必须被你选择的编解码器在你的 FFmpeg 构建版本中支持。                                                                         |
| `g`               | `int`              | `2`                | GOP 大小——每 `g` 帧一个关键帧。以 FFmpeg 选项 `g` 的形式传递。                                                                             |
| `crf`             | `int` 或 `float`   | `30`               | 抽象质量值，按编解码器映射（见下方的 [映射](#videoencoderconfig--ffmpeg-选项的映射)）。值越低 → 质量越高 / 输出越大（在映射单调的情况下）。 |
| `preset`          | `int` 或 `str`     | `12` \*            | 编码器速度预设；含义取决于编解码器。<br/>\* 当未设置且 `vcodec=libsvtav1` 时，LeRobot 默认为 `12`。                                         |
| `fast_decode`     | `int`              | `0`                | `libsvtav1`：`0–2`，通过 `svtav1-params` 传递。<br/>`h264` / `hevc`（软件）：如果 `>0`，设置 `tune=fastdecode`。<br/>其他编解码器：通常未使用。 |
| `video_backend`   | `str`              | `"pyav"`           | 目前视频编码仅实现了 `"pyav"`。                                                                                                            |
| `extra_options`   | `dict`             | `{}`               | 额外的 FFmpeg 或编解码器特定选项，在结构化字段之后合并。不能覆盖已通过这些字段设置的键。                                                   |

通过并行的 **`depth_encoder`** 块（`DepthEncoderConfig`）配置深度流水线。它拥有 `RGBEncoderConfig` 的所有字段（`vcodec`、`pix_fmt`、`crf` 等），并额外增加了四个量化参数，通过 `--dataset.depth_encoder.<field>` 设置：

```bash
lerobot-record \
    ... \
    --dataset.depth_encoder.vcodec=hevc \
    --dataset.depth_encoder.depth_min=0.05 \
    --dataset.depth_encoder.depth_max=5.0 \
    --dataset.depth_encoder.use_log=true
```

| 参数              | 类型    | 默认值                            | 描述                                                                                                                        |
| ----------------- | ------- | --------------------------------- | --------------------------------------------------------------------------------------------------------------------------- |
| `vcodec`          | `str`   | `"hevc"`                          | HEVC Main 12（一种支持 12 位且兼容 MP4 的编解码器）。                                                                       |
| `extra_options`   | `dict`  | `{"x265-params": "lossless=1"}`   | **深度默认使用无损编码**（精确往返）；`crf` 被忽略。传入 `extra_options={}` 并设置 `crf` 可获得更小的有损流。                 |
| `pix_fmt`         | `str`   | `"gray12le"`                      | 用于承载量化编码的单通道 12 位像素格式。                                                                                    |
| `depth_min`       | `float` | `0.01`                            | 映射到量子值 `0` 的深度（米）。低于此值的深度在解码时会被裁剪。                                                             |
| `depth_max`       | `float` | `10.0`                            | 映射到量子值 `4095` 的深度（米）。高于此值的深度在解码时会被裁剪。                                                          |
| `shift`           | `float` | `3.5`                             | 对数量化中用于数值稳定性的预对数偏移量（米），在零值附近有效。必须满足 `depth_min + shift > 0`。                            |
| `use_log`         | `bool`  | `True`                            | 如果为 `true`，在对数空间中量化（推荐用于典型深度传感器）。设置为 `false` 则使用均匀/线性量化。                             |

> [!TIP]
> `depth_min`、`depth_max` 和 `shift` 始终以**米**为单位解释，无论输入深度的实际单位是什么。输入会自动检测：整数数组（例如直接来自 RealSense 的 `uint16` 毫米值）视为毫米，浮点数组视为米。
> 选择 `depth_min` / `depth_max` 时要覆盖传感器实际的工���范围——超出该范围的量子值会饱和，可能会丢失边界处的细节。

深度特征在 `meta/info.json` 中以 `"is_depth_map": true` 标记，其量化器设置（`video.depth_min`、`video.depth_max`、`video.shift`、`video.use_log`）会被持久化——这使得深度在加载时能够**反量化回物理单位**。

### 加载时的输出单位

`depth_encoder` 是一个**录制时**的关注点。深度图在**加载时**（例如在训练期间）反量化为哪个单位，由读取时的标志 `--dataset.depth_output_unit` 单独设置：

```bash
lerobot-train \
    --dataset.repo_id=<my_username>/<my_dataset_name> \
    --dataset.depth_output_unit=m \
    --policy.type=act
```

| 参数                | 类型  | 默认值 | 描述                                                         |
| ------------------- | ----- | ------ | ------------------------------------------------------------ |
| `depth_output_unit` | `str` | `"mm"` | 深度图在加载时反量化为的物理单位：`"mm"`（毫米）或 `"m"`（米）。 |

> [!TIP]
> 这纯粹是解码时的呈现选择——它**不会**更改存储的视频或其元数据，因此同一数据集可以在不重新编码的情况下以 `mm` 或 `m` 读取。对没有深度相机的数据集没有影响。

## 在数据集元数据中的持久化

在视频流的第一个片段编码完成后，编码器配置会**持久化到数据集元数据**（`meta/info.json`）中，存储在对应的视频特征下，同时还会保存从文件本身探测到的值。对于视频特征 `observation.images.<camera>`，`info.json` 中的结构如下：

```json
{
  "features": {
    "observation.images.laptop": {
      "dtype": "video",
      "shape": [480, 640, 3],
      "info": {
        "video.height": 480,
        "video.width": 640,
        "video.codec": "h264",
        "video.pix_fmt": "yuv420p",
        "video.fps": 30,
        "video.channels": 3,
        "is_depth_map": false,
        "video.g": 2,
        "video.crf": 30,
        "video.preset": "fast",
        "video.fast_decode": 0,
        "video.video_backend": "pyav",
        "video.extra_options": { "tune": "film", "profile:v": "high", "bf": 2 }
      }
    }
  }
}
```

`info` 块由两个来源提供数据：

| 来源                   | 数据来源                           | 字段                                                                                                                 |
| ---------------------- | ---------------------------------- | -------------------------------------------------------------------------------------------------------------------- |
| **流派生（Stream-derived）**   | 使用 PyAV 从编码后的 MP4 中读回。 | `video.height`、`video.width`、`video.codec`、`video.pix_fmt`、`video.fps`、`video.channels`、`is_depth_map`、`audio.*` |
| **编码器派生（Encoder-derived）** | 取自 `RGBEncoderConfig` / `DepthEncoderConfig`。 | `video.g`、`video.crf`、`video.preset`、`video.fast_decode`、`video.video_backend`、`video.extra_options`               |

> [!IMPORTANT]
> 该信息块仅在**第一个**片段时填充。它假定数据集中的每个
> 片段都使用相同的 `rgb_encoder` 编码。不支持在录制过程中途更改
> 编码器设置——`info.json` 仅反映第一个片段使用的参数。

## 合并数据集

当使用 `merge_datasets` 聚合数据集时，视频文件按原样拼接（不重新编码），`info.json` 中的编码器字段按键合并：

| 合并规则           | 字段                                                                            | 行为                                                                                                                 |
| ------------------ | ------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------- |
| **必须匹配**       | `video.codec`、`video.pix_fmt`、`video.height`、`video.width`、`video.fps`       | 流派生字段必须在各来源之间匹配，否则 FFmpeg 的 concat 解复用器会失败。                                                  |
| **宽松合并**       | `video.g`、`video.crf`、`video.preset`、`video.fast_decode`、`video.extra_options` | 编码器调优字段。如果所有来源的值一致，则保留该值；如果不一致，则设为 `null`（对于 `video.extra_options` 设为 `{}`）并记录警告。 |
