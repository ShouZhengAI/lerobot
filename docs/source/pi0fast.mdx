# π₀-FAST (Pi0-FAST)

π₀-FAST 是一个 **面向通用机器人控制的视觉-语言-动作模型**，它使用自回归式的下一个词元预测来建模连续的机器人动作。

## 模型概述

π₀-FAST 将视觉-语言模型的能力与一种新颖的动作词元化方法——**FAST（频域动作序列词元化，Frequency-space Action Sequence Tokenization）** 相结合。这使得我们能够在高灵巧度任务上训练自回归式的视觉-语言-动作模型，而这些任务使用标准的分箱离散化方法无法实现，同时训练速度**比基于扩散的方法（如 π₀）快 5 倍**。

### 为什么选择 FAST？

标准的机器人动作词元化方法使用简单的逐维度、逐时间步分箱方案。对于简单行为尚可接受，但对于需要精度和高频控制的复杂灵巧技能，这种方法很快就会失效。

FAST 通过使用信号处理技术压缩动作序列来解决这一问题，生成密集的动作词元序列，可以像语言词元一样进行自回归预测。

### FAST 词元化的工作原理

FAST 词元化器通过以下步骤压缩动作序列：

1. **归一化**：获取形状为 `(H, D)` 的连续动作块（chunk），其中 `H` 是预测范围（horizon），`D` 是动作维度。使用支持的归一化方法之一进行归一化（推荐使用分位数法来处理异常值）。

2. **离散余弦变换（DCT）**：对每个动作维度分别应用离散余弦变换（通过 scipy）。DCT 是一种常用于图像和音频编解码器（JPEG、MP3）的压缩算法。

3. **量化**：对每个动作维度进行四舍五入并移除不显著的系数，生成稀疏的频率矩阵。

4. **展开**：将矩阵展开为一维向量，低频分量排在前列。

5. **字节对编码（BPE）**：训练一个 BPE 词元化器，将 DCT 系数压缩为密集的动作词元，通常比之前的词元化方法达到 **10 倍压缩率**。

这种方法可以将**任何现有的视觉-语言模型**转化为视觉-语言-动作模型，只需训练它预测这些 FAST 词元。

## 训练自定义 FAST 词元化器

FAST 词元化器有两种选择：

1. **使用预训练的词元化器**：`lerobot/fast-action-tokenizer` 词元化器在 100 万以上的真实机器人动作序列上训练而成，可用作通用词元化器。

2. **训练自己的词元化器**：为了在特定数据集上获得最佳性能，可以在自己的数据上微调词元化器。

### 训练自己的词元化器

```bash
lerobot-train-tokenizer \
    --repo_id "user/my-lerobot-dataset" \
    --action_horizon 10 \
    --encoded_dims "0:6" \
    --vocab_size 1024 \
    --scale 10.0 \
    --normalization_mode QUANTILES \
    --output_dir "./my_fast_tokenizer" \
    --push_to_hub \
    --hub_repo_id "username/my-action-tokenizer"
```

### 关键词元化器参数

| 参数                   | 描述                                   | 默认值       |
| ---------------------- | -------------------------------------- | ------------ |
| `--repo_id`            | LeRobot 数据集仓库 ID                   | 必填         |
| `--action_horizon`     | 每个块中的未来动作数量                   | `10`         |
| `--encoded_dims`       | 要编码的维度范围，逗号分隔（如 `"0:6,7:23"`） | `"0:6,7:23"` |
| `--vocab_size`         | BPE 词汇表大小                         | `1024`       |
| `--scale`              | 量化的 DCT 缩放因子                     | `10.0`       |
| `--normalization_mode` | 归一化模式（`MEAN_STD`, `MIN_MAX`, `QUANTILES`, `QUANTILE10`, `IDENTITY`） | `QUANTILES`  |
| `--sample_fraction`    | 每段中采样的块的比例                    | `0.1`        |

## 使用方法

要在 LeRobot 中使用 π₀-FAST，请将策略类型指定为：

```python
policy.type=pi0_fast
```

## 训练

训练 π₀-FAST 可以使用 LeRobot 训练脚本：

```bash
lerobot-train \
    --dataset.repo_id=your_dataset \
    --policy.type=pi0_fast \
    --output_dir=./outputs/pi0fast_training \
    --job_name=pi0fast_training \
    --policy.pretrained_path=lerobot/pi0fast-base \
    --policy.dtype=bfloat16 \
    --policy.gradient_checkpointing=true \
    --policy.chunk_size=10 \
    --policy.n_action_steps=10 \
    --policy.max_action_tokens=256 \
    --steps=100000 \
    --batch_size=4 \
    --policy.device=cuda
```

### 关键训练参数

| 参数                                  | 描述                                          | 默认值                          |
| ------------------------------------- | --------------------------------------------- | ------------------------------- |
| `--policy.gradient_checkpointing=true` | 显著降低训练时的内存占用                         | `false`                         |
| `--policy.dtype=bfloat16`             | 使用混合精度训练以提高效率                       | `float32`                       |
| `--policy.chunk_size`                 | 预测的动作步数（动作预测范围）                    | `50`                            |
| `--policy.n_action_steps`             | 执行的动作步数                                 | `50`                            |
| `--policy.max_action_tokens`          | 每个动作块的最大 FAST 词元数                    | `256`                           |
| `--policy.action_tokenizer_name`      | 要使用的 FAST 词元化器                         | `lerobot/fast-action-tokenizer` |
| `--policy.compile_model=true`         | 启用 torch.compile 以加速训练                  | `false`                         |

## 推理

### KV 缓存加速推理

π₀-FAST 支持 **KV 缓存（KV-caching）**，这是大语言模型推理中广泛使用的优化技术。它会缓存注意力机制中的键-值对，避免自回归解码时重复计算。

```python
# KV 缓存默认启用
policy.use_kv_cache=true
```

### 推理示例

```python
from lerobot.policies.pi0_fast import PI0FastPolicy, PI0FastConfig

# 加载策略
policy = PI0FastPolicy.from_pretrained("your-model-path")

# 推理时
actions = policy.predict_action_chunk(batch)
```

## 模型架构

π₀-FAST 采用基于 PaliGemma 的架构：

- **视觉编码器**：用于图像理解的 SigLIP 视觉塔
- **语言模型**：用于处理语言指令和预测动作词元的 Gemma 2B

模型接收图像、文本指令和机器人状态作为输入，输出离散的 FAST 词元，这些词元再被解码为连续的动作。

## 配置选项

| 参数                 | 描述                              | 默认值       |
| -------------------- | --------------------------------- | ------------ |
| `paligemma_variant`  | VLM 骨干网络变体（`gemma_300m`, `gemma_2b`） | `gemma_2b` |
| `max_state_dim`      | 最大状态向量维度（会进行填充）                   | `32`         |
| `max_action_dim`     | 最大动作向量维度（会进行填充）                   | `32`         |
| `temperature`        | 采样温度（0.0 表示贪心解码）                    | `0.0`        |
| `max_decoding_steps` | 最大解码步数                                   | `256`        |
| `use_kv_cache`       | 启用 KV 缓存以加速推理                         | `true`       |

## 与 π₀ 的对比

| 特性           | π₀                              | π₀-FAST                            |
| -------------- | ------------------------------- | ----------------------------------- |
| 动作表示方式   | 流匹配（扩散）                   | 自回归词元（FAST）                   |
| 训练速度       | 1x                              | **快 5倍**                          |
| 灵巧度         | 高                              | 高                                  |
| 推理方法       | 迭代去噪                         | 自回归解码                           |
| KV 缓存        | 不支持                           | 支持                                |

## 复现 π₀Fast 的结果

我们使用 LeRobot 实现，在 LIBERO 基准上复现了 π₀Fast 的结果。我们采用 LeRobot 的 PiFast 基础模型 [lerobot/pi0fast-base](https://huggingface.co/lerobot/pi0fast-base)，使用 [HuggingFace LIBERO 数据集](https://huggingface.co/datasets/HuggingFaceVLA/libero)，在 8 块 H100 GPU 上以 bfloat16 精度、批次大小 256 进行额外 40k 步的微调。

微调后的模型可在此处找到：

- **π₀Fast LIBERO**: [lerobot/pi0fast-libero](https://huggingface.co/lerobot/pi0fast-libero)

使用以下训练命令：

```bash
lerobot-train \
  --dataset.repo_id=lerobot/libero \
  --output_dir=outputs/libero_pi0fast \
  --job_name=libero_pi0fast \
  --policy.path=lerobot/pi0fast-base \
  --policy.dtype=bfloat16 \
  --steps=100000 \
  --save_freq=20000 \
  --batch_size=4 \
  --policy.device=cuda \
  --policy.scheduler_warmup_steps=4000 \
  --policy.scheduler_decay_steps=100000 \
  --policy.scheduler_decay_lr=1e-5 \
  --policy.gradient_checkpointing=true \
  --policy.chunk_size=10 \
  --policy.n_action_steps=10 \
  --policy.max_action_tokens=256 \
  --policy.empty_cameras=1 \
```

然后，我们使用 LeRobot 的 LIBERO 实现，通过运行以下命令来评估微调后的模型：

```bash
tasks="libero_object,libero_spatial,libero_goal,libero_10"
lerobot-eval \
  --policy.path=lerobot/pi0fast-libero \
  --policy.max_action_tokens=256 \
  --env.type=libero \
  --policy.gradient_checkpointing=false \
  --env.task=${tasks} \
  --eval.batch_size=1 \
  --eval.n_episodes=1 \
  --rename_map='{"observation.images.image":"observation.images.base_0_rgb","observation.images.image2":"observation.images.left_wrist_0_rgb"}'
```

**注意：** 我们设置 `n_action_steps=10`，与原始 OpenPI 实现保持一致。
