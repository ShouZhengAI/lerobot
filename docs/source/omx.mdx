## 订购和组装零件

首先，按照官方组装指南组装 OMX 硬件。

OMX 组装指南：https://ai.robotis.com/omx/assembly_guide_omx.html

OMX 机器人在出厂时已完成预配置。电机 ID、通信参数和关节偏移量均已设置完成，因此在使用 LeRobot 前无需额外的电机设置或标定步骤。

## 安装 LeRobot 🤗

要安装 LeRobot，请遵循我们的[安装指南](./installation)

除此之外，您还需要安装 Dynamixel SDK：

```bash
pip install -e ".[dynamixel]"
```

## 连接机器人

要查找每个总线舵机适配器的端口，请运行以下脚本：

```bash
lerobot-find-port
```

运行此命令后，在提示时断开主臂或从臂的 USB 线缆并按下 Enter。输出将显示 'The port of this MotorsBus is [port]'（该电机总线端口为 [port]），标识已断开连接的机械臂对应的端口。对另一只机械臂重复此步骤以识别两个端口。

<hfoptions id="find_port">
<hfoption id="Mac">

macOS 上的示例输出：

```
Finding all available ports for the MotorBus.
['/dev/tty.usbmodem575E0032081', '/dev/tty.usbmodem575E0031751']
Remove the USB cable from your MotorsBus and press Enter when done.

[...断开对应的主臂或从臂并按下 Enter...]

The port of this MotorsBus is /dev/tty.usbmodem575E0032081
Reconnect the USB cable.
```

找到的端口为：`/dev/tty.usbmodem575E0032081`，对应您的主臂或从臂。

</hfoption>
<hfoption id="Linux">

在 Linux 上，我们强烈建议使用 udev 规则为 OMX 主臂和从臂分配持久且人类可读的设备名称。这可以避免当机器人被拔插或系统重启时，ttyACM0 和 ttyACM1 等设备名称发生变化的问题。

#### 1. 查找设备序列号

您应该已使用 `lerobot-find-port` 获得了主臂和从臂的端口号（如 ../../ttyACM?）。您可以使用 `ls -l /dev/serial/by-id/` 命令将这些结果与序列号进行匹配。要创建 udev 规则，您需要每个 OMX 设备唯一的序列号。最简单的方法是列出以下目录下的设备：

```bash
ls -l /dev/serial/by-id/
```

您将看到类似以下的输出：

```bash
usb-ROBOTIS_OpenRB-150_228BDD7B503059384C2E3120FF0A2B19-if00 -> ../../ttyACM0
usb-ROBOTIS_OpenRB-150_67E1ED68503059384C2E3120FF092234-if00 -> ../../ttyACM1
```

在每行中，序列号是 `usb-ROBOTIS_OpenRB-150_` 之后、`-if00` 之前的长字符串。

从臂序列号：`228BDD7B503059384C2E3120FF0A2B19`

主臂序列号：`67E1ED68503059384C2E3120FF092234`

#### 2. 创建 udev 规则

创建一个新的 udev 规则文件：

```bash
sudo nano /etc/udev/rules.d/99-omx.rules
```

粘贴以下内容，将序列号替换为您找到的值：

```bash
SUBSYSTEM=="tty", ATTRS{idVendor}=="0403", ATTRS{serial}=="228BDD7B503059384C2E3120FF0A2B19", SYMLINK+="omx_follower"
SUBSYSTEM=="tty", ATTRS{idVendor}=="0403", ATTRS{serial}=="67E1ED68503059384C2E3120FF092234", SYMLINK+="omx_leader"
```

保存文件并重新加载 udev 规则：

```bash
sudo udevadm control --reload-rules
sudo udevadm trigger
```

然后拔下并重新插入两个设备各一次。

#### 3. 验证符号链接

检查持久设备名称是否存在：

```bash
ls -l /dev/omx_follower /dev/omx_leader
```

您应该看到它们指向 ttyACM* 设备：

```bash
/dev/omx_follower -> ttyACM*
/dev/omx_leader   -> ttyACM*
```

这些名称在重启和重新连接后仍将保持稳定。

## 遥操作

识别正确的端口后，您可以直接使用主臂遥操作从臂。

<hfoptions id="teleoperate">
<hfoption id="Mac">

### 不带摄像头的遥操作

```bash
lerobot-teleoperate \
  --robot.type=omx_follower \
  --robot.port=<your_follower_port> \
  --robot.id=omx_follower_arm \
  --teleop.type=omx_leader \
  --teleop.port=<your_leader_port> \
  --teleop.id=omx_leader_arm
```

在遥操作过程中，主臂的运动会被从臂实时镜像。OMX 已预先配置好，无需任何标定步骤即可立即开始遥操作。

### 带摄像头的遥操作

您还可以通过为从臂提供摄像头配置，在遥操作期间启用摄像头输入。

```bash
lerobot-teleoperate \
  --robot.type=omx_follower \
  --robot.port=<your_follower_port> \
  --robot.id=omx_follower_arm \
  --robot.cameras="{front: {type: opencv, index_or_path: '/dev/video0', width: 640, height: 480, fps: 30}}" \
  --teleop.type=omx_leader \
  --teleop.port=<your_leader_port> \
  --teleop.id=omx_leader_arm \
  --display_data=true
```

启用摄像头后，摄像头画面会实时显示，并与机器人状态同步。此设置对于视觉监控非常有用，稍后还可用于演示记录和模仿学习。
