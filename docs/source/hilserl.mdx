# HIL-SERL 真实机器人训练工作流程指南

在本教程中，您将使用 LeRobot 完成完整的"人在环路-样本高效强化学习"（Human-in-the-Loop Sample-Efficient Reinforcement Learning，简称 HIL-SERL）工作流程。您将掌握在真实机器人上仅用几个小时就用强化学习（Reinforcement Learning，简称 RL）训练策略的技能。

HIL-SERL 是一种样本高效的强化学习算法，它结合了人类演示、在线学习和人类干预。该方法从少量人类演示开始，使用这些演示训练一个奖励分类器（reward classifier），然后采用"演员-学习者"（actor-learner）架构，在策略执行过程中人类可以进行干预以引导探索并纠正不安全行为。在本教程中，您将使用游戏手柄进行干预并在学习过程中控制机器人。

它结合了三个关键要素：

1. **离线演示和奖励分类器：** 少量的人类远程操控回合加上一个基于视觉的成功检测器，为策略提供一个特征化的起点。

2. **带人类干预的机器上演员/学习者循环：** 一种分布式 Soft Actor Critic（SAC）学习器更新策略，同时演员在实际机器人上进行探索；人类可以随时介入以纠正危险或无效的行为。

3. **安全与效率工具：** 关节/末端执行器（end-effector，简称 EE）边界、感兴趣区域（region of interest，简称 ROI）裁剪预处理和 WandB 监控，确保数据有效且硬件安全。

这些元素共同作用，使 HIL-SERL 能够达到近乎完美的任务成功率，并且周期时间比纯模仿学习基线更短。

## 需要什么？

- 一个游戏手柄（推荐）或键盘来控制机器人
- 一块 Nvidia GPU
- 带有从属臂（follower）和领导臂（leader）的真实机器人（如果使用键盘或游戏手柄则可选）
- 用于运动学包的机器人 URDF 文件（请查看 `lerobot/model/kinematics.py`）

## 可以训练什么样的任务？

可以使用 HIL-SERL 训练各种操作任务。一些建议：

- 从一个简单的任务开始，了解系统的工作原理。
  - 将方块推到目标区域
  - 用夹爪拿起并举起方块
- 避免非常长时域（long horizon）的任务。专注于可以在 5-10 秒内完成的任务。
- 对系统的工作方式有了较好的了解后，可以尝试更复杂的任务和更长的时域。
  - 拿起并放置方块
  - 双臂操作任务，用两只手臂取物体
  - 交接任务，将物体从一只手臂转移到另一只手臂
  - 尽情发挥！

## 使用 HIL-SERL 安装 LeRobot

要安装带有 HIL-SERL 的 LeRobot，需要安装 `hilserl` 额外依赖。

```bash
pip install -e ".[hilserl]"
```

## 真实机器人训练工作流程

### 理解配置

训练过程从正确配置 HILSERl 环境开始。主要的配置类是 `lerobot/rl/gym_manipulator.py` 中的 `GymManipulatorConfig`，它包含了嵌套的 `HILSerlRobotEnvConfig`（定义在 `lerobot/envs/configs.py` 中）和 `DatasetConfig`。配置被组织成聚焦的嵌套子配置：

<!-- prettier-ignore-start -->
```python
class GymManipulatorConfig:
    env: HILSerlRobotEnvConfig    # 环境配置（嵌套）
    dataset: DatasetConfig    # 数据集录制/回放配置（嵌套）
    mode: str | None = None    # "record"、"replay"或 None（用于训练）
    device: str = "cpu"    # 计算设备

class HILSerlRobotEnvConfig(EnvConfig):
    robot: RobotConfig | None = None    # 主要机器人智能体（定义在 `lerobot/robots`）
    teleop: TeleoperatorConfig | None = None    # 远程操控设备，如游戏手柄或领导臂
    processor: HILSerlProcessorConfig    # 处理流水线配置（嵌套）
    name: str = "real_robot"    # 环境名称
    task: str | None = None    # 任务标识符
    fps: int = 10    # 控制频率

# 嵌套的处理器配置
class HILSerlProcessorConfig:
    control_mode: str = "gamepad"    # 控制模式
    observation: ObservationConfig | None = None    # 观测处理设置
    image_preprocessing: ImagePreprocessingConfig | None = None    # 图像裁剪/缩放设置
    gripper: GripperConfig | None = None    # 夹爪控制和惩罚设置
    reset: ResetConfig | None = None    # 环境重置和计时设置
    inverse_kinematics: InverseKinematicsConfig | None = None    # 逆运动学处理设置
    reward_classifier: RewardClassifierConfig | None = None    # 奖励分类器设置
    max_gripper_pos: float | None = 100.0    # 最大夹爪位置

# 子配置类
class ObservationConfig:
    add_joint_velocity_to_observation: bool = False    # 将关节速度添加到状态中
    add_current_to_observation: bool = False    # 将电机电流添加到状态中
    add_ee_pose_to_observation: bool = False    # 将末端执行器位姿添加到状态中
    display_cameras: bool = False    # 执行时显示摄像头画面

class ImagePreprocessingConfig:
    crop_params_dict: dict[str, tuple[int, int, int, int]] | None = None    # 图像裁剪参数
    resize_size: tuple[int, int] | None = None    # 目标图像尺寸

class GripperConfig:
    use_gripper: bool = True    # 启用夹爪控制
    gripper_penalty: float = 0.0    # 不当使用夹爪的惩罚

class ResetConfig:
    fixed_reset_joint_positions: Any | None = None    # 用于重置的关节位置
    reset_time_s: float = 5.0    # 重置等待时间
    control_time_s: float = 20.0    # 最大回合持续时间
    terminate_on_success: bool = True    # 检测到成功后是否终止回合

class InverseKinematicsConfig:
    urdf_path: str | None = None    # 机器人 URDF 文件路径
    target_frame_name: str | None = None    # 末端执行器坐标系名称
    end_effector_bounds: dict[str, list[float]] | None = None    # EE 工作空间边界
    end_effector_step_sizes: dict[str, float] | None = None    # 每轴 EE 步长

class RewardClassifierConfig:
    pretrained_path: str | None = None    # 预训练奖励分类器的路径
    success_threshold: float = 0.5    # 成功检测阈值
    success_reward: float = 1.0    # 成功回合的奖励值

# 数据集配置
class DatasetConfig:
    repo_id: str    # LeRobot 数据集仓库 ID
    task: str    # 任务标识符
    root: str | None = None    # 本地数据集根目录
    num_episodes_to_record: int = 5    # 录制的回合数
    replay_episode: int | None = None    # 回放的回合索引
    push_to_hub: bool = False    # 是否将数据集推送到 Hub
```
<!-- prettier-ignore-end -->

### 处理器流水线架构

HIL-SERL 使用模块化的处理器流水线架构，通过一系列可组合的步骤处理机器人观测和动作。流水线分为两个主要组件：

#### 环境处理器流水线

环境处理器（`env_processor`）处理传入的观测和环境状态：

1. **VanillaObservationProcessorStep**：将原始机器人观测转换为标准化格式
2. **JointVelocityProcessorStep**（可选）：将关节速度信息添加到观测中
3. **MotorCurrentProcessorStep**（可选）：将电机电流读数添加到观测中
4. **ForwardKinematicsJointsToEE**（可选）：从关节位置计算末端执行器位姿
5. **ImageCropResizeProcessorStep**（可选）：裁剪和缩放摄像头图像
6. **TimeLimitProcessorStep**（可选）：强制执行回合时间限制
7. **GripperPenaltyProcessorStep**（可选）：对不当使用夹爪施加惩罚
8. **RewardClassifierProcessorStep**（可选）：使用视觉模型自动化奖励检测
9. **AddBatchDimensionProcessorStep**：将数据转换为批次格式以供神经网络处理
10. **DeviceProcessorStep**：将数据移动到指定的计算设备（CPU/GPU）

#### 动作处理器流水线

动作处理器（`action_processor`）处理输出的动作和人类干预：

1. **AddTeleopActionAsComplimentaryDataStep**：捕获远程操控动作以记录日志
2. **AddTeleopEventsAsInfoStep**：记录干预事件和回合控制信号
3. **InterventionActionProcessorStep**：处理人类干预和回合终止
4. **逆运动学流水线**（启用时）：
   - **MapDeltaActionToRobotActionStep**：将增量动作转换为机器人动作格式
   - **EEReferenceAndDelta**：计算末端执行器参考和增量运动
   - **EEBoundsAndSafety**：强制执行工作空间安全边界
   - **InverseKinematicsEEToJoints**：将末端执行器动作转换为关节目标
   - **GripperVelocityToJoint**：处理夹爪控制命令

#### 配置示例

**基本观测处理**：

```json
{
  "env": {
    "processor": {
      "observation": {
        "add_joint_velocity_to_observation": true,
        "add_current_to_observation": false,
        "display_cameras": false
      }
    }
  }
}
```

**图像处理**：

```json
{
  "env": {
    "processor": {
      "image_preprocessing": {
        "crop_params_dict": {
          "observation.images.front": [180, 250, 120, 150],
          "observation.images.side": [180, 207, 180, 200]
        },
        "resize_size": [128, 128]
      }
    }
  }
}
```

**逆运动学设置**：

```json
{
  "env": {
    "processor": {
      "inverse_kinematics": {
        "urdf_path": "path/to/robot.urdf",
        "target_frame_name": "end_effector",
        "end_effector_bounds": {
          "min": [0.16, -0.08, 0.03],
          "max": [0.24, 0.2, 0.1]
        },
        "end_effector_step_sizes": {
          "x": 0.02,
          "y": 0.02,
          "z": 0.02
        }
      }
    }
  }
}
```

### 高级观测处理

HIL-SERL 框架支持额外的观测处理功能，可以改进策略学习：

#### 关节速度处理

启用关节速度估计以为策略提供运动信息：

```json
{
  "env": {
    "processor": {
      "observation": {
        "add_joint_velocity_to_observation": true
      }
    }
  }
}
```

该处理器：

- 使用连续关节位置读数之间的有限差分估计关节速度
- 将速度信息添加到观测状态向量中
- 对于需要运动感知的动态任务策略非常有用

#### 电机电流处理

监控电机电流以检测接触力和负载条件：

```json
{
  "env": {
    "processor": {
      "observation": {
        "add_current_to_observation": true
      }
    }
  }
}
```

该处理器：

- 从机器人的控制系统读取电机电流值
- 将电流测量值添加到观测状态向量中
- 帮助检测接触事件、物体重量和机械阻力
- 对于接触密集的操作任务非常有用

#### 组合观测处理

可以同时启用多个观测处理功能：

```json
{
  "env": {
    "processor": {
      "observation": {
        "add_joint_velocity_to_observation": true,
        "add_current_to_observation": true,
        "display_cameras": false
      }
    }
  }
}
```

**注意**：启用额外的观测功能会增加状态空间的维度，这可能需要调整策略网络架构并可能收集更多的训练数据。

### 寻找机器人工作空间边界

在收集演示之前，需要确定机器人的适当操作边界。

这通过两种方式帮助简化在真实机器人上学习的问题：1）将机器人的操作空间限制在解决任务所需的特定区域，避免不必要的或不安全的探索；2）允许在末端执行器空间而非关节空间中进行训练。经验表明，在关节空间中进行操作任务的强化学习通常更难——某些任务在关节空间中几乎无法学习，但当动作空间转换为末端执行器坐标后变得可学习。

**使用 lerobot-find-joint-limits**

此脚本帮助您找到机器人末端执行器的安全操作边界。假设您同时有从属臂和领导臂，可以使用该脚本找到训练时将应用的从属臂边界。限制动作空间可以减少智能体的冗余探索并保证安全。

```bash
lerobot-find-joint-limits \
  --robot.type=so100_follower \
  --robot.port=/dev/tty.usbmodem58760431541 \
  --robot.id=black \
  --teleop.type=so100_leader \
  --teleop.port=/dev/tty.usbmodem58760431551 \
  --teleop.id=blue
```

**工作流程**

1. 运行脚本并在求解任务的空间中移动机器人
2. 脚本将记录最小和最大末端执行器位置以及关节角度，并将它们打印到控制台，例如：
   ```
   Max ee position [0.2417 0.2012 0.1027]
   Min ee position [0.1663 -0.0823 0.0336]
   Max joint positions [-20.0, -20.0, -20.0, -20.0, -20.0, -20.0]
   Min joint positions [50.0, 50.0, 50.0, 50.0, 50.0, 50.0]
   ```
3. 在环境配置的 `env.processor.inverse_kinematics.end_effector_bounds` 中使用这些值（参见 `lerobot/envs/configs.py` 中的 `InverseKinematicsConfig`）

**配置示例**

```json
{
  "env": {
    "processor": {
      "inverse_kinematics": {
        "end_effector_bounds": {
          "max": [0.24, 0.2, 0.1],
          "min": [0.16, -0.08, 0.03]
        }
      }
    }
  }
}
```

### 收集演示

确定了边界后，就可以安全地收集训练所需的演示了。使用离线策略算法进行 RL 训练使我们能够利用收集到的离线数据集来提高学习过程的效率。

**设置录制模式**

创建一个用于录制演示的配置文件（或编辑现有文件，如 [env_config.json](https://huggingface.co/datasets/lerobot/config_examples/resolve/main/rl/env_config.json)）：

1. 在根级别将 `mode` 设置为 `"record"`
2. 在 `dataset` 部分指定一个唯一的 `repo_id` 作为数据集名称（例如，"username/task_name"）
3. 在 `dataset` 部分将 `num_episodes_to_record` 设置为要收集的演示数量
4. 先将 `env.processor.image_preprocessing.crop_params_dict` 设置为 `{}`（我们稍后确定裁剪参数）
5. 在 `env` 部分配置 `env.robot`、`env.teleop` 和其他硬件设置

配置示例：

```json
{
  "env": {
    "type": "gym_manipulator",
    "name": "real_robot",
    "fps": 10,
    "processor": {
      "control_mode": "gamepad",
      "observation": {
        "display_cameras": false
      },
      "image_preprocessing": {
        "crop_params_dict": {},
        "resize_size": [128, 128]
      },
      "gripper": {
        "use_gripper": true,
        "gripper_penalty": 0.0
      },
      "reset": {
        "reset_time_s": 5.0,
        "control_time_s": 20.0
      }
    },
    "robot": {
      // ... 机器人配置 ...
    },
    "teleop": {
      // ... 远程操控配置 ...
    }
  },
  "dataset": {
    "repo_id": "username/pick_lift_cube",
    "root": null,
    "task": "pick_and_lift",
    "num_episodes_to_record": 15,
    "replay_episode": 0,
    "push_to_hub": true
  },
  "mode": "record",
  "device": "cpu"
}
```

### 使用远程操控设备

除了机器人之外，还需要一个远程操控设备来控制它，以便收集任务数据集并在在线训练期间执行干预。我们支持使用游戏手柄、键盘或机器人的领导臂。

HIL-SERL 在机器人的末端执行器空间中学习动作。因此，远程操控将控制末端执行器的 x、y、z 位移。

末端执行器变换由处理器流水线（`InverseKinematicsRLStep`、`EEBoundsAndSafety`、`EEReferenceAndDelta`、`GripperVelocityToJoint`）应用，在 `env.processor.inverse_kinematics`（`InverseKinematicsConfig`）和 `env.processor.gripper` / `env.processor.max_gripper_pos` 下配置。与末端执行器空间相关的默认值为：

<!-- prettier-ignore-start -->
```python
class InverseKinematicsConfig:
    """逆运动学处理的配置。"""

    urdf_path: str | None = None
    target_frame_name: str | None = None
    # 末端执行器在 x、y、z 方向上的边界
    end_effector_bounds: dict[str, list[float]] | None = None
    # 末端执行器在 x、y、z 方向上的最大步长
    end_effector_step_sizes: dict[str, float] | None = None

class HILSerlProcessorConfig:
    ...
    # 夹爪完全打开时的最大位置
    max_gripper_pos: float | None = 100.0
```
<!-- prettier-ignore-end -->

`Teleoperator` 定义了远程操控设备。您可以查看 `lerobot/teleoperators` 中可用的远程操控设备列表。

游戏手柄提供了一种非常方便的方式来控制机器人和回合状态。

```json
    "teleop": {
      "type": "gamepad",
      "use_gripper": true
    },
    "processor": {
      "control_mode": "gamepad",
      "gripper": {
        "use_gripper": true
      }
    }
  }
}
```

**设置 SO101 领导臂**

SO101 领导臂具有减速齿轮，使其能够在探索期间移动并跟踪从属臂。因此，接管操作比无齿轮的 SO100 更加平滑。

要设置 SO101 领导臂，需要将 `control_mode` 设置为 `"leader"`，并在配置文件中定义 `teleop` 部分。

```json
{
  "env": {
    "teleop": {
      "type": "so101_leader",
      "port": "/dev/tty.usbmodem585A0077921",
      "use_degrees": true
    },
    "processor": {
      "control_mode": "leader",
      "gripper": {
        "use_gripper": true
      }
    }
  }
}
```

为了标注回合的成功/失败，**您需要**使用键盘按 `s` 键表示成功，按 `esc` 键表示失败。
在在线训练期间，按 `space` 键接管策略，再按一次 `space` 键将控制权交还给策略。

**录制演示**

启动录制过程，示例配置文件见[此处](https://huggingface.co/datasets/aractingi/lerobot-example-config-files/blob/main/env_config_so100.json)：

```bash
python -m lerobot.rl.gym_manipulator --config_path src/lerobot/configs/env_config_so100.json
```

录制过程中：

1. 机器人将重置到配置文件中定义的初始位置 `env.processor.reset.fixed_reset_joint_positions`
2. 成功完成任务
3. 按下"成功"按钮时，回合以奖励为 1 结束
4. 如果达到时间限制，或按下"失败"按钮，回合以奖励为 0 结束
5. 可以通过按下"重新录制"按钮重新录制回合
6. 过程自动继续到下一个回合
7. 录制完所有回合后，数据集被推送到 Hugging Face Hub（可选）并保存到本地

### 处理数据集

收集完演示后，对其进行处理以确定最佳的摄像头裁剪参数。
强化学习对背景干扰很敏感，因此将图像裁剪到相关工作空间区域非常重要。

基于视觉的 RL 算法直接从像素输入学习，使其容易受到无关视觉信息的影响。背景元素如变化的照明、阴影、人员移动或工作空间外的物体都可能干扰学习过程。良好的 ROI 选择应该：

- 仅包含任务发生的关键工作空间
- 捕获机器人的末端执行器和任务中涉及的所有物体
- 排除不必要的背景元素和干扰

注意：如果您已经知道裁剪参数，可以跳过此步骤，只需在录制时直接在配置文件中设置 `crop_params_dict`。

**确定裁剪参数**

使用 `crop_dataset_roi.py` 脚本交互式地在摄像头图像中选择感兴趣区域：

```bash
python -m lerobot.rl.crop_dataset_roi --repo-id username/pick_lift_cube
```

1. 对于每个摄像头视角，脚本将显示第一帧
2. 在相关工作空间区域周围绘制一个矩形
3. 按 'c' 键确认选择
4. 对所有摄像头视角重复此步骤
5. 脚本输出裁剪参数并创建一个新的裁剪过的数据集

**推荐的图像分辨率**

### 训练奖励分类器

奖励分类器在 HIL-SERL 工作流程中扮演着重要角色，它通过自动分配奖励和自动检测回合成功来工作。无需手动定义奖励函数或依赖每个时间步的人工反馈，奖励分类器学习从视觉观测预测成功/失败。这使得 RL 算法能够基于机器人的摄像头输入提供一致且自动的奖励信号，从而高效地学习。

本指南介绍了如何为 LeRobot 的人在环路强化学习实现训练奖励分类器。奖励分类器学习根据给定的状态预测奖励值，这可以在 RL 设置中用于训练策略。

**注意**：训练奖励分类器是可选的。您可以先通过游戏手柄或键盘设备手动标注成功/失败来开始第一轮 RL 实验。

奖励分类器实现位于 `lerobot/rewards/classifier/modeling_classifier.py`，它使用预训练的视觉模型处理图像。它可以输出单个值用于二元奖励（预测成功/失败情况），也可以输出多个值用于多类别设置。

**为奖励分类器收集数据集**

在训练之前，需要收集一个带有标签示例的数据集。在配置中设置 `mode: "record"` 并运行 `gym_manipulator.py` 即可收集包含观测、动作和奖励的数据集。

要收集数据集，需要根据 HILSerlRobotEnvConfig 修改环境配置中的一些参数。

```bash
python -m lerobot.rl.gym_manipulator --config_path src/lerobot/configs/reward_classifier_train_config.json
```

**数据收集的关键参数**

- **mode**：设置为 `"record"` 以收集数据集（根级别）
- **dataset.repo_id**：`"hf_username/dataset_name"`，数据集名称和 Hub 上的仓库名
- **dataset.num_episodes_to_record**：要录制的回合数
- **env.processor.reset.terminate_on_success**：检测到成功时是否自动终止回合（默认值：`true`）
- **env.fps**：每秒录制的帧数
- **dataset.push_to_hub**：是否将数据集推送到 Hub

`env.processor.reset.terminate_on_success` 参数允许您控制回合终止行为。当设置为 `false` 时，即使在检测到成功后回合也会继续，允许您收集更多带有 reward=1 标签的正样本。这对于训练奖励分类器至关重要，因为它会在数据集中提供更多的成功状态示例。当设置为 `true`（默认值）时，检测到成功后回合会立即终止。

**重要提示**：对于奖励分类器训练，请设置 `terminate_on_success: false` 以收集足够的正样本。对于常规的 HIL-SERL 训练，保持为 `true` 以在任务成功完成后自动终止回合。

### 使用演员-学习者模式训练

LeRobot 系统使用分布式的演员-学习者（actor-learner）架构进行训练。这种架构将机器人交互与学习过程解耦，使它们能够并发运行而不互相阻塞。演员（actor）服务器处理机器人观测和动作，将交互数据发送给学习者（learner）服务器。学习者服务器执行梯度下降并定期更新演员的策略权重。您需要启动两个进程：一个学习者和一个演员。

**配置设置**

创建一个训练配置文件（示例见[此处](https://huggingface.co/datasets/lerobot/config_examples/resolve/main/rl/train_config.json)）。训练配置基于 `lerobot/rl/train_rl.py` 中的主 `TrainRLServerPipelineConfig` 类。

1. 配置策略设置（`type="gaussian_actor"`、`device` 等）
2. 在顶层的 `algorithm` 块下配置算法设置（`type="sac"`、学习率、折扣因子等，定义在 `lerobot/rl/algorithms/sac/configuration_sac.py` 中）
3. 将 `dataset` 设置为裁剪后的数据集
4. 配置环境设置，包含裁剪参数
5. 在 [configuration_gaussian_actor.py](https://github.com/huggingface/lerobot/blob/main/src/lerobot/policies/gaussian_actor/configuration_gaussian_actor.py#L79) 中检查与 Gaussian Actor 相关的其他参数
6. 验证 `policy` 配置是否正确，确保 `input_features` 和 `output_features` 与您的任务匹配

**启动学习者**

首先，启动学习者服务器进程：

```bash
python -m lerobot.rl.learner --config_path src/lerobot/configs/train_config_hilserl_so100.json
```

学习者会：

- 初始化策略网络
- 准备回放缓冲区（replay buffer）
- 打开一个 `gRPC` 服务器与演员通信
- 处理转移（transition）并更新策略

**启动演员**

在另一个终端中，使用相同的配置启动演员进程：

```bash
python -m lerobot.rl.actor --config_path src/lerobot/configs/train_config_hilserl_so100.json
```

演员会：

- 通过 `gRPC` 连接到学习者
- 初始化环境
- 执行策略的 rollout 以收集经验
- 将转移发送给学习者
- 接收更新后的策略参数

**训练流程**

训练自动进行：

1. 演员在环境中执行策略
2. 收集转移并发送给学习者
3. 学习者基于这些转移更新策略
4. 更新后的策略参数被发送回演员
5. 该过程持续进行，直到达到指定的步数限制

**人在环路**

- 高效学习的关键在于人类干预，以提供纠正性反馈并完成任务，帮助策略学习和探索。
- 要执行人类干预，可以按下游戏手柄的右上扳机键（或键盘上的 `space` 键）。这将暂停策略动作并允许您接管。
- 一个成功的实验表现为：一开始人类需要频繁干预，但随着策略的改进，干预次数逐渐减少。您可以在 `wandb` 仪表板中监控干预率。

### 人类干预指南

学习过程对干预策略非常敏感。通常需要几次运行才能理解如何有效干预。一些技巧和提示：

- 在训练开始时，允许策略探索几个回合。
- 避免长时间干预。尽量在机器人偏离轨道时进行干预以纠正其行为。
- 一旦策略开始完成任务，即使不完美，您也可以将干预限制在简单的快速动作上，比如简单的抓取命令。

### 需要调整的关键超参数

一些配置值对训练稳定性和速度有不成比例的影响：

- **`temperature_init`**（`algorithm.temperature_init`）—— SAC 中的初始熵温度。较高的值鼓励更多探索；较低的值使策略在早期更加确定性。一个好的起点是 `1e-2`。我们观察到设置过高会使人类干预失效并减慢学习速度。
- **`policy_parameters_push_frequency`**（`policy.actor_learner_config.policy_parameters_push_frequency`）—— 学习者向演员推送两次权重之间的间隔（以_秒_为单位）。默认值为 `4 秒`。减小到 **1-2 秒** 可以提供更新鲜的权重（代价是增加网络流量）；只有在连接较慢时才增加此值，因为这会降低样本效率。
- **`storage_device`**（`policy.storage_device`）—— 学习者存储策略参数的设备。如果 GPU 内存有余，将其设置为 `"cuda"`（而不是默认的 `"cpu"`）。将权保留在 GPU 上可以消除 CPU→GPU 传输开销，并能显著增加每秒的学习者更新次数。
