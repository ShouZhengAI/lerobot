# Reachy 2

Reachy 2 是由 Pollen Robotics 制造的开源仿人机器人，专为具身人工智能和实际应用的开发而设计。
请访问 [Pollen Robotics 官网](https://www.pollen-robotics.com/reachy/) 或查阅 [Reachy 2 文档](https://docs.pollen-robotics.com/) 了解更多平台信息！

## 遥操作 Reachy 2

目前有两种方式可以遥操作 Reachy 2：

- Pollen Robotics 的 VR 遥操作（不包含在 LeRobot 中）。
- 机器人到机器人遥操作（使用一台 Reachy 2 控制另一台）。

## Reachy 2 仿真

**（仅限 Linux）** 您可以使用提供的 [Docker 镜像](https://hub.docker.com/r/pollenrobotics/reachy2_core)在仿真环境（Gazebo 或 MuJoCo）中运行 Reachy 2。

1. 安装 [Docker Engine](https://docs.docker.com/engine/)。
2. 运行（针对 MuJoCo）：

```
docker run --rm -it \
  --name reachy \
  --privileged \
  --network host \
  --ipc host \
  --device-cgroup-rule='c 189:* rwm' \
  --group-add audio \
  -e ROS_DOMAIN_ID="$ROS_DOMAIN_ID" \
  -e DISPLAY="$DISPLAY" \
  -e RCUTILS_CONSOLE_OUTPUT_FORMAT="[{severity}]: {message}" \
  -e REACHY2_CORE_SERVICE_FAKE="${REACHY2_CORE_SERVICE_FAKE:-true}" \
  -v /dev:/dev \
  -v "$HOME/.reachy_config":/home/reachy/.reachy_config_override \
  -v "$HOME/.reachy.log":/home/reachy/.ros/log \
  -v /usr/lib/x86_64-linux-gnu:/opt/host-libs \
  --entrypoint /package/launch.sh \
  pollenrobotics/reachy2_core:1.7.5.9_deploy \
  start_rviz:=true start_sdk_server:=true mujoco:=true
```

> [!NOTE]
> 如果 MuJoCo 运行缓慢（仿真频率低），请在之前命令中添加 `-e LD_LIBRARY_PATH="/opt/host-libs:$LD_LIBRARY_PATH" \` 以提高性能：
>

## 设置

### 前提条件

- 在您的机器人上，确保**服务镜像**满足最低版本要求：
  - **reachy2-core >= 1.7.5.2**
  - **webrtc >= 2.0.1.1**

然后，如果您想使用 VR 遥操作：

- 安装 [Reachy 2 遥操作应用](https://docs.pollen-robotics.com/teleoperation/teleoperation-introduction/discover-teleoperation/)。
  使用版本 **>=v1.2.0**

我们建议使用两台电脑：一台用于遥操作（需要 Windows），另一台用于使用 LeRobot 录制。

### 安装 LeRobot

按照[安装说明](https://github.com/huggingface/lerobot#installation)安装 LeRobot。

安装带有 Reachy 2 依赖的 LeRobot：

```bash
pip install -e ".[reachy2]"
```

### （可选但推荐）安装 pollen_data_acquisition_server

管理 Reachy 2 录制会话的方式由您决定，但**最简单**的方法是使用此服务器，这样您可以直接从 VR 遥操作应用控制会话。

> **注意：** 目前，只有 VR 遥操作应用可以作为此服务器的客户端，因此此步骤主要针对遥操作场景。您可以根据需要自由开发自定义客户端来管理会话。

在您的 LeRobot 环境中，从源代码安装服务器：

```bash
git clone https://github.com/pollen-robotics/pollen_data_acquisition_server.git
cd pollen_data_acquisition_server
pip install -e .
```

在此查看 [pollen_data_acquisition_server 文档](https://github.com/pollen-robotics/pollen_data_acquisition_server)。

## 第 1 步：录制

### 获取 Reachy 2 的 IP 地址

在开始遥操作和数据录制之前，请找到[机器人的 IP 地址](https://docs.pollen-robotics.com/getting-started/setup-reachy2/connect-reachy2/)。
我们强烈建议通过**以太网**连接所有设备（电脑和机器人）。

### 启动录制

使用 Reachy 2 VR 遥操作应用时，有两种管理录制会话的方式：

- **使用数据采集服务器（VR 遥操作推荐）**：VR 应用编排会话（通过服务器告诉 LeRobot 何时创建数据集、开始/结束片段），同时控制机器人的运动。
- **使用 LeRobot 的录制脚本**：LeRobot 控制会话，决定何时开始/结束片段。如果您同时使用 VR 遥操作应用，它仅用于运动控制。

### 选项 1：使用 Pollen 数据采集服务器（VR 遥操作推荐）

确保已按照设置部分中的说明安装了 `pollen_data_acquisition_server`。

启动数据采集服务器，以便能够直接从遥操作应用管理会话：

```bash
python -m pollen_data_acquisition_server.server
```

然后进入遥操作应用并选择"数据采集会话"。
最后按照显示的屏幕设置您的会话。

> 即使没有 VR 应用，您也可以结合自己的客户端实现来使用 `pollen_data_acquisition_server`。

### 选项 2：使用 lerobot.record

Reachy 2 完全支持 LeRobot 的录制功能。
如果您选择此选项但仍想使用 VR 遥操作应用，请在应用中选择"标准会话"。

**示例：在没有移动底盘的情况下开始录制：**
首先将 `reachy2` 和 `reachy2_teleoperator` 添加到录制脚本的导入中。然后可以使用以下命令：

```bash
lerobot-record \
    --robot.type=reachy2 \
    --robot.ip_address=192.168.0.200 \
    --robot.id=r2-0000 \
    --robot.use_external_commands=true \
    --robot.with_mobile_base=false \
    --teleop.type=reachy2_teleoperator \
    --teleop.ip_address=192.168.0.200 \
    --teleop.with_mobile_base=false \
    --robot.with_torso_camera=true \
    --dataset.repo_id=pollen_robotics/record_test \
    --dataset.single_task="Reachy 2 recording test" \
    --dataset.num_episodes=1 \
    --dataset.episode_time_s=5 \
    --dataset.fps=15 \
    --dataset.push_to_hub=true \
    --dataset.private=true \
    --dataset.streaming_encoding=true \
    --dataset.encoder_threads=2 \
    # --dataset.rgb_encoder.vcodec=auto \
    --display_data=true
```

#### 特定参数

**完整设置概览（包含所有选项）：**

```bash
lerobot-record \
    --robot.type=reachy2 \
    --robot.ip_address=192.168.0.200 \
    --robot.use_external_commands=true \
    --robot.with_mobile_base=true \
    --robot.with_l_arm=true \
    --robot.with_r_arm=true \
    --robot.with_neck=true \
    --robot.with_antennas=true \
    --robot.with_left_teleop_camera=true \
    --robot.with_right_teleop_camera=true \
    --robot.with_torso_camera=false \
    --robot.camera_width=640 \
    --robot.camera_height=480 \
    --robot.disable_torque_on_disconnect=false \
    --robot.max_relative_target=5.0 \
    --teleop.type=reachy2_teleoperator \
    --teleop.ip_address=192.168.0.200 \
    --teleop.use_present_position=false \
    --teleop.with_mobile_base=false \
    --teleop.with_l_arm=true \
    --teleop.with_r_arm=true \
    --teleop.with_neck=true \
    --teleop.with_antennas=true \
    --dataset.repo_id=pollen_robotics/record_test \
    --dataset.single_task="Reachy 2 recording test" \
    --dataset.num_episodes=1 \
    --dataset.episode_time_s=5 \
    --dataset.fps=15 \
    --dataset.push_to_hub=true \
    --dataset.private=true \
    --dataset.streaming_encoding=true \
    --dataset.encoder_threads=2 \
    # --dataset.rgb_encoder.vcodec=auto \
    --display_data=true
```

##### `--robot.use_external_commands`

决定 LeRobot 的 `robot.send_action()` 是否向机器人发送指令。
**必须**在使用 VR 遥操作应用时设置为 `false`，因为该应用已负责发送指令。

##### `--teleop.use_present_position`

决定遥操作设备读取机器人的目标位置还是当前位置。
如果使用一台柔顺模式的 Reachy 2 来控制另一台机器人，则必须设置为 `true`。

##### 启用相关的部件

根据我们的初步测试，当只有部分关节运动时录制**所有**关节，可能会降低某些策略的模型质量。
为避免此问题，您可以使用以下命令排除特定部件的录制和回放：

```bash
--robot.with_<part>=false
```

其中 `<part>` 是以下之一：`mobile_base`、`l_arm`、`r_arm`、`neck`、`antennas`。
该参数决定相应部件是否记录在观测中。如果未设置，默认为 `true`。

默认情况下，**所有部件都会被录制**。

`reachy2_teleoperator` 也提供了相同的逐部件机制：

```bash
--teleop.with\_<part>
```

其中 `<part>` 是以下之一：`mobile_base`、`l_arm`、`r_arm`、`neck`、`antennas`。
该参数决定相应部件是否记录在动作中。如果未设置，默认为 `true`。

> **重要：** 在一次会话中，**机器人端和遥操作端启用的部件必须一致**。
> 例如，如果机器人使用 `--robot.with_mobile_base=false` 运行，遥操作端也必须禁用相同部件 `--teleoperator.with_mobile_base=false`。

##### 启用相关的摄像头

您也可以对**摄像头**进行同样操作。使用以下命令启用或禁用每个摄像头并使用默认参数：

```bash
--robot.with_left_teleop_camera=<true|false> \
--robot.with_right_teleop_camera=<true|false> \
--robot.with_torso_camera=<true|false>
```

默认情况下，不录制任何摄像头，所有摄像头参数都设置为 `false`。
如果需要，您可以使用 `--robot.camera_width` 和 `--robot.camera_height` 参数为 Reachy 2 的摄像头指定自定义的 `width` 和 `height` 参数：

```bash
--robot.camera_width=1920 \
--robot.camera_height=1080
```

这将改变所有三个默认机器人摄像头（由上述布尔参数启用的）的分辨率。

如果需要，您还可以像往常一样添加机器人自带摄像头之外的额外摄像头：

```bash
--robot.cameras="{ extra: {type: opencv, index_or_path: 42, width: 640, height: 480, fps: 30}}" \
```

## 第 2 步：回放

确保机器人的配置与数据集中的部件一致：

```bash
lerobot-replay \
    --robot.type=reachy2 \
    --robot.ip_address=192.168.0.200 \
    --robot.use_external_commands=false \
    --robot.with_mobile_base=false \
    --dataset.repo_id=pollen_robotics/record_test \
    --dataset.episode=0
```

## 第 3 步：训练

```bash
lerobot-train \
  --dataset.repo_id=pollen_robotics/record_test \
  --policy.type=act \
  --output_dir=outputs/train/reachy2_test \
  --job_name=reachy2 \
  --policy.device=mps \
  --wandb.enable=true \
  --policy.repo_id=pollen_robotics/record_test_policy
```

## 第 4 步：评估

```bash
lerobot-eval \
  --robot.type=reachy2 \
  --robot.ip_address=192.168.0.200 \
  --dataset.repo_id=pollen_robotics/eval_record_test \
  --dataset.single_task="Evaluate reachy2 policy" \
  --dataset.num_episodes=10 \
  --policy.path=outputs/train/reachy2_test/checkpoints/last/pretrained_model
```
