# 策略部署（Policy Deployment，lerobot-rollout）

`lerobot-rollout` 是在真实机器人上部署训练后策略的单一 CLI 工具。它支持多种执行策略和推理后端，从快速评估到持续记录以及人在回路中的数据收集。

## 快速开始

除了你的机器人和策略依赖外，不需要额外的依赖。

```bash
lerobot-rollout \
    --strategy.type=base \
    --policy.path=lerobot/act_koch_real \
    --robot.type=koch_follower \
    --robot.port=/dev/ttyACM0 \
    --task="拾取立方体" \
    --duration=30
```

该命令将在不记录数据的情况下运行策略 30 秒。

## 策略模式（Strategies）

使用 `--strategy.type=<name>` 选择策略模式。每种模式定义了不同的控制循环，具有各自的记录和交互语义。

### 基础模式（`--strategy.type=base`）

自主策略执行，不记录数据。适用于快速评估、演示，或仅需观察机器人时使用。

```bash
lerobot-rollout \
    --strategy.type=base \
    --policy.path=${HF_USER}/my_policy \
    --robot.type=so100_follower \
    --robot.port=/dev/ttyACM0 \
    --robot.cameras="{ front: {type: opencv, index_or_path: 0, width: 640, height: 480, fps: 30}}" \
    --task="将乐高积木放入盒子" \
    --duration=60
```

| 标志               | 描述                               |
| ------------------ | ---------------------------------- |
| `--duration`       | 运行时间（秒），0 表示无限期运行    |
| `--task`           | 传递给策略的任务描述                 |
| `--display_data`   | 将观测/动作流式传输到 Rerun 进行可视化 |

### 哨兵模式（`--strategy.type=sentry`）

连续自主记录，并定期上传到 Hugging Face Hub。片段边界会根据摄像头分辨率和 FPS 自动计算，使每个保存的片段生成完整的视频文件，保持上传效率。

策略状态（隐藏状态、RTC 队列）在片段边界之间保持不变：机器人在片段之间不会重置。

```bash
lerobot-rollout \
    --strategy.type=sentry \
    --strategy.upload_every_n_episodes=5 \
    --policy.path=${HF_USER}/my_policy \
    --robot.type=so100_follower \
    --robot.port=/dev/ttyACM0 \
    --robot.cameras="{ front: {type: opencv, index_or_path: 0, width: 640, height: 480, fps: 30}}" \
    --dataset.repo_id=${HF_USER}/rollout_eval_data \
    --dataset.single_task="将乐高积木放入盒子" \
    --duration=3600
```

| 标志                                     | 描述                                       |
| ---------------------------------------- | ----------------------------------------- |
| `--strategy.upload_every_n_episodes`     | 每 N 个片段推送到 Hub（默认值：5）         |
| `--strategy.target_video_file_size_mb`   | 片段轮换的目标视频文件大小（默认值：自动） |
| `--dataset.repo_id`                      | **必需。**记录数据集的 Hub 仓库            |
| `--dataset.push_to_hub`                  | 是否在结束时推送到 Hub（默认值：true）     |

### 高亮模式（`--strategy.type=highlight`）

通过内存有界环形缓冲区（ring buffer）实现按需记录的自主部署。机器人持续运行，同时缓冲区捕获最近 N 秒的遥测数据。按下保存键以刷新缓冲区并开始实时记录；再次按下以保存片段。

```bash
lerobot-rollout \
    --strategy.type=highlight \
    --strategy.ring_buffer_seconds=30 \
    --strategy.save_key=s \
    --strategy.push_key=h \
    --policy.path=${HF_USER}/my_policy \
    --robot.type=koch_follower \
    --robot.port=/dev/ttyACM0 \
    --dataset.repo_id=${HF_USER}/rollout_highlight_data \
    --dataset.single_task="拾取红色立方体"
```

**键盘控制：**

| 键                   | 操作                             |
| -------------------- | -------------------------------- |
| `s`（可配置）        | 开始记录（刷新缓冲区）/ 停止并保存片段 |
| `h`（可配置）        | 将数据集推送到 Hub                 |
| `ESC`                | 停止会话                           |

| 标志                                     | 描述                                     |
| ---------------------------------------- | --------------------------------------- |
| `--strategy.ring_buffer_seconds`         | 缓冲遥测数据的持续时间（默认值：30）     |
| `--strategy.ring_buffer_max_memory_mb`   | 环形缓冲区的内存上限（默认值：2048）     |
| `--strategy.save_key`                    | 切换记录的按键（默认值：`s`）            |
| `--strategy.push_key`                    | 推送到 Hub 的按键（默认值：`h`）         |

### DAgger 模式（`--strategy.type=dagger`）

人在回路中的数据收集。通过遥操作器在自主策略执行和人工干预之间交替。干预帧被标记为 `intervention=True`。需要一个遥操作器（`--teleop.type`）。

详细教程请参阅[人在回路数据收集指南](./hil_data_collection)。

**仅纠正模式**（默认）：仅记录人工纠正窗口。每个纠正成为一个片段。

```bash
lerobot-rollout \
    --strategy.type=dagger \
    --strategy.num_episodes=20 \
    --policy.path=outputs/pretrain/checkpoints/last/pretrained_model \
    --robot.type=bi_openarm_follower \
    --teleop.type=bi_openarm_mini \
    --dataset.repo_id=${HF_USER}/rollout_hil_data \
    --dataset.single_task="折叠 T 恤"
```

**连续记录模式**（`--strategy.record_autonomous=true`）：同时记录自主帧和纠正帧，基于时间进行片段轮换（与哨兵模式相同）。

```bash
lerobot-rollout \
    --strategy.type=dagger \
    --strategy.record_autonomous=true \
    --strategy.num_episodes=50 \
    --policy.path=${HF_USER}/my_policy \
    --robot.type=so100_follower \
    --robot.port=/dev/ttyACM0 \
    --teleop.type=so101_leader \
    --teleop.port=/dev/ttyACM1 \
    --dataset.repo_id=${HF_USER}/rollout_dagger_data \
    --dataset.single_task="抓取积木"
```

**键盘控制**（默认输入设备）：

| 键       | 操作                            |
| -------- | ------------------------------- |
| `Space`  | 暂停 / 恢复策略执行              |
| `Tab`    | 开始 / 停止人工纠正              |
| `Enter`  | 将数据集推送到 Hub（仅纠正模式） |
| `ESC`    | 停止会话                         |

也支持通过 `--strategy.input_device=pedal` 使用脚踏板输入。使用 `--strategy.pedal.*` 标志配置踏板编码。

| 标志                                   | 描述                                              |
| -------------------------------------- | ------------------------------------------------- |
| `--strategy.num_episodes`              | 要记录的纠正片段数（默认值：10）                  |
| `--strategy.record_autonomous`         | 是否也记录自主帧（默认值：false）                 |
| `--strategy.upload_every_n_episodes`   | 每 N 个片段推送到 Hub（默认值：5）                |
| `--strategy.input_device`              | 输入设备：`keyboard` 或 `pedal`（默认值：keyboard）|
| `--teleop.type`                        | **必需。**遥操作器类型                            |

### 分段模式（`--strategy.type=episodic`）

面向片段的记录，模拟 `lerobot-record` 的行为。策略在每个片段中驱动机器人；可选的遥操作器可以在片段之间的重置阶段驱动机器人。

```bash
lerobot-rollout \
    --strategy.type=episodic \
    --policy.path=${HF_USER}/my_policy \
    --robot.type=so100_follower \
    --robot.port=/dev/ttyACM0 \
    --teleop.type=so100_leader \
    --teleop.port=/dev/ttyACM1 \
    --dataset.repo_id=${HF_USER}/my_eval_data \
    --dataset.num_episodes=20 \
    --dataset.episode_time_s=30 \
    --dataset.reset_time_s=10 \
    --dataset.single_task="拾取红色立方体"
```

遥操作器是可选的——如果省略，机器人在重置阶段保持其位置。

**键盘控制：**

| 键              | 操作                  |
| --------------- | --------------------- |
| `→`（右箭头）   | 提前结束当前片段      |
| `←`（左箭头）   | 丢弃片段并重新记录    |
| `ESC`           | 停止记录会话          |

| 标志                                               | 描述                                                  |
| -------------------------------------------------- | ---------------------------------------------------- |
| `--dataset.num_episodes`                           | 要记录的片段数                                        |
| `--dataset.episode_time_s`                         | 每个记录片段的持续时间（秒）                          |
| `--dataset.reset_time_s`                           | 片段之间重置阶段的持续时间（秒）                      |
| `--teleop.type`                                    | 可选。在重置期间驱动机器人的遥操作器                  |
| `--strategy.reset_to_initial_position`             | 是否在片段之间将机器人重置到初始位置                  |
| `--strategy.smooth_leader_to_follower_handover`    | 是否启用或禁用从主臂到从臂的平滑交接行为              |

## 推理后端（Inference Backends）

使用 `--inference.type=<name>` 选择推理后端。所有策略模式都可与两种后端配合使用。

### 同步模式（默认）

每个控制周期进行一次策略调用。主循环阻塞，直到动作计算完成。

适用于所有策略。无需额外标志。

### 实时分块（Real-Time Chunking，`--inference.type=rtc`）

后台线程异步产生动作块。主控制循环轮询下一个就绪的动作，同时策略并行计算下一个块。

将 RTC 与大型、慢速的 VLA 模型（Pi0、Pi0.5、SmolVLA）一起使用，尽管推理延迟高，仍能实现平滑、连续的运动。

```bash
lerobot-rollout \
    --strategy.type=base \
    --inference.type=rtc \
    --inference.rtc.execution_horizon=10 \
    --inference.rtc.max_guidance_weight=10.0 \
    --policy.path=${HF_USER}/pi0_policy \
    --robot.type=so100_follower \
    --robot.port=/dev/ttyACM0 \
    --robot.cameras="{ front: {type: opencv, index_or_path: 0, width: 640, height: 480, fps: 30}}" \
    --task="拾取立方体" \
    --duration=60 \
    --device=cuda
```

| 标志                                            | 描述                                              |
| ----------------------------------------------- | ------------------------------------------------- |
| `--inference.rtc.execution_horizon`             | 与上一个块混合的步数（默认值：因策略而异）        |
| `--inference.rtc.max_guidance_weight`           | 一致性强制强度（默认值：因策略而异）              |
| `--inference.rtc.prefix_attention_schedule`     | 混合调度：`LINEAR`、`EXP`、`ONES`、`ZEROS`        |
| `--inference.queue_threshold`                   | 回压前的最大队列大小（默认值：30）                |

有关调优 RTC 参数的详细信息，请参阅[实时分块指南](./rtc)。

## 通用标志

| 标志                                | 描述                                        | 默认值 |
| ----------------------------------- | ------------------------------------------- | ------ |
| `--policy.path`                     | **必需。**HF Hub 模型 ID 或本地检查点路径   | --     |
| `--robot.type`                      | **必需。**机器人类型（如 `so100_follower`、`koch_follower`） | --     |
| `--robot.port`                      | 机器人的串口                                | --     |
| `--robot.cameras`                   | 摄像头配置（JSON 字典）                     | --     |
| `--fps`                             | 控制循环频率                                | 30     |
| `--duration`                        | 运行时间（秒），0 表示无限期运行             | 0      |
| `--device`                          | Torch 设备（`cpu`、`cuda`、`mps`）          | auto   |
| `--task`                            | 任务描述（未提供数据集时使用）               | --     |
| `--display_data`                    | 将遥测数据流式传输到 Rerun 可视化            | false  |
| `--display_ip` / `--display_port`   | 远程 Rerun 服务器地址                        | --     |
| `--interpolation_multiplier`        | 动作插值因子                                | 1      |
| `--use_torch_compile`               | 启用 `torch.compile` 进行推理               | false  |
| `--resume`                          | 恢复之前的记录会话                          | false  |
| `--play_sounds`                     | 事件的声音合成                              | true   |

## 编程式使用

对于自定义部署（例如使用运动学处理器），可以直接使用 rollout 模块的 API：

```python
from lerobot.rollout import BaseStrategyConfig, RolloutConfig, build_rollout_context
from lerobot.rollout.inference import SyncInferenceConfig
from lerobot.rollout.strategies import BaseStrategy
from lerobot.utils.process import ProcessSignalHandler

cfg = RolloutConfig(
    robot=my_robot_config,
    policy=my_policy_config,
    strategy=BaseStrategyConfig(),
    inference=SyncInferenceConfig(),
    fps=30,
    duration=60,
    task="我的任务",
)

signal_handler = ProcessSignalHandler(use_threads=True)
ctx = build_rollout_context(
    cfg,
    signal_handler.shutdown_event,
    robot_action_processor=my_custom_action_processor,       # 可选
    robot_observation_processor=my_custom_obs_processor,     # 可选
)

strategy = BaseStrategy(cfg.strategy)
try:
    strategy.setup(ctx)
    strategy.run(ctx)
finally:
    strategy.teardown(ctx)
```

有关包含运动学处理器的完整示例，请参阅 `examples/so100_to_so100_EE/rollout.py` 和 `examples/phone_to_so100/rollout.py`。
