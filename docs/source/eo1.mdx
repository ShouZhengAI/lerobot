# EO-1

EO-1 是一种用于机器人控制的**视觉-语言-动作策略（Vision-Language-Action Policy）**。LeRobot 实现将 EO-1 集成了标准的 LeRobot 训练、评估和处理器接口。

## 模型概述

EO-1 使用 Qwen2.5-VL 骨干网络进行视觉-语言理解，并添加了一个连续流匹配动作头用于机器人控制。该策略将每个机器人控制样本格式化为多模态对话：相机图像传入 Qwen2.5-VL，机器人状态用 EO-1 状态令牌（state tokens）表示，未来的动作块用 EO-1 动作令牌（action tokens）表示。

  alt="EO-1 概览"

在训练过程中，EO-1 学习在动作令牌位置上对连续动作块进行去噪。在推理过程中，它采样一个动作块，返回连续动作，并执行该块中的 `n_action_steps` 步动作，然后再次采样。

### LeRobot 集成涵盖的内容

- 通过 LeRobot 标准化的 `policy.type=eo1` 配置
- 通过策略处理器（policy processors）进行 Qwen2.5-VL 图像和文本预处理
- 连续流匹配动作预测
- 通过 LeRobot 策略 API 进行检查点保存/加载
- 使用 `lerobot-train` 进行训练，使用 `lerobot-eval` 进行评估

更广泛的 EO-1 项目还包括交错的视觉-文本-动作预训练和多模态推理工作流。本页面聚焦于 LeRobot 机器人控制策略路径。

## 安装要求

1. 按照[安装指南](./installation)安装 LeRobot。
2. 安装 EO-1 依赖项，运行：

   ```bash
   pip install -e ".[eo1]"
   ```

3. 如果要在 LIBERO 上进行训练或评估，也请安装 LIBERO 依赖项：

   ```bash
   pip install -e ".[eo1,libero]"
   ```

EO-1 可以通过 `policy.attn_implementation=sdpa` 使用标准的 PyTorch 缩放点积注意力后端。如果你的环境安装了兼容的 `flash_attn`，可以使用 `policy.attn_implementation=flash_attention_2`。

## 数据要求

EO-1 期望的 LeRobot 数据集包含：

- 至少一个视觉观测，例如 `observation.images.image`
- `observation.state`
- `action`
- 通过数据集 `task` 字段提供的语言任务指令

如果你的数据集使用了不同的观测名称，请使用 `rename_map` 将它们与你训练或评估配置中期望的名称对齐。

## 使用方法

要在 LeRobot 配置中使用 EO-1，将策略类型指定为：

```python
policy.type=eo1
```

默认情况下，新的 EO-1 策略从以下位置初始化其骨干网络：

```python
policy.vlm_base=Qwen/Qwen2.5-VL-3B-Instruct
```

一旦有了 LeRobot 格式的 EO-1 检查点，通过以下方式加载：

```python
policy.path=your-org/your-eo1-checkpoint
```

## 训练

### 训练命令示例

```bash
lerobot-train \
  --dataset.repo_id=your_org/your_dataset \
  --policy.type=eo1 \
  --policy.vlm_base=Qwen/Qwen2.5-VL-3B-Instruct \
  --policy.dtype=bfloat16 \
  --policy.attn_implementation=sdpa \
  --policy.gradient_checkpointing=false \
  --output_dir=./outputs/eo1_training \
  --job_name=eo1_training \
  --steps=300000 \
  --batch_size=16 \
  --policy.device=cuda
```

### 关键训练参数

| 参数                                  | 默认值                         | 描述                                          |
| ------------------------------------- | -------------------------------- | ------------------------------------------------------------ |
| `policy.vlm_base`                      | `Qwen/Qwen2.5-VL-3B-Instruct`  | 用于初始化新策略的 Qwen2.5-VL 检查点                         |
| `policy.dtype`                         | `auto`                          | 骨干网络数据类型请求：`auto`、`bfloat16` 或 `float32`          |
| `policy.attn_implementation`           | `None`                          | 可选的 Qwen 注意力后端，例如 `sdpa`                            |
| `policy.gradient_checkpointing`        | `false`                         | 在训练期间减少内存使用                        |
| `policy.chunk_size`                    | `8`                             | 每个块预测的未来动作数量                      |
| `policy.n_action_steps`                | `8`                             | 从采样块中消费的动作数量                     |
| `policy.num_denoise_steps`             | `10`                            | 采样过程中使用的流匹配去噪步数            |
| `policy.max_state_dim`                 | `32`                            | 状态填充维度                                  |
| `policy.max_action_dim`                | `32`                            | 动作填充维度                                 |
| `policy.force_fp32_autocast`           | `true`                          | 即使骨干网络使用混合精度，也将流头保持在 fp32              |
| `policy.supervise_padding_action_dims` | `true`                          | 控制是否对填充的动作维度进行监督               |
| `policy.supervise_padding_actions`     | `true`                          | 控制是否对填充的未来动作行进行监督              |

## 评估

一旦有了 LeRobot 格式的检查点，可以通过 `lerobot-eval` 评估 EO-1：

```bash
lerobot-eval \
  --policy.path=your-org/your-eo1-checkpoint \
  --env.type=libero \
  --env.task=libero_object \
  --eval.batch_size=1 \
  --eval.n_episodes=20
```

如果数据集或环境的相机名称与检查点配置不同，传入 `rename_map`：

```bash
lerobot-eval \
  --policy.path=your-org/your-eo1-checkpoint \
  --env.type=libero \
  --env.task=libero_object \
  --rename_map='{"observation.images.image2":"observation.images.wrist_image"}'
```

## 配置说明

### 图像处理

EO-1 使用 Qwen2.5-VL 处理器（processor）。`policy.image_min_pixels` 和 `policy.image_max_pixels` 设置控制图像在视觉令牌传入骨干网络之前的缩放范围。

### 状态和动作维度

该策略在 EO-1 流头之前将状态和动作向量填充到 `policy.max_state_dim` 和 `policy.max_action_dim`。预测结果在由策略返回之前裁剪回原始动作维度。

### 注意力后端

使用 `policy.attn_implementation=sdpa` 实现可移植配置。仅在安装了 `flash_attn` 且与你的环境兼容时使用 `flash_attention_2`。
