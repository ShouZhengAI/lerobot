# VLA-JEPA

这是 LeRobot 移植的 **VLA-JEPA**，一个将 Qwen3-VL 语言骨干网络与自监督视频世界模型（V-JEPA2）以及流匹配 DiT 动作头相结合的视觉-语言-动作模型。

---

## 架构概述

VLA-JEPA 包含三个主要组件：

| 组件                        | 模块                              | 角色                                        |
| --------------------------- | --------------------------------- | ------------------------------------------- |
| **Qwen3-VL 骨干网络**       | `Qwen3VLInterface`                | 将图像和语言指令融合为上下文令牌（context tokens） |
| **DiT-B 动作头**            | `VLAJEPAActionHead`               | 在动作片段上执行流匹配扩散（flow-matching diffusion） |
| **V-JEPA2 世界模型**        | `ActionConditionedVideoPredictor` | 自监督视频预测损失（仅训练阶段）               |

### 数据流

**训练阶段：**

1. V-JEPA2 将包含 `num_video_frames` 帧的视频片段编码为逐帧分块令牌（patch tokens）。
2. Qwen3-VL 骨干网络处理多视角图像和任务指令，生成一个包含特殊动作令牌（用于世界模型条件化）和具身令牌的上下文令牌序列。
3. 动作头将这些上下文令牌作为交叉注意力（cross-attention）的键/值输入，通过流匹配预测去噪后的动作片段。
4. 世界模型预测器使用从 Qwen 提取的动作令牌，预测未来的 V-JEPA2 帧嵌入；在这些预测上计算回归损失，并添加到动作损失中。

**推理阶段：**
仅使用 Qwen 和动作头。推理时不需要世界模型。

### 动作头详情

通过 `action_model_type` 可用的预设：

| 预设   | 隐藏维度 | 注意力头数 | 每个头的维度 |
| ------ | -------- | ---------- | ------------ |
| `DiT-B` | 768      | 12         | 64           |
| `DiT-L` | 1536     | 32         | 48           |

### 世界模型详情

视频预测器是一个 ViT 风格的 Transformer（`ActionConditionedVideoPredictor`），接收以下输入：

- **帧令牌**：V-JEPA2 分块嵌入，投影到 `predictor_embed_dim`
- **动作令牌**：Qwen 动作令牌嵌入，投影到 `predictor_embed_dim`

它使用块级因果注意力（block-causal attention），因此每个时间步可以关注所有之前的时间步。预测器的输入 `embed_dim` 等于 `num_views × video_encoder_hidden_size`（例如，预训练检查点为 2 个视角 × 1024 = 2048）。

## 预训练检查点

LeRobot 组织内直接提供了三个检查点：[`lerobot/VLA-JEPA`](https://huggingface.co/collections/lerobot/vla-jepa)，从 [ginwind/VLA-JEPA](https://huggingface.co/ginwind/VLA-JEPA) 转换而来：

| 检查点                          | 数据集              | 相机数量              | 世界模型 | 动作维度 |
| ------------------------------- | ------------------- | ---------------------- | ----------- | --------- |
| `lerobot/VLA-JEPA-LIBERO`       | LIBERO-10           | 2（机械臂视角 + 腕部） | 启用        | 7         |
| `lerobot/VLA-JEPA-Pretrain`     | DROID 1.0.1         | 2（外部左视角）        | 启用        | 7         |
| `lerobot/VLA-JEPA-SimplerEnv`   | OXE Bridge / RT-1   | 1（复制为 2 个视角）  | 启用        | 7         |

所有检查点均使用 `Qwen/Qwen3-VL-2B-Instruct` 作为语言骨干网络。

## 配置

`VLAJEPAConfig` 中的关键参数：

| 参数                      | 默认值   | 描述                                                                                                                     |
| ------------------------- | -------- | ------------------------------------------------------------------------------------------------------------------------ |
| `chunk_size`              | 7        | 每次推理调用预测的动作数量                                                                                               |
| `n_action_steps`          | 7        | 在重新规划之前，从预测片段中执行的动作步数                                                                               |
| `num_video_frames`        | 8        | 输入给世界模型的视频片段长度                                                                                             |
| `enable_world_model`      | `True`   | 是否加载和训练 V-JEPA2 预测器                                                                                            |
| `world_model_loss_weight` | 0.1      | JEPA 预测损失相对于动作损失的权重                                                                                        |
| `num_inference_timesteps` | 4        | 动作去噪的欧拉积分步数                                                                                                   |
| `freeze_qwen`             | `False`  | 冻结 Qwen3-VL 骨干网络，仅训练动作头                                                                                    |
| `reinit_modules`          | `None`   | 允许在加载时随机重新初始化的键前缀（用于跨具身迁移，请参见[在不同具身上微调](#在不同具身上微调)）                          |
| `gripper_dim`             | 6        | 动作向量中夹爪维度的索引（例如，对于 7 自由度机械臂且夹爪作为最后一个关节，则为 6）                                       |
| `gripper_threshold`       | 0.5      | `pre_snap_gripper_action` 和 `binarize_gripper_action` 用于二值化夹爪维度的阈值                                           |
| `pre_snap_gripper_action` | `True`   | 在反归一化之前将夹爪维度固定到 {0, 1}。对于没有二值夹爪的机器人，设置为 `False`                                           |
| `binarize_gripper_action` | `True`   | 在反归一化之后将夹爪维度二值化为 {-1, 1}。对于没有二值夹爪的机器人，设置为 `False`                                        |

## 训练

训练步数可能因数据集大小和计算预算而异。原始论文在 ssv2 + droid 联合数据集上预训练了 5 万步，然后在 LIBERO 上额外训练了 3 万步，但在提供的预训练检查点上进行微调时，较少的步数仍可能获得良好的性能。

### 从头开始完整训练

```bash
lerobot-train \
  policy.type=vla_jepa \
  policy.repo_id=your_org/your_repo \
  dataset.repo_id=your_org/your_dataset
```

### 从预训练检查点微调

```bash
lerobot-train \
  --policy.path=lerobot/VLA-JEPA-Pretrain \
  --policy.repo_id=your_org/your_repo \
  --dataset.repo_id=your_org/your_dataset
```

如果你想冻结 Qwen 骨干网络，仅训练动作头，设置 `policy.freeze_qwen=True`：

```bash
lerobot-train \
  --policy.path=lerobot/VLA-JEPA-Pretrain \
  --policy.repo_id=your_org/your_repo \
  --policy.freeze_qwen=true \
  --dataset.repo_id=your_org/your_dataset
```

### 在不同具身上微调

当目标机器人的动作或状态维度与预训练检查点不同时，动作头的输入/输出投影层的形状将不匹配，无法直接加载。`reinit_modules` 允许你列出允许不匹配的键前缀——这些层将被随机重新初始化，而其他所有权重则从检查点复用。列出的前缀之外的任何形状不匹配都会引发错误。

依赖于 `action_dim` 和 `state_dim` 的层如下：

| 层                                         | 键前缀                                    |
| ----------------------------------------- | --------------------------------------- |
| 动作编码器（action_dim → inner_dim）       | `model.action_model.action_encoder`     |
| 动作解码器（hidden_size → action_dim）     | `model.action_model.action_decoder`     |
| 状态编码器（state_dim → inner_dim）       | `model.action_model.state_encoder`      |

```bash
lerobot-train \
  --policy.path=lerobot/VLA-JEPA-Pretrain \
  --policy.repo_id=your_org/your_repo \
  --policy.freeze_qwen=true \
  --policy.reinit_modules='["model.action_model.action_encoder", "model.action_model.action_decoder", "model.action_model.state_encoder"]' \
  --dataset.repo_id=your_org/your_dataset
```

如果你的机器人没有本体感知状态，请从列表中省略 `model.action_model.state_encoder`。

### 复现 LIBERO 结果

**在 LIBERO 上训练：**
从 Pretrain 检查点开始训练，在 LIBERO 数据集上训练 3 万步。原始论文提到在 8 个 GPU 上训练，批次大小为 32，即全局批次大小为 256。

```bash
lerobot-train \
  --policy.path=lerobot/VLA-JEPA-Pretrain \
  --policy.repo_id=your_org/your_repo \
  --dataset.repo_id=HuggingFaceVLA/libero \
  --steps=30000
```

**评估预训练的 LIBERO-10 检查点：**

```bash
lerobot-eval \
  --policy.path=lerobot/VLA-JEPA-LIBERO \
  --env.type=libero \
  --env.task=libero_spatial,libero_object,libero_goal,libero_10 \
  --eval.n_episodes=10 \
  --eval.batch_size=5
```

仅评估部分任务：

```bash
lerobot-eval \
  --policy.path=lerobot/VLA-JEPA-LIBERO \
  --env.type=libero \
  --env.task=libero_10 \
  --env.task_ids='[0,1,2]' \
  --eval.n_episodes=10 \
  --eval.batch_size=5
```

**预期结果：**

| 测试集           | 片段数 | 成功数 | 成功率      |
| ---------------- | ------ | ------ | ----------- |
| libero_spatial   | 100    | 93     | **95.0%**   |
| libero_object    | 100    | 100    | **100.0%**  |
| libero_goal      | 100    | 98     | **98.0%**   |
| libero_10        | 100    | 96     | **93.0%**   |
| **总计**         | **400**| **387**| **96.5%**   |

## 在具有不同相机数量的数据集上微调

预训练的世界模型预测器使用 `embed_dim = jepa_tubelet_size × 1024`（默认 `jepa_tubelet_size=2`）进行训练。

**默认行为——视角填充/裁剪（无需操作）**

当从 `VLA-JEPA-Pretrain` 微调时，模型会自动调整输入到世界模型的视角数量，以匹配 `jepa_tubelet_size`：

- **单视角数据集（例如 BridgeV2）：** 单视角潜空间表示被复制，生成双视角的世界模型输入，从而保留 JEPA 自监督信号，无需任何权重调整。
- **多于 2 个视角的数据集（例如 DROID 有 3 个视角）：** 所有视角都传递给 Qwen 骨干网络（以获得更丰富的上下文），但只有前 `jepa_tubelet_size` 个视角（一个腕部视角加一个第三人称视角，按配置的视角顺序）用于世界模型。

**选项 1 —— 禁用世界模型**

设置 `enable_world_model=False` 以完全跳过 JEPA 损失。仅加载和训练 Qwen 骨干网络和动作头。这对良好的动作性能来说已经足够。

```bash
lerobot-train \
  --policy.path=lerobot/VLA-JEPA-Pretrain \
  --policy.enable_world_model=false \
  --policy.repo_id=your_org/your_repo \
  --dataset.repo_id=your_org/single_camera_dataset
```

**选项 2 —— 重新初始化预测器输入投影**

如果你希望将 `jepa_tubelet_size` 改为 2 以外的值，请使用 `strict=False` 加载检查点，并为新的 `embed_dim` 重新初始化 `model.video_predictor.predictor_embed`。所有其他预测器块权重（注意力、MLP、归一化、输出投影）与相机数量无关，可以从预训练检查点复用。
