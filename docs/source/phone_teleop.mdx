# 手机遥操作

使用你的手机（iOS 或 Android）来控制机器人。

**本指南将教你：**

- 如何连接 iOS/Android 手机
- 手机姿态（pose）如何映射到机器人末端执行器（EE）目标
- 如何调整安全限制、夹爪控制和逆运动学（IK）设置

要使用手机控制机器人，请安装相关依赖：

```bash
pip install lerobot[phone]
```

## 快速入门

### 支持的平台

- **iOS**：使用 HEBI Mobile I/O 应用（ARKit 姿态 + 按钮）。先下载该应用，打开后示例程序会自动发现手机并流式传输手机姿态和输入。
- **Android**：使用 `teleop` 包（WebXR）。启动 Python 进程时，会打印一个本地 URL。在手机上打开该链接，点击 Start，然后使用 Move 来流式传输姿态。

链接：

- Android WebXR 库：[PyPI 上的 `teleop`](https://pypi.org/project/teleop/)
- iOS 应用：[HEBI Mobile I/O](https://docs.hebi.us/tools.html#mobile-io)

### 手机方向和控制

- **朝向**：握住手机时屏幕朝上，手机顶部边缘指向与机器人夹爪相同的方向。这样可以确保校准时手机坐标系与机器人坐标系对齐，使操作感觉自然，参见下图。
- **启用/停用**：
  - iOS：按住 `B1` 启用遥操作，松开停止。首次按下时会捕获参考姿态。
  - Android：按住 `Move` 按钮，松开停止。首次按下时会捕获参考姿态。
- **夹爪控制**：
  - iOS：模拟输入 `A3` 作为速度输入控制夹爪。
  - Android：按钮 `A` 和 `B` 起到增加/减少的作用（A 打开，B 关闭）。可以在 `GripperVelocityToJoint` 步骤中调整速度。

### 第一步：选择平台

在 `PhoneConfig` 中将示例修改为使用 `PhoneOS.IOS` 或 `PhoneOS.ANDROID`。不同平台的 API 完全相同，仅输入源不同。所有示例都在 `examples/` 目录下，并包含 `phone_so100_*.py` 变体。

遥操作示例：

```python
from lerobot.teleoperators.phone import Phone, PhoneConfig
from lerobot.teleoperators.phone.config_phone import PhoneOS

teleop_config = PhoneConfig(phone_os=PhoneOS.IOS)  # 或 PhoneOS.ANDROID
teleop_device = Phone(teleop_config)
```

### 第二步：连接和校准

创建 `Phone(teleop_config)` 并调用 `connect()` 时，系统会自动提示校准。按照上述方向握住手机，然后：

- iOS：按住 `B1` 捕获参考姿态。
- Android：在 WebXR 页面上按下 `Move` 按钮捕获参考姿态。

为什么要校准？我们捕获当前姿态，以便后续姿态在机器人对齐的坐标系中表示。当你再次按下按钮启用控制时，位置会重新捕获，以避免停用期间重新放置手机时产生漂移。

### 第三步：运行示例

运行示例脚本来执行遥操作、录制数据集、回放数据集或评估策略。

所有示例都假设你已经配置好机器人（例如 SO-100 从动臂）并设置了正确的串行端口。

此外，你需要**将机器人的 URDF 文件复制到示例文件夹中**。对于本教程中的示例（使用 SO100/SO101），请将 [SO-ARM100 仓库](https://github.com/TheRobotStudio/SO-ARM100/blob/main/Simulation/SO101) 中的 `SO101` 文件夹复制到 `examples/phone_to_so100/` 目录下，使 URDF 文件路径变为 `examples/phone_to_so100/SO101/so101_new_calib.urdf`。

- 运行以下示例进行遥操作：

  ```bash
  cd examples/phone_to_so100
  python teleoperate.py
  ```

运行示例后：

- Android：启动脚本后，在手机上打开打印的本地 URL，点击 Start，然后按住 Move。
- iOS：先打开 HEBI Mobile I/O；B1 启用运动。A3 控制夹爪。

此外，你可以通过编辑示例中显示的 processor 步骤来自定义映射或安全限制。你还可以重新映射输入（例如使用不同的模拟输入）或通过修改输入和运动学步骤来适配其他机器人（例如 LeKiwi）。更多信息请参阅[机器人和遥操作的处理器指南](./processors_robots_teleop)。

- 运行以下示例录制数据集，该数据集会保存绝对的末端执行器观测值和动作：

  ```bash
  cd examples/phone_to_so100
  python record.py
  ```

- 运行以下示例回放录制的片段：

  ```bash
  cd examples/phone_to_so100
  python replay.py
  ```

- 运行以下示例评估预训练策略：

  ```bash
  cd examples/phone_to_so100
  python evaluate.py
  ```

### 重要的 pipeline 步骤和选项

- 运动学计算在多个步骤中使用。我们使用 [Placo](https://github.com/Rhoban/placo)，它是 Pinocchio 的封装库，用于处理运动学。我们通过传入机器人的 URDF 和目标坐标系（frame）来构建运动学对象。我们将 `target_frame_name` 设置为夹爪坐标系。

  ```python
  kinematics_solver = RobotKinematics(
    urdf_path="./SO101/so101_new_calib.urdf",
    target_frame_name="gripper_frame_link",
    joint_names=list(robot.bus.motors.keys()),
  )
  ```

- `MapPhoneActionToRobotAction` 步骤将校准后的手机姿态和输入转换为目标增量（delta）和夹爪指令，下面展示了该步骤的输出。

  ```python
  action["enabled"] = enabled
        action["target_x"] = -pos[1] if enabled else 0.0
        action["target_y"] = pos[0] if enabled else 0.0
        action["target_z"] = pos[2] if enabled else 0.0
        action["target_wx"] = rotvec[1] if enabled else 0.0
        action["target_wy"] = rotvec[0] if enabled else 0.0
        action["target_wz"] = -rotvec[2] if enabled else 0.0
        action["gripper_vel"] = gripper_vel  # 禁用时仍发送夹爪动作
  ```

- `EEReferenceAndDelta` 步骤将目标增量转换为绝对的目标末端执行器位姿，在启用时存储一个参考值。`end_effector_step_sizes` 是末端执行器位姿的步长，可以修改以改变运动速度。

  ```python
  EEReferenceAndDelta(
      kinematics=kinematics_solver,
      end_effector_step_sizes={"x": 0.5, "y": 0.5, "z": 0.5},
      motor_names=list(robot.bus.motors.keys()),
      use_latched_reference=True,
  ),
  ```

- `EEBoundsAndSafety` 步骤将末端执行器运动限制在工作空间内，并检查大幅的末端执行器步进跳跃以确保安全。`end_effector_bounds` 是末端执行器位姿的边界，可以修改以改变工作空间。`max_ee_step_m` 是末端执行器位姿的步进限制，可以修改以调整安全限制。

  ```python
  EEBoundsAndSafety(
      end_effector_bounds={"min": [-1.0, -1.0, -1.0], "max": [1.0, 1.0, 1.0]},
      max_ee_step_m=0.10,
  )
  ```

- `GripperVelocityToJoint` 步骤将类似速度的夹爪输入转换为绝对的夹爪位置，使用当前测量的状态。`speed_factor` 是速度的乘法因子。

  ```python
  GripperVelocityToJoint(speed_factor=20.0)
  ```

#### 不同的逆运动学初始猜测

我们在运动学步骤中使用不同的逆运动学初始猜测。初始猜测可以使用当前测量的关节值，也可以使用上一次的逆运动学求解结果。

- 闭环（用于录制/评估）：设置 `initial_guess_current_joints=True`，使逆运动学每帧从测量的关节值开始。

  ```python
  InverseKinematicsEEToJoints(
      kinematics=kinematics_solver,
      motor_names=list(robot.bus.motors.keys()),
      initial_guess_current_joints=True,  # 闭环
  )
  ```

- 开环（用于回放）：设置 `initial_guess_current_joints=False`，使逆运动学从上一次求解结果继续，而非使用测量的状态。这可以在无反馈回放时保持动作的稳定性。

  ```python
  InverseKinematicsEEToJoints(
      kinematics=kinematics_solver,
      motor_names=list(robot.bus.motors.keys()),
      initial_guess_current_joints=False,  # 开环
  )
  ```

### Pipeline 步骤说明

- MapPhoneActionToRobotAction：将校准后的手机姿态和输入转换为目标增量（delta）和夹爪指令。运动受启用信号门控（iOS 上为 B1，Android 上为 Move）。
- EEReferenceAndDelta：在启用时锁存一个参考末端执行器位姿，并将其与目标增量结合，每帧生成一个绝对的目标末端执行器位姿。禁用时会持续发送上一次指令的位姿。
- EEBoundsAndSafety：将末端执行器位姿限制在工作空间内，并对跳跃进行速率限制以确保安全。同时声明 `action.ee.*` 特征。
- InverseKinematicsEEToJoints：通过逆运动学将末端执行器位姿转换为关节位置。闭环控制建议 `initial_guess_current_joints=True`；开环回放设置为 `False` 以保持稳定性。
- GripperVelocityToJoint：将类似速度的夹爪输入积分成绝对的夹爪位置，使用当前测量的状态。
- ForwardKinematicsJointsToEE：根据观测到的关节值计算 `observation.state.ee.*`，用于日志记录和在末端执行器状态上进行训练。

### 故障排除

- iOS 无法发现：确保 HEBI Mobile I/O 已打开，并且你的电脑和手机在同一个网络中。
- Android URL 无法访问：检查你是否使用了 `https` 而非 `http`，使用脚本打印的确切 IP 地址，并允许浏览器访问并忽略证书问题。
- 运动方向感觉相反：调整 `MapPhoneActionToRobotAction` 中的正负号翻转子，或交换坐标轴以适配你的设置。
