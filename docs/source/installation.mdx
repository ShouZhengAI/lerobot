# 安装（Installation）

本指南使用 `conda`（通过 miniforge）来管理环境（推荐）。如果你偏好其他环境管理器（例如 `uv`、`venv`），请确保已安装 Python >= 3.12 并支持 PyTorch >= 2.10，然后直接跳转到[环境设置](#step-2-environment-setup)。

## 第 1 步（仅 `conda`）：安装 [`miniforge`](https://conda-forge.org/download/)

```bash
wget "https://github.com/conda-forge/miniforge/releases/latest/download/Miniforge3-$(uname)-$(uname -m).sh"
bash Miniforge3-$(uname)-$(uname -m).sh
```

使用 Python 3.12 创建一个虚拟环境：

### 安装 `ffmpeg`（用于视频解码）

LeRobot 默认使用 [TorchCodec](https://github.com/meta-pytorch/torchcodec) 进行视频解码，这需要 `ffmpeg`。

> [!NOTE]
> **平台支持：** TorchCodec 在 macOS Intel（x86_64）、Linux ARM（aarch64、arm64、armv7l）或 PyTorch < 2.8 的 Windows 上**不可用**。在这些平台上，LeRobot 会自动回退到 `pyav`——因此你无需安装 `ffmpeg`，可以直接跳到第 3 步。

如果你的平台支持 TorchCodec，请使用以下方法之一安装 `ffmpeg`：

<!-- prettier-ignore-start -->

<hfoptions id="install_ffmpeg">
<hfoption id="conda（任何 PyTorch 版本）">

在 conda 环境中安装 `ffmpeg`。这适用于**所有 PyTorch 版本**，并且对于 **PyTorch < 2.10** 是**必须的**：

```bash
conda install ffmpeg -c conda-forge
```

> [!TIP]
> 这通常会安装带有 `libsvtav1` 编码器的 `ffmpeg 8.X`。如果遇到问题（例如缺少 `libsvtav1`——使用 `ffmpeg -encoders` 检查——或与 `torchcodec` 的版本不匹配），你可以显式安装 `ffmpeg 7.1.1`：
>
> ```bash
> conda install ffmpeg=7.1.1 -c conda-forge
> ```

</hfoption>
<hfoption id="uv（仅 PyTorch >= 2.10）">

从 **PyTorch >= 2.10**（TorchCodec ≥ 0.10）开始，TorchCodec 可以动态链接到系统级的 `ffmpeg` 安装。这在使用 `uv` 或其他非 `conda` 环境管理器时很有用：

```bash
# Ubuntu/Debian
sudo apt install ffmpeg

# macOS（Apple Silicon）
brew install ffmpeg
```

> [!IMPORTANT]
> 系统级 `ffmpeg` **仅支持 PyTorch >= 2.10**（TorchCodec ≥ 0.10）。对于较旧的 PyTorch 版本，你**必须**改用 `conda install ffmpeg -c conda-forge`。

</hfoption>
</hfoptions>
<!-- prettier-ignore-end -->

## 第 3 步：安装 LeRobot 🤗

基础的 `lerobot` 安装刻意保持**轻量级**——仅包含核心机器学习依赖（PyTorch、torchvision、numpy、opencv、einops、draccus、huggingface-hub、gymnasium、safetensors）。较重的依赖通过可选扩展（extras）进行门控，因此你只需安装所需的部分。

### 从源码安装

首先，克隆仓库并进入目录：

```bash
git clone https://github.com/huggingface/lerobot.git
cd lerobot
```

然后以可编辑模式安装库。如果你打算为代码做贡献，这将非常有用。

<!-- prettier-ignore-start -->
<hfoptions id="install_lerobot_src">
<hfoption id="conda">
```bash
pip install -e ".[core_scripts]"  # 用于机器人工作流程（记录、回放、校准）
pip install -e ".[training]"      # 用于训练策略
pip install -e ".[all]"           # 全部（所有策略、环境、硬件、开发工具）
```
</hfoption>
<hfoption id="uv">
```bash
uv pip install -e ".[core_scripts]"  # 用于机器人工作流程（记录、回放、校准）
uv pip install -e ".[training]"      # 用于训练策略
uv pip install -e ".[all]"           # 全部（所有策略、环境、硬件、开发工具）
```
</hfoption>

_这仅安装核心机器学习依赖。大多数工作流程需要添加额外的扩展。_

**功能扩展：**
LeRobot 提供了**按功能划分的扩展**，映射到常见工作流程。如果你使用 `uv`，请将下方命令中的 `pip install` 替换为 `uv pip install`。

| 扩展           | 添加的内容                                      | 典型使用场景                      |
| -------------- | ----------------------------------------------- | --------------------------------- |
| `dataset`      | `datasets`、`av`、`torchcodec`、`jsonlines`     | 加载和创建数据集                  |
| `training`     | `dataset` + `accelerate`、`wandb`               | 训练策略                          |
| `hardware`     | `pynput`、`pyserial`、`deepdiff`                | 连接真实机器人                    |
| `viz`          | `rerun-sdk`                                     | 记录/评估期间的可视化             |

**组合扩展**将功能扩展组合，适用于常见的 CLI 脚本：

| 扩展            | 包含的内容                       | 典型使用场景                                              |
| --------------- | -------------------------------- | --------------------------------------------------------- |
| `core_scripts`  | `dataset` + `hardware` + `viz`   | `lerobot-record`、`lerobot-replay`、`lerobot-calibrate`   |
| `evaluation`    | `av`                             | `lerobot-eval`（按需添加策略和环境扩展）                   |
| `dataset_viz`   | `dataset` + `viz`                | `lerobot-dataset-viz`、`lerobot-imgtransform-viz`          |

```bash
pip install 'lerobot[core_scripts]'          # 记录、回放、校准
pip install 'lerobot[training]'              # 训练策略
pip install 'lerobot[core_scripts,training]' # 记录 + 训练
pip install 'lerobot[all]'                   # 全部
```

**策略、环境和硬件扩展**仍可用于特定依赖：

```bash
pip install 'lerobot[pi]'             # Pi0/Pi0.5/Pi0-FAST 策略依赖
pip install 'lerobot[smolvla]'        # SmolVLA 策略依赖
pip install 'lerobot[diffusion]'      # 扩散策略依赖（diffusers）
pip install 'lerobot[aloha,pusht]'    # 仿真环境
pip install 'lerobot[feetech]'        # Feetech 电机支持
```

_多个扩展可以组合使用（例如 `.[core_scripts,pi,pusht]`）。有关可用扩展的完整列表，请参阅 `pyproject.toml`。_

### PyTorch CUDA 变体（仅 Linux）

在 Linux 上，安装路径决定了你获取哪个 CUDA wheel。macOS 和 Windows 安装使用 PyPI 默认值（分别为 MPS / CPU / CUDA-Windows wheel），可以跳过本节。

<!-- prettier-ignore-start -->

<hfoptions id="cuda_variant">
<hfoption id="uv-source">

**通过 `uv` 从源码安装（`uv sync` 或 `uv pip install -e .`）**

`torch` 和 `torchvision` 由项目固定到 **CUDA 12.8** PyTorch 索引（`https://download.pytorch.org/whl/cu128`，驱动最低版本 **570.86**）——覆盖 Ampere/Ada/Hopper/Blackwell GPU。典型 NVIDIA 设置无需额外操作。

要覆盖为不同的 CUDA 变体：

```bash
uv pip install --force-reinstall torch torchvision \
    --index-url https://download.pytorch.org/whl/cu126   # 较旧的驱动；或 cu130，适用于驱动 ≥ 580 的 Blackwell
```

</hfoption>
<hfoption id="pip-conda">

**通过 `pip`/`conda` 从源码安装，或从 PyPI `pip install lerobot`**

PyPI 默认的 torch wheel 当前是包含 cu130 的 Linux wheel，驱动最低版本 **580.65**。

要选择特定的 CUDA 变体：

**使用 `pip` 或 `conda`**——首先使用显式索引安装 torch，然后安装 lerobot：

```bash
pip install --index-url https://download.pytorch.org/whl/cu128 torch torchvision
pip install -e ".[all]"          # 从源码
# — 或 —
pip install lerobot              # 从 PyPI
```

**使用 `uv` 从 PyPI 安装**——通过 `--torch-backend` 一行搞定（uv ≥ 0.6）：

```bash
uv pip install --torch-backend cu128 lerobot
```

支持的参数值包括 `auto`、`cpu`、`cu126`、`cu128`、`cu129`、`cu130`，以及各种 `rocm*` 和 `xpu`。根据你的驱动进行相应替换。

</hfoption>
</hfoptions>

### 故障排除

如果遇到构建错误，你可能需要安装额外的系统依赖：`cmake`、`build-essential` 和 `ffmpeg` 库。
在 Linux 上安装这些依赖：

```bash
sudo apt-get install cmake build-essential python3-dev pkg-config libavformat-dev libavcodec-dev libavdevice-dev libavutil-dev libswscale-dev libswresample-dev libavfilter-dev
```

其他系统请参阅：[编译 PyAV](https://pyav.org/docs/develop/overview/installation.html#bring-your-own-ffmpeg)

## 可选依赖

LeRobot 为特定功能提供可选扩展。多个扩展可以组合使用（例如 `.[aloha,feetech]`）。有关所有可用扩展，请参阅 `pyproject.toml`。如果你使用 `uv`，请将下方命令中的 `pip install` 替换为 `uv pip install`。

### 仿真

安装环境包：`aloha`（[gym-aloha](https://github.com/huggingface/gym-aloha)）或 `pusht`（[gym-pusht](https://github.com/huggingface/gym-pusht)）。
这些扩展会自动包含 `dataset` 扩展。

```bash
pip install -e ".[aloha]" # 或 "[pusht]"
```

### 电机控制

对于 Koch v1.1，安装 Dynamixel SDK；对于 SO100/SO101/Moss，安装 Feetech SDK。

```bash
pip install -e ".[feetech]" # 或 "[dynamixel]"
```

### 实验跟踪

Weights and Biases 包含在 `training` 扩展中。要使用 [Weights and Biases](https://docs.wandb.ai/quickstart) 进行实验跟踪，请登录：

```bash
wandb login
```
