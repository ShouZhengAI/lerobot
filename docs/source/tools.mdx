# 工具

LeRobot v3.1 支持策略中的**工具调用**——助手消息可以发出结构化的调用，如 `say(text="OK, starting now")`，运行时将其分发到实际的实现（TTS、控制器、日志记录器等）。

本页内容涵盖：

1. 工具目录存放位置。
2. 标注流水线（annotation pipeline）如何生成工具调用原子。
3. 如何添加你自己的工具。

## 工具声明位置

分为两层。

**目录**——OpenAI 风格的函数结构描述列表——存放在每个数据集的 `meta/info.json["tools"]` 中。示例：

```json
{
  "features": { "...": "..." },
  "tools": [
    {
      "type": "function",
      "function": {
        "name": "say",
        "description": "通过 TTS 执行器向用户说出一句简短的话语。",
        "parameters": {
          "type": "object",
          "properties": {
            "text": {
              "type": "string",
              "description": "要说的原文文本。"
            }
          },
          "required": ["text"]
        }
      }
    }
  ]
}
```

通过数据集元数据访问器读取：

```python
from lerobot.datasets.dataset_metadata import LeRobotDatasetMetadata

meta = LeRobotDatasetMetadata(repo_id="pepijn/super_poulain_final_annotations")
tools = meta.tools     # list[dict] — OpenAI 工具结构描述
```

如果数据集的 `info.json` 没有声明任何工具，`meta.tools` 返回 `lerobot.datasets.language` 中的 `DEFAULT_TOOLS`——当前是一个单项列表，包含规范的 `say` 结构描述。因此，未标注的数据集和聊天模板消费者无需任何配置即可继续工作：

```python
prompt_str = tokenizer.apply_chat_template(
    sample["messages"],
    tools=meta.tools,                 # 两种方式均可
    add_generation_prompt=False,
    tokenize=False,
)
```

**实现层**——可运行的 Python 代码——将位于 `src/lerobot/tools/` 目录下，每个工具一个文件。运行时调度器和规范的 `say` 实现（封装了 Kyutai 的 pocket-tts）不属于本文档描述的目录层；目前这一层仅提供结构描述存储和 `DEFAULT_TOOLS` 回退常量。

## 每行的工具_调用_

上面的目录描述了_可以调用什么_。实际的_调用_——函数名称加上参数值——按行存储在 `language_events` 中的助手原子（assistant atoms）上：

```python
{
  "role": "assistant",
  "content": null,
  "style": null,
  "timestamp": 12.4,
  "camera": null,
  "tool_calls": [
    { "type": "function",
      "function": { "name": "say", "arguments": { "text": "马上处理。" } } }
  ]
}
```

配方（Recipe）通过 `tool_calls_from` 将这些调用拼接到渲染后的消息中：

```yaml
user_interjection_response:
  bindings:
    speech: "emitted_at(t, role=assistant, tool_name=say)"
  messages:
    - { role: user, content: "${task}", stream: high_level }
    - {
        role: assistant,
        content: "${current_plan}",
        stream: high_level,
        target: true,
        tool_calls_from: speech,
      }
```

模型的训练目标是一个同时携带计划文本_和_ `say` 工具调用的助手轮次。在推理时，运行时将生成的文本解析回结构化的 `tool_calls`，并分发到对应的实现。

## 如何添加你自己的工具

> **注意：** 下面的第 2 步和第 3 步描述了运行时层（`src/lerobot/tools/`、`Tool` 协议、`TOOL_REGISTRY`、`get_tools(meta)`），这些不属于目前已发布的目录层——这些模块在代码树中尚不存在。仅执行第 1 步就足以使工具通过 `meta.tools` 在聊天模板中可见，让模型学会_生成_调用；在推理时执行该调用需要运行时层。

三步完成。具体示例：一个 `record_observation` 工具，策略可以调用它在常规控制循环之外捕获额外的观测数据。

### 第 1 步——声明结构描述

在 `meta/info.json["tools"]` 下添加一个条目。你可以在_运行标注流水线之前_直接编辑磁盘上的文件（它会被保留），或者通过配置参数将其传递给 `lerobot-annotate`。

```json
{
  "tools": [
    { "type": "function", "function": { "name": "say", "...": "..." } },
    {
      "type": "function",
      "function": {
        "name": "record_observation",
        "description": "为用户捕获一张高分辨率静态图像。",
        "parameters": {
          "type": "object",
          "properties": {
            "label": {
              "type": "string",
              "description": "保存图像的简短标签。"
            }
          },
          "required": ["label"]
        }
      }
    }
  ]
}
```

该结构描述完全遵循 OpenAI 的函数调用约定，因此聊天模板可以原生地渲染它。

### 第 2 步——实现调用

创建 `src/lerobot/tools/record_observation.py`：

```python
from .base import Tool
from typing import Any

RECORD_OBSERVATION_SCHEMA: dict[str, Any] = { "...": "..." }   # 与上面的 JSON 对应


class RecordObservationTool:
    name = "record_observation"
    schema = RECORD_OBSERVATION_SCHEMA

    def __init__(self, schema: dict | None = None, output_dir: str = "."):
        self.output_dir = output_dir

    def call(self, arguments: dict) -> str:
        label = arguments["label"]
        # ... 将最新的摄像头帧保存到 <output_dir>/<label>.png ...
        return f"saved {label}.png"
```

每个工具一个文件可以保持依赖隔离——`record_observation` 可能需要引入 `pillow`，而 `say` 需要引入 `pocket-tts`。仅安装所需工具的用户可以避免繁重的传递依赖。

### 第 3 步——注册

添加到 `src/lerobot/tools/registry.py`：

```python
from .record_observation import RecordObservationTool

TOOL_REGISTRY["record_observation"] = RecordObservationTool
```

这样就完成了。在运行时，`get_tools(meta)` 查找 `meta.tools` 中的每个结构描述、实例化匹配的已注册类，并返回一个名称到实例的字典，供调度器路由使用。

如果你希望在_不_编写实现的情况下使用某个工具（例如，仅用于训练时的聊天模板格式化），只需第 1 步就足够了——模型仍然学会_生成_调用。第 2 步和第 3 步仅用于在推理时实际_执行_调用。
