# LIBERO-plus

LIBERO-plus 是一个基于 [LIBERO](./libero) 构建的**鲁棒性基准测试**，专门用于评估视觉-语言-动作（VLA，Vision-Language-Action）模型。它通过对原始 LIBERO 任务集施加**七个独立的扰动维度**来系统性地测试策略的可靠性，揭示标准基准测试无法发现的失效模式。

- 论文：[In-depth Robustness Analysis of Vision-Language-Action Models](https://arxiv.org/abs/2510.13626)
- GitHub：[sylvestf/LIBERO-plus](https://github.com/sylvestf/LIBERO-plus)
- 数据集：[lerobot/libero_plus](https://huggingface.co/datasets/lerobot/libero_plus)

![LIBERO-plus 基准测试扰动维度概览](https://github.com/sylvestf/LIBERO-plus/raw/main/static/images/libero-plus.jpg)

## 扰动维度

LIBERO-plus 通过对每个原始 LIBERO 任务沿以下方向施加扰动，创建了约 10,000 个任务变体：

| 维度             | 变化内容                                      |
| ---------------- | --------------------------------------------- |
| 物体布局         | 目标位置、干扰物体的存在与否                  |
| 摄像头视角       | 摄像头位置、方向、视场角                      |
| 机器人初始状态   | 机械臂起始姿态                                |
| 语言指令         | 经过 LLM 重写的任务描述（释义/同义词替换）     |
| 光照条件         | 光照强度、方向、颜色、阴影                    |
| 背景纹理         | 场景表面和物体外观                            |
| 传感器噪声       | 光度失真和图像退化                            |

## 可用任务套件

LIBERO-plus 覆盖与 LIBERO 相同的五个套件：

| 套件           | CLI 名称          | 任务数 | 最大步数 | 描述                        |
| -------------- | ----------------- | ------ | -------- | --------------------------- |
| LIBERO-Spatial | `libero_spatial`  | 10     | 280      | 需要推理空间关系的任务      |
| LIBERO-Object  | `libero_object`   | 10     | 280      | 围绕操作不同物体的任务      |
| LIBERO-Goal    | `libero_goal`     | 10     | 300      | 目标条件任务，目标会变化    |
| LIBERO-90      | `libero_90`       | 90     | 400      | 来自 LIBERO-100 集合的短周期任务 |
| LIBERO-Long    | `libero_10`       | 10     | 520      | 来自 LIBERO-100 集合的长周期任务 |

<Tip warning={true}>
  安装 LIBERO-plus 会**替换**原版 LIBERO——它会卸载 `hf-libero`，使得 `import libero` 解析到 LIBERO-plus 分支版本。两者无法同时安装。要切换回原版 LIBERO，请卸载分支版本并重新运行 `pip install -e ".[libero]"`。
</Tip>

## 安装

### 系统依赖（仅限 Linux）

```bash
sudo apt install libexpat1 libfontconfig1-dev libmagickwand-dev
```

### Python 包

```bash
pip install -e ".[libero]" "robosuite==1.4.1" bddl easydict mujoco wand scikit-image gym
git clone https://github.com/sylvestf/LIBERO-plus.git
cd LIBERO-plus && pip install --no-deps -e .
pip uninstall -y hf-libero  # 使 `import libero` 解析到分支版本
```

LIBERO-plus 从其 GitHub 分支安装，而非 pyproject 的 extras 依赖——该分支以 pip 无法处理的命名空间包形式发布，因此必须克隆并添加到 `PYTHONPATH`。请参见 `docker/Dockerfile.benchmark.libero_plus` 查看标准安装方式。需要 MuJoCo，因此仅支持 Linux。

<Tip>
在运行评估前设置 MuJoCo 渲染后端：

```bash
export MUJOCO_GL=egl   # 无头服务器 / HPC / 云环境
```

</Tip>

### 下载 LIBERO-plus 资源文件

LIBERO-plus 将其扩展资源包单独发布。从 [Hugging Face 数据集](https://huggingface.co/datasets/Sylvest/LIBERO-plus/tree/main) 下载 `assets.zip` 并将其解压到 LIBERO-plus 包目录中：

```bash
# 安装包后，找到其安装位置：
python -c "import libero; print(libero.__file__)"
# 然后将 assets.zip 解压到 <package_root>/libero/assets/
```

## 评估

### 默认评估（推荐）

跨四个标准套件进行评估（每个任务 10 个片段）：

```bash
lerobot-eval \
  --policy.path="your-policy-id" \
  --env.type=libero_plus \
  --env.task=libero_spatial,libero_object,libero_goal,libero_10 \
  --eval.batch_size=1 \
  --eval.n_episodes=10 \
  --env.max_parallel_tasks=1
```

### 单套件评估

在单个 LIBERO-plus 套件上评估：

```bash
lerobot-eval \
  --policy.path="your-policy-id" \
  --env.type=libero_plus \
  --env.task=libero_spatial \
  --eval.batch_size=1 \
  --eval.n_episodes=10
```

- `--env.task` 选择套件（`libero_spatial`、`libero_object` 等）。
- `--env.task_ids` 限制为特定的任务索引（如 `[0]`、`[1,2,3]` 等）。省略则运行套件中的所有任务。
- `--eval.batch_size` 控制并行运行的环境数量。
- `--eval.n_episodes` 设置每个任务运行的片段数。

### 多套件评估

通过逗号分隔的列表一次性在多个套件上评估策略：

```bash
lerobot-eval \
  --policy.path="your-policy-id" \
  --env.type=libero_plus \
  --env.task=libero_spatial,libero_object \
  --eval.batch_size=1 \
  --eval.n_episodes=10
```

### 控制模式

LIBERO-plus 支持两种控制模式——`relative`（相对，默认）和 `absolute`（绝对）。不同的 VLA 检查点使用不同的动作参数化方式，因此请确保模式与你的策略匹配：

```bash
--env.control_mode=relative   # 或 "absolute"
```

### 策略输入和输出

**观测信息：**

- `observation.state` — 8 维本体感受特征（末端执行器位置、轴角方向、夹爪位置）
- `observation.images.image` — 主摄像头视图（`agentview_image`），HWC uint8 格式
- `observation.images.image2` — 腕部摄像头视图（`robot0_eye_in_hand_image`），HWC uint8 格式

**动作：**

- `Box(-1, 1, shape=(7,))` 中的连续控制——6 维末端执行器增量 + 1 维夹爪

### 推荐评估片段数

为了可重复的基准测试，建议在全部四个标准套件（Spatial、Object、Goal、Long）中每个任务运行 **10 个片段**。总计 400 个片段，与已发表结果使用的协议一致。

## 训练

### 数据集

LIBERO-plus 的 LeRobot 格式训练数据集可在以下位置获取：

- [lerobot/libero_plus](https://huggingface.co/datasets/lerobot/libero_plus)

### 训练命令示例

```bash
lerobot-train \
    --policy.type=smolvla \
    --policy.repo_id=${HF_USER}/smolvla_libero_plus \
    --policy.load_vlm_weights=true \
    --dataset.repo_id=lerobot/libero_plus \
    --env.type=libero_plus \
    --env.task=libero_spatial \
    --output_dir=./outputs/ \
    --steps=100000 \
    --batch_size=4 \
    --eval.batch_size=1 \
    --eval.n_episodes=1 \
    --env_eval_freq=1000
```

## 与 LIBERO 的关系

LIBERO-plus 是 LIBERO 的直接扩展：

- 相同的 Python gym 接口（`LiberoEnv`、`LiberoProcessorStep`）
- 相同的摄像头命名和观测/动作格式
- 相同的任务套件名称
- 使用相同的 `libero` Python 包名安装（不同的 GitHub 仓库）

要使用原始 LIBERO 基准测试，请参见 [LIBERO](./libero) 并使用 `--env.type=libero`。
