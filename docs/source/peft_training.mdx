# 使用 🤗 PEFT 进行参数高效微调

[🤗 PEFT](https://github.com/huggingface/peft)（Parameter-Efficient Fine-Tuning，参数高效微调）是一个高效适配大型预训练模型的库，适用于预训练策略（如 SmolVLA、π₀ 等）以适应新任务，无需训练模型的所有参数，同时能取得可比的性能。

安装 `lerobot[peft]` 可选包即可启用 PEFT 支持。

如需了解所有可用的适配方法，请参阅 [🤗 PEFT 文档](https://huggingface.co/docs/peft/index)。

## 训练 SmolVLA

本节将演示如何使用 PEFT 在 libero 数据集上训练预训练的 SmolVLA 策略。为简洁起见，我们仅使用 `libero_spatial` 子集进行训练。我们将使用 `lerobot/smolvla_base` 作为模型进行参数高效微调：

```
lerobot-train \
 --policy.path=lerobot/smolvla_base \
 --policy.repo_id=your_hub_name/my_libero_smolvla \
 --dataset.repo_id=HuggingFaceVLA/libero \
 --policy.output_features=null \
 --policy.input_features=null \
 --policy.optimizer_lr=1e-3 \
 --policy.scheduler_decay_lr=1e-4 \
 --env.type=libero \
 --env.task=libero_spatial \
 --steps=100000 \
 --batch_size=32 \
 --peft.method_type=LORA \
 --peft.r=64 \
 --peft.lora_alpha=64
```

请注意 `--peft.method_type` 参数，它允许你选择使用哪种 PEFT 方法。这里我们使用了
[LoRA](https://huggingface.co/docs/peft/main/en/package_reference/lora)（Low-Rank Adapter，低秩适配器），这可能是目前最流行的微调方法。低秩适配意味着我们只微调一个秩相对较低的矩阵，而不是完整的权重矩阵。这个秩可以通过 `--peft.r` 参数指定，LoRA 的缩放因子通过 `--peft.lora_alpha` 指定（其中 `scaling = lora_alpha / r`）。秩越高，效果越接近全参数微调。

还有更复杂的方法，这些方法具有更多参数。但这些方法尚未支持，如果你希望支持特定的 PEFT 方法，欢迎提交 issue。

默认情况下，PEFT 将针对 SmolVLA 中 LM 专家（LM expert）的 `q_proj` 和 `v_proj` 层。它还会针对状态（state）和动作（action）投影矩阵，因为它们很可能与任务相关。如果需要针对不同的层，可以使用 `--peft.target_modules` 指定目标层。请参考相应 PEFT 方法的文档了解支持的输入格式（例如 [LoRA 的 target_modules 文档](https://huggingface.co/docs/peft/main/en/package_reference/lora#peft.LoraConfig.target_modules)）。通常支持后缀列表或正则表达式。例如，要针对 `lm_expert` 的 MLP 层而不是 `q` 和 `v` 投影，请使用：

```
--peft.target_modules='(model\.vlm_with_expert\.lm_expert\..*\.(down|gate|up)_proj|.*\.(state_proj|action_in_proj|action_out_proj|action_time_mlp_in|action_time_mlp_out))'
```

若你需要对某层进行全参数微调而不是仅适配，可以将层后缀列表提供给 `--peft.full_training_modules` 参数：

```
--peft.full_training_modules=["state_proj"]
```

学习率和调度的目标学习率通常可以比全参数微调的学习率扩大 10 倍（例如，全参数微调学习率为 1e-4，那么使用 LoRA 时可以为 1e-3）。
