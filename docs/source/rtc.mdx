# 实时分块（RTC）

实时分块（Real-Time Chunking，RTC）是一种推理时方法，使基于流匹配的大规模机器人策略（如 [Pi0](./pi0)、[Pi0.5](./pi05) 和 [SmolVLA](./smolvla)）能够在推理延迟较高的情况下，生成平滑、连续且具有响应性的运动。

这些策略生成的是未来动作的分块（例如每次生成 50 步），而不是单个动作。由于模型较大，生成每个分块所需的时间比机器人执行该分块的时间更长。简单地执行分块会导致问题，例如暂停、运动不连贯，或者当下一个分块延迟到达时与已执行的动作不一致，导致策略突然变化。

RTC 通过以下方式解决这些问题：在机器人继续执行当前分块的同时异步生成下一个分块，并通过引导使新分块与之前已执行的部分平滑对齐。

## RTC 工作原理（简化版）

RTC 让机器人在运动过程中同时提前计算下一步。当机器人正在执行一个动作分块时，RTC 会提前开始创建下一个分块。但由于新分块就绪时机器人已经继续移动了一段距离，RTC 必须确保新分块与机器人当前正在执行的动作平滑衔接。

为实现这一点，RTC 将新分块的起始部分视为一个修补（inpainting）或"填空"问题：它轻柔地调整新分块的起始部分，使其自然地融合到机器人正在进行的运动中。最终效果是没有暂停、没有突然跳跃。

从技术角度来看，RTC 在流匹配去噪过程中添加了一个引导项，强制新分块中重叠的时间步与前一分块已执行的部分保持接近，通常使用软过渡掩码（soft transition mask）来实现。

## 快速开始

### 安装

RTC 已内置在 LeRobot 中。只需安装所需的策略依赖：

```bash
# 对于 Pi0 或 Pi0.5
pip install -e ".[pi]"

# 对于 SmolVLA
pip install -e ".[smolvla]"
```

### 在 Pi0 中使用 RTC

可以使用 `lerobot-rollout --strategy.type=base --inference.type=rtc` 在实际机器人上部署 RTC。以下代码片段提供了一个简化的伪代码示例，说明 RTC 如何在你的流程中与 Pi0 配合使用：

```python
from lerobot.policies.pi0 import PI0Policy, PI0Config
from lerobot.configs import RTCAttentionSchedule
from lerobot.policies.rtc import RTCConfig, ActionQueue

# 加载启用了 RTC 的 Pi0
policy_cfg = PI0Config()

# 启用 RTC
policy_cfg.rtc_config = RTCConfig(
    enabled=True,
    execution_horizon=10,  # 与前一动作分块混合的步数
    max_guidance_weight=10.0,  # 强制一致性的强度
    prefix_attention_schedule=RTCAttentionSchedule.EXP,  # 指数混合
)

# 加载策略
policy = PI0Policy.from_pretrained("lerobot/pi0_base", policy_cfg=policy_cfg, device="cuda")

# 现在使用 RTC 参数调用 predict_action_chunk
inference_delay = 4  # 推理延迟的步数，该值应根据策略的推理延迟计算

# 初始化动作队列
action_queue = ActionQueue(policy_cfg.rtc_config)

# 在单独的线程中启动以下函数
def get_actions():
  while True:
    if should_get_actions:

      prev_actions = action_queue.get_left_over()
      obs = get_robot_observations(robot)

      # 使用 RTC 生成动作
      actions = policy.predict_action_chunk(
          obs,
          inference_delay=inference_delay,
          prev_chunk_left_over=prev_actions,
      )

      action_queue.merge(
          actions, actions, inference_delay
      )

for step in range(num_steps):
    action = action_queue.get()

    # 执行前 N 个动作
    execute_actions(action)
```

## 关键参数

`RTCConfig` 包含以下可调参数：

**`execution_horizon`**：与前一动作分块保持一致的步数。值越大，过渡越平滑，但响应性可能降低。

典型值：8-12 步

```python
RTCConfig(execution_horizon=10)
```

**`max_guidance_weight`**：强制与前一动作分块保持一致的强度。这是一个超参数，可在过渡平滑度和策略响应性之间进行权衡。对于 10 步流匹配（SmolVLA、Pi0、Pi0.5），10.0 是一个较优值。

**`prefix_attention_schedule`**：如何在重叠区域中加权一致性。

- `LINEAR`：从 `inference_delay` 到 `execution_horizon` 线性衰减
- `EXP`：指数衰减（推荐入门使用）
- `ONES`：在整个 `execution_horizon` 范围内全部权重
- `ZEROS`：二值化（在 `inference_delay` 之前全部权重，之后归零）

**`inference_delay`**：系统推理延迟的步数。该参数传递给 `predict_action_chunk()` 而非配置，因为它可能随运行状态变化。

## 离线测试 RTC

在实际机器人上运行之前，可以使用数据集样本测试 RTC，以可视化其工作原理：

```bash
python examples/rtc/eval_dataset.py \
    --policy.path=lerobot/pi0_libero_finetuned \
    --dataset.repo_id=HuggingFaceVLA/libero \
    --rtc.execution_horizon=10 \
    --rtc.max_guidance_weight=10.0 \
    --device=cuda
```

该脚本会生成去噪过程的可视化结果，比较标准生成（左侧）与 RTC（右侧）。在 RTC 的图表中，可以看到前几步（蓝/紫色线条）被引导以匹配红色真实轨迹（前一动作分块的尾部），从而确保分块之间的平滑过渡。

## 与 LeRobot 中异步推理的区别

RTC 和[异步推理](./async)都能改善机器人实时控制，但它们解决的是不同的问题。

| 方面          | 异步推理 (Async Inference)                                         | RTC                                                  |
| ------------- | ------------------------------------------------------------------- | ---------------------------------------------------- |
| **问题**      | 等待推理时出现的空闲帧                                               | 动作分块之间的不连续性                                |
| **解决方案**  | 将预测与执行解耦                                                     | 引导新分块从前一分块平滑延续                          |
| **优势**      | 无需等待，连续动作输出                                               | 平滑过渡，自然运动                                    |
| **最佳适用**  | 异步推理最适合推理延迟高的大模型                                     | 基于流匹配的策略                                      |

**同时使用两者**可获得最大的平滑度和响应性！

## 高级用法：调试追踪

RTC 包含内置的调试追踪功能，帮助你了解推理过程中的情况：

```python
# 启用调试追踪
policy_cfg.rtc_config.debug = True
policy_cfg.rtc_config.debug_maxlen = 100

# 推理后访问调试数据
debug_data = policy.rtc_processor.get_debug_data()

# 可视化去噪步骤、校正等信息
from lerobot.policies.rtc.debug_visualizer import RTCDebugVisualizer
visualizer = RTCDebugVisualizer()
# ... 创建图表
```

请参阅 `examples/rtc/eval_dataset.py` 获取离线 RTC 可视化的完整示例。
