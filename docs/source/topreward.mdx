# TOPReward

TOPReward 是一种**零样本奖励模型**，它从现成的视觉语言模型（VLM, Vision-Language Model）中提取 token 对数概率作为机器人奖励信号。给定一段视频轨迹和一个任务指令，它返回 VLM 对该指令为真的对数似然——无需微调。

**论文**：[TOPReward: Token Probabilities as Hidden Zero-Shot Rewards for Robotics](https://arxiv.org/abs/2602.19313)
**项目主页**：[topreward.github.io](https://topreward.github.io/webpage/)
**原始代码**：[github.com/TOPReward/TOPReward](https://github.com/TOPReward/TOPReward)
**默认骨干网络**：[Qwen/Qwen3-VL-8B-Instruct](https://huggingface.co/Qwen/Qwen3-VL-8B-Instruct)

## 概述

TOPReward 让一个通用的 VLM **以机器人尝试完成任务的视频为条件**，判断一条任务指令为真的可能性。具体来说，给定：

- 一段轨迹视频（帧序列）。
- 一条任务指令（如"打开抽屉"）。

它会构建一个如下形式的聊天提示：

```text
<video>
"The above video shows a robot manipulation trajectory that completes the
 following task: <instruction> Decide whether the above statement is True
 or not. The answer is: True"
```

将其送入 VLM，对除最后一个 token 之外的所有内容进行标签掩码（label-mask），然后读出该 token 的对数概率——默认是模板末尾的纯文本 `"True"`。最终 `log P("True" | video + prompt + instruction)` 即为奖励值。

由于该方法仅依赖于一个冻结的 VLM，TOPReward 是**零样本**的：无需托管任何微调后的权重。LeRobot 中的"模型"是一个围绕 `transformers` 的 `Qwen3VLForConditionalGeneration` 加上标签掩码逻辑的轻量封装。处理器（processor）负责管理分词器并构建完整的聊天提示（采用 EO-1/Robometer 模式）。

## LeRobot 集成涵盖的内容

- 通过 LeRobot 的标准 `reward_model.type=topreward` 配置。
- 通过 `transformers` 的 `Qwen3VLForConditionalGeneration` API 加载 VLM。
- 在处理器中进行提示组装和分词（与上游 `QwenClient.compute_instruction_reward` 一致）。
- `compute_reward()` 为每个样本返回一个标量对数概率值。
- LeRobot 奖励模型的保存/加载——`save_pretrained` 仅写入 `config.json`（VLM 通过 `vlm_name` 标识）。
- 一个离线标注脚本，写入 `topreward_progress.parquet`（SARM 兼容模式）用于 RA-BC 训练和叠加可视化。

当前 LeRobot 移植版本**仅支持 Qwen3-VL 客户端**。其他上游客户端（Gemini、OpenAI、Gemma、Molmo）可作为后续扩展添加。

## 安装要求

1. 按照[安装指南](./installation)安装 LeRobot。
2. 安装 TOPReward 可选扩展：

```bash
pip install -e ".[topreward]"
```

或者，使用 `uv` 从源码检出目录安装：

```bash
uv sync --extra topreward
```

这将引入 `transformers`。首次运行 TOPReward 时，Hugging Face 还会从 Hub 下载 VLM 权重（Qwen3-VL-8B-Instruct 约 16 GB）。强烈建议使用 GPU。

## 模型输入与输出

TOPReward 期望接收：

- 一段轨迹视频或帧序列。
- 一条自然语言的任务描述。

在 LeRobot 数据集中，预处理器读取：

| 配置字段                     | 默认值                       | 说明                                         |
| ---------------------------- | ---------------------------- | -------------------------------------------- |
| `reward_model.image_key`     | `observation.images.top`     | TOPReward 使用的摄像头观测数据                |
| `reward_model.task_key`      | `task`                       | 补充数据中用于获取任务字符串的键              |
| `reward_model.max_frames`    | `16`                         | 每个样本的最大帧数上限                       |
| `reward_model.fps`           | `2.0`                        | 传递给 Qwen 视频处理器的元数据               |
| `reward_model.vlm_name`      | `Qwen/Qwen3-VL-8B-Instruct`  | 底层 VLM 的 Hugging Face Hub ID              |

模型返回：

- `compute_reward(batch)`：每个样本一个对数概率值。值越高表示任务与视频的对齐程度越好。当 `success_threshold` 为有限值时，返回二值化阈值结果。

## 使用方式

### 直接加载奖励模型

```python
from lerobot.rewards.topreward import TOPRewardConfig, TOPRewardModel

cfg = TOPRewardConfig(
    vlm_name="Qwen/Qwen3-VL-8B-Instruct",
    device="cuda",
)
reward_model = TOPRewardModel(cfg)
```

### 使用奖励工厂

```python
from lerobot.rewards import make_reward_model, make_reward_model_config, make_reward_pre_post_processors

cfg = make_reward_model_config(
    "topreward",
    vlm_name="Qwen/Qwen3-VL-8B-Instruct",
    device="cuda",
    image_key="observation.images.top",
)
reward_model = make_reward_model(cfg)
preprocessor, postprocessor = make_reward_pre_post_processors(cfg)
```

预处理器对完整提示（视频 + 前缀 + 指令后缀）进行分词，在 `observation.topreward.*` 下写入 Qwen-VL 的张量和 `prompt_length`。模型读取这些张量，基于 `prompt_length` 进行标签掩码，并提取对数概率奖励。

### 离线数据集标注

生成 `topreward_progress.parquet` 用于 RA-BC 训练和叠加视频：

```bash
# 稀疏-密集模式（每片段 15 个锚点，与上游一致）
uv run python -m lerobot.rewards.topreward.compute_rabc_weights \
    --dataset-repo-id lerobot/libero_10_image \
    --num-samples 15 \
    --device cuda
```

然后为任意片段渲染进度叠加视频：

```bash
uv run examples/dataset/create_progress_videos.py \
    --repo-id lerobot/libero_10_image \
    --episode 0 \
    --progress-file topreward_progress.parquet \
    --gif
```

## 配置说明

### 提示参数

默认提示与上游论文保持一致：

```text
prompt_prefix = "The above video shows a robot manipulation trajectory that completes the following task: "
prompt_suffix_template = "{instruction} Decide whether the above statement is True or not. The answer is: True"
```

两者均暴露在 `TOPRewardConfig` 中，可用于消融实验。后缀模板**必须**包含 `{instruction}`。

### 聊天模板

`add_chat_template=True` 会在分词之前使用分词器的聊天模板包裹完整提示（包括指令）。默认值为 `False`，与上游论文的主要实验一致。

## 局限性

- 当前 LeRobot 移植版本**仅支持推理和零样本**；`forward()` 未被重写，`is_trainable` 返回 `False`。
- 仅支持 **Qwen3-VL 系列**；其他上游客户端不在当前范围内。
- TOPReward 继承了底层 VLM 的偏见。
