## 遥操作（Teleoperate）

在本示例中，我们将演示如何遥操作 SO101 机器人。每个命令都附有对应的 API 示例。

请注意，与机器人关联的 `id` 用于存储校准文件。在使用相同设置进行遥操作、记录和评估时，务必使用相同的 `id`。

<hfoptions id="teleoperate_so101">
<hfoption id="命令">
```bash
lerobot-teleoperate \
    --robot.type=so101_follower \
    --robot.port=/dev/tty.usbmodem58760431541 \
    --robot.id=my_awesome_follower_arm \
    --teleop.type=so101_leader \
    --teleop.port=/dev/tty.usbmodem58760431551 \
    --teleop.id=my_awesome_leader_arm
```
</hfoption>
<hfoption id="API 示例">

<!-- prettier-ignore-start -->

```python
from lerobot.robots.so_follower import SO101Follower, SO101FollowerConfig
```

```python
while True:
    start_time = time.perf_counter()

    observation = robot.get_observation()
    action = teleop_device.get_action()
    robot.send_action(action)

```


现在你可以记录数据集了。要记录 5 个片段（episode）并将数据集上传到 Hub，请根据你的机器人调整以下代码，然后执行命令或 API 示例。

<hfoptions id="record">
<hfoption id="命令">
```bash
lerobot-record \
    --robot.type=so101_follower \
    --robot.port=/dev/tty.usbmodem585A0076841 \
    --robot.id=my_awesome_follower_arm \
    --robot.cameras="{ front: {type: opencv, index_or_path: 0, width: 1920, height: 1080, fps: 30}}" \
    --teleop.type=so101_leader \
    --teleop.port=/dev/tty.usbmodem58760431551 \
    --teleop.id=my_awesome_leader_arm \
    --display_data=true \
    --dataset.repo_id=${HF_USER}/record-test \
    --dataset.num_episodes=5 \
    --dataset.single_task="抓取黑色立方体" \
    --dataset.streaming_encoding=true \
    # --dataset.rgb_encoder.vcodec=auto \
    --dataset.encoder_threads=2
```
</hfoption>
<hfoption id="API 示例">

<!-- prettier-ignore-start -->
```python
from lerobot.cameras.opencv import OpenCVCameraConfig
from lerobot.datasets.lerobot_dataset import LeRobotDataset
from lerobot.utils.feature_utils import hw_to_dataset_features
from lerobot.robots.so_follower import SO101Follower, SO101FollowerConfig
from lerobot.teleoperators.so_leader.config_so_leader import SO101LeaderConfig
from lerobot.teleoperators.so_leader.so_leader import SO101Leader
from lerobot.common.control_utils import init_keyboard_listener
from lerobot.utils.utils import log_say
from lerobot.utils.visualization_utils import init_visualization
from lerobot.scripts.lerobot_record import record_loop
from lerobot.processor import make_default_processors

NUM_EPISODES = 5
FPS = 30
EPISODE_TIME_SEC = 60
RESET_TIME_SEC = 10
TASK_DESCRIPTION = "我的任务描述"

def main():
    # 创建机器人配置
    robot_config = SO101FollowerConfig(
        port="/dev/tty.usbmodem5AB90687491",
        id="my_follower_arm",
        cameras={
            "wrist": OpenCVCameraConfig(index_or_path=0, width=640, height=480, fps=30),
            "top": OpenCVCameraConfig(index_or_path=1, width=640, height=480, fps=30)
        }
    )

    teleop_config = SO101LeaderConfig(
        port="/dev/tty.usbmodem5AB90689011",
        id="my_leader_arm",
    )

    # 初始化机器人和遥操作器
    robot = SO101Follower(robot_config)
    teleop = SO101Leader(teleop_config)

    # 配置数据集特征
    action_features = hw_to_dataset_features(robot.action_features, "action")
    obs_features = hw_to_dataset_features(robot.observation_features, "observation")
    dataset_features = {**action_features, **obs_features}

    # 创建数据集
    dataset = LeRobotDataset.create(
        repo_id="<hf_username>/<dataset_repo_id>",
        fps=FPS,
        features=dataset_features,
        robot_type=robot.name,
        use_videos=True,
        image_writer_threads=4,
    )

    # 初始化键盘监听器和 rerun 可视化
    _, events = init_keyboard_listener()
    init_visualization("rerun", session_name="recording")

    # 连接机器人和遥操作器
    robot.connect()
    teleop.connect()

    # 创建所需的处理器
    teleop_action_processor, robot_action_processor, robot_observation_processor = make_default_processors()

    episode_idx = 0
    while episode_idx < NUM_EPISODES and not events["stop_recording"]:
        log_say(f"正在记录第 {episode_idx + 1} 个片段，共 {NUM_EPISODES} 个")

        record_loop(
            robot=robot,
            events=events,
            fps=FPS,
            teleop_action_processor=teleop_action_processor,
            robot_action_processor=robot_action_processor,
            robot_observation_processor=robot_observation_processor,
            teleop=teleop,
            dataset=dataset,
            control_time_s=EPISODE_TIME_SEC,
            single_task=TASK_DESCRIPTION,
            display_data=True,
        )

        # 如果未停止或需要重录，则重置环境
        if not events["stop_recording"] and (episode_idx < NUM_EPISODES - 1 or events["rerecord_episode"]):
            log_say("重置环境")
            record_loop(
                robot=robot,
                events=events,
                fps=FPS,
                teleop_action_processor=teleop_action_processor,
                robot_action_processor=robot_action_processor,
                robot_observation_processor=robot_observation_processor,
                teleop=teleop,
                control_time_s=RESET_TIME_SEC,
                single_task=TASK_DESCRIPTION,
                display_data=True,
            )

        if events["rerecord_episode"]:
            log_say("正在重新记录片段")
            events["rerecord_episode"] = False
            events["exit_early"] = False
            dataset.clear_episode_buffer()
            continue

        dataset.save_episode()
        episode_idx += 1

    # 最终确定数据集
    log_say("正在最终确定数据集...")
    dataset.finalize()
    # 清理
    log_say("停止记录")
    robot.disconnect()
    teleop.disconnect()
    dataset.push_to_hub()


if __name__ == "__main__":
    main()
```

#### 记录函数

`record` 函数提供了一套用于在机器人操作过程中捕获和管理数据的工具：

##### 1. 数据存储

- 数据使用 `LeRobotDataset` 格式存储，记录期间会保存在磁盘上。
- 默认情况下，记录完成后数据集会被推送到你的 Hugging Face 页面。
  - 要禁用上传，请使用 `--dataset.push_to_hub=False`。

##### 2. 检查点与恢复

- 记录过程中会自动创建检查点（checkpoint）。
- 如果出现问题，或者你想在同一数据集中记录更多片段，可以通过重新运行相同的命令并加上 `--resume=true` 来恢复。恢复记录时，`--dataset.num_episodes` 必须设置为**要额外记录的片段数量**，而不是数据集中目标片段总数！请确保同时设置 `--dataset.root="local_path"`，这是一个保存数据集新部分的本地路径，是恢复记录所必需的。
- 要重新开始记录，请**手动删除**数据集目录。

##### 3. 记录参数

使用命令行参数设置数据记录流程：

- `--dataset.episode_time_s=60`
  每个数据记录片段的持续时间（默认值：**60 秒**）。
- `--dataset.reset_time_s=60`
  每个片段后重置环境的持续时间（默认值：**60 秒**）。
- `--dataset.num_episodes=50`
  要记录的总片段数（默认值：**50**）。

##### 4. 记录过程中的键盘控制

使用键盘快捷键控制数据记录流程：

- 按**右箭头键（`→`）**或 **`n`**：提前结束当前片段或重置时间，并进入下一个。
- 按**左箭头键（`←`）**或 **`r`**：取消当前片段并重新记录。
- 按**退出键（`ESC`）**或 **`q`**：立即停止会话，编码视频并上传数据集。

<Tip>

这些控制流程快捷键适用于 **X11、Wayland 以及无头/SSH** 会话。当全局键盘后端不可用时（Wayland、无头机器，或未授予辅助功能权限的 macOS），`lerobot-record` 会自动从终端读取相同的按键——请在交互式终端中启动它并保持焦点。你也可以使用字母等效键 **`n`**（下一个，与 `→` 相同）、**`r`**（重新记录，与 `←` 相同）和 **`q`**（退出，与 `ESC` 相同）。不需要设置 `$DISPLAY`。

这仅适用于记录控制流程。键盘**遥操作**（用键盘驱动机器人）仍然需要全局键盘后端，因此它只在 X11 会话、Windows 桌面或已授予辅助功能/输入监控权限的 macOS 上有效——不适用于 Wayland 或无头会话。

</Tip>

#### 数据收集提示

熟悉数据记录后，你可以创建更大的数据集用于训练。一个好的起始任务是抓取不同位置的物体并将其放入收纳盒。我们建议至少记录 50 个片段，每个位置 10 个片段。保持摄像头固定，并在整个记录过程中保持一致抓取行为。同时确保你操作的物体在摄像头画面中可见。一个好的经验法则是：你应当能够仅通过观察摄像头图像就完成该任务。

在接下来的章节中，你将训练神经网络。在实现可靠的抓取性能后，你可以开始在数据收集过程中引入更多变化，例如更多的抓取位置、不同的抓取技术以及改变摄像头位置。

避免过快地引入过多变化，因为这可能会影响效果。

如果你想更深入地了解这个重要话题，可以查看我们撰写的关于[什么构成了好数据集](https://huggingface.co/blog/lerobot-datasets#what-makes-a-good-dataset)的博客文章。

#### 故障排除：

- 在 Linux 上，记录控制流程按键（箭头键、退出键）在 X11、Wayland 和无头/SSH 会话中均可使用，只要 `lerobot-record` 在交互式终端中运行即可——无需设置 `$DISPLAY`。如果按键无效，请确保你在交互式（TTY）终端中运行（而不是管道/非 TTY 会话），并且终端处于焦点状态；字母等效键 `n` / `r` / `q` 同样有效。键盘_遥操作_（相对于记录控制流程）仍然需要全局键盘后端——X11 会话、Windows 桌面或已授予辅助功能/输入监控权限的 macOS——在 Wayland 或无头机器上不可用。请参阅 [pynput 限制说明](https://pynput.readthedocs.io/en/latest/limitations.html#linux)。

## 回放片段

一个有用的功能是 `replay` 函数，它允许你回放已记录的任何片段，或来自任何现有数据集的片段。该功能帮助你测试机器人动作的可重复性，并评估相同型号机器人之间的可迁移性。

你可以使用以下命令或 API 示例在机器人上回放第一个片段：

<hfoptions id="replay">
<hfoption id="命令">
```bash
lerobot-replay \
    --robot.type=so101_follower \
    --robot.port=/dev/tty.usbmodem58760431541 \
    --robot.id=my_awesome_follower_arm \
    --dataset.repo_id=${HF_USER}/record-test \
    --dataset.episode=0 # 选择你要回放的片段
```
</hfoption>
<hfoption id="API 示例">

<!-- prettier-ignore-start -->
```python
import time

from lerobot.datasets import LeRobotDataset
from lerobot.robots.so_follower import SO100Follower, SO100FollowerConfig
from lerobot.utils.robot_utils import precise_sleep
from lerobot.utils.utils import log_say

episode_idx = 0

robot_config = SO100FollowerConfig(port="/dev/tty.usbmodem5AB90687491", id="my_follower_arm")

robot = SO100Follower(robot_config)
robot.connect()

dataset = LeRobotDataset("<hf_username>/<dataset_repo_id>", episodes=[episode_idx])
actions = dataset.select_columns("action")

log_say(f"正在回放第 {episode_idx} 个片段")
for idx in range(dataset.num_frames):
    t0 = time.perf_counter()

    action = {
        name: float(actions[idx]["action"][i]) for i, name in enumerate(dataset.features["action"]["names"])
    }
    robot.send_action(action)

    precise_sleep(max(1.0 / dataset.fps - (time.perf_counter() - t0), 0.0))

robot.disconnect()
```

## 训练策略

要训练策略来控制你的机器人，请使用 [`lerobot-train`](https://github.com/huggingface/lerobot/blob/main/src/lerobot/scripts/lerobot_train.py) 脚本。需要提供一些参数。以下是一个示例命令：

```bash
lerobot-train \
  --dataset.repo_id=${HF_USER}/so101_test \
  --policy.type=act \
  --output_dir=outputs/train/act_so101_test \
  --job_name=act_so101_test \
  --policy.device=cuda \
  --wandb.enable=true \
  --policy.repo_id=${HF_USER}/my_policy
```

下面解释一下这个命令：

1. 我们通过 `--dataset.repo_id=${HF_USER}/so101_test` 参数指定了数据集。
2. 我们通过 `policy.type=act` 指定了策略。这会从 [`configuration_act.py`](https://github.com/huggingface/lerobot/blob/main/src/lerobot/policies/act/configuration_act.py) 加载配置。重要的是，该策略会自动适应已保存在数据集中你的机器人的电机状态数、电机动作数和摄像头数（例如 `laptop` 和 `phone`）。
3. 我们设置了 `policy.device=cuda`，因为我们是在 Nvidia GPU 上训练，但你可以使用 `policy.device=mps` 在 Apple Silicon 上进行训练。
4. 我们设置了 `wandb.enable=true` 以使用 [Weights and Biases](https://docs.wandb.ai/quickstart) 来可视化训练曲线。这是可选的，但如果使用它，请确保通过运行 `wandb login` 登录。

训练需要几个小时。你可以在 `outputs/train/act_so101_test/checkpoints` 中找到检查点。

要从检查点恢复训练，以下是从 `act_so101_test` 策略的 `last` 检查点恢复的示例命令：

```bash
lerobot-train \
  --config_path=outputs/train/act_so101_test/checkpoints/last/pretrained_model/train_config.json \
  --resume=true
```

`--config_path` 也接受 **Hub 仓库 ID**：如果某次运行将其检查点推送到了 Hub（使用 `--save_checkpoint_to_hub=true`），你可以直接从该仓库恢复训练——系统会下载其最新检查点并继续训练，恢复优化器、调度器、步骤计数器和数据顺序：

```bash
lerobot-train --config_path=${HF_USER}/my_policy --resume=true
```

如果不想在训练后将模型推送到 Hub，请使用 `--policy.push_to_hub=false`。

此外，你还可以添加额外的 `tags` 或为模型指定 `license`（许可协议），或者通过添加以下参数使模型仓库变为 `private`（私有）：`--policy.private=true --policy.tags=\[ppo,rl\] --policy.license=mit`

#### 使用 Google Colab 训练

如果你的本地计算机没有强大的 GPU，你可以通过 [ACT 训练笔记本](./notebooks#training-act) 使用 Google Colab 来训练模型。

#### 使用 Hugging Face Jobs 训练

Hugging Face Jobs 让你可以轻松选择硬件并在云端运行训练。因此，如果你没有强大的 GPU，或者需要更多显存，或者只是想更快地训练模型，可以使用 HF Jobs！它是按使用量付费的，你只需为每秒的使用付费，你可以[在此处](https://huggingface.co/docs/hub/jobs)查看定价和其他信息。

`lerobot-train` 默认在本地运行。要在 HuggingFace GPU 上运行，请传入 `--job.target` 并指定硬件规格名称：

```bash
lerobot-train \
  --dataset.repo_id=${HF_USER}/so101_test \
  --policy.type=act \
  --policy.repo_id=${HF_USER}/my_policy \
  --job.target=a10g-small
```

使用 `hf jobs hardware` 查看可用的硬件规格和定价。运行时会将其日志流式传输到你的终端；按 Ctrl-C 可以断开连接（作业会在云端继续运行）。重新连接或取消作业：

```bash
hf jobs logs <job-id>
hf jobs cancel <job-id>
```

如果你的数据集仅存在于本地（尚未上传到 Hub），它会自动被推送到一个**私有** Hub 仓库，以便作业可以通过 `repo_id` 下载它（没有任何内容被公开）。训练完成后，训练好的模型会被推送到模型仓库。如果还想在保存中间检查点时立即将其推送到 Hub（以便在运行过程中监控进度），请添加 `--save_checkpoint_to_hub=true`——这需要运行时镜像包含此功能。

每个作业（以及任务推送的任何数据集）都会被标记为 `lerobot`，以便在 Hub 上轻松找到。使用 `--job.tags '["my-tag"]'` 添加你自己的标签。

默认情况下，作业的墙上时钟时间上限为 `2d`（48 小时）。使用 HF Jobs 的持续时间字符串覆盖它，例如 `--job.timeout=4h` 更快失败，或 `--job.timeout=7d` 进行更长时间运行。

> **注意：** 模型仓库是预先创建的（它保存了作业运行的暂存训练配置）。如果运行在模型推送之前失败，该仓库会留在 Hub 上以便你检查——它不会自动删除，因此重复失败可能会留下空仓库。使用 `hf repo delete <repo-id>` 删除一个。

**前提条件：** 在提交之前运行 `hf auth login`。如需与 Weights & Biases 集成，请在你的机器上运行 `wandb login` 或设置 `WANDB_API_KEY`——密钥会自动转发到作业中。

**在作业上恢复训练。** 在恢复命令中添加 `--job.target` 将在云端执行恢复——相同的命令在本地或远程都有效。检查点仓库是事实来源，新检查点会延续同一仓库中的训练序列：

```bash
# 在作业上恢复 Hub 上的运行（检查点已在 Hub 上）
lerobot-train --config_path=${HF_USER}/my_policy --resume=true --job.target=a10g-small

# 在作业上恢复 LOCAL 运行——检查点首先上传到私有 Hub 仓库，
# 然后作业从中恢复（仅本地的数据集也以相同方式上传）
lerobot-train \
  --config_path=outputs/train/act_so101_test/checkpoints/last/pretrained_model/train_config.json \
  --resume=true \
  --job.target=a10g-small
```

作业设置来自当前命令，因此根据需要覆盖 `--job.target`、`--job.timeout` 等；要使恢复后的运行本身也可恢复，请保持 `--save_checkpoint_to_hub=true`。

#### 上传策略检查点

训练完成后，使用以下命令上传最新的检查点：

```bash
hf upload ${HF_USER}/act_so101_test \
  outputs/train/act_so101_test/checkpoints/last/pretrained_model
```

你也可以上传中间检查点：

```bash
CKPT=010000
hf upload ${HF_USER}/act_so101_test${CKPT} \
  outputs/train/act_so101_test/checkpoints/${CKPT}/pretrained_model
```

## 运行推理并评估策略

使用 `lerobot-rollout` 在机器人上部署训练好的策略。你可以根据需求选择不同的策略（strategy）：

下面的示例从 `--policy.path` 加载模型。要固定某个特定的推送版本——当 `--save_checkpoint_to_hub=true` 已经提交了多个检查点时很有用——添加 `--policy.pretrained_revision` 并指定提交哈希、分支或标签。每个推送的检查点都带有其步骤标签（例如 `--policy.pretrained_revision=010000`），因此你可以按步骤恢复检查点，而无需查找其提交 SHA。

<hfoptions id="eval">
<hfoption id="基础模式（不记录）">
```bash
lerobot-rollout \
  --strategy.type=base \
  --policy.path=${HF_USER}/my_policy \
  --robot.type=so100_follower \
  --robot.port=/dev/ttyACM1 \
  --robot.cameras="{ up: {type: opencv, index_or_path: /dev/video10, width: 640, height: 480, fps: 30}, side: {type: intelrealsense, serial_number_or_name: 233522074606, width: 640, height: 480, fps: 30}}" \
  --task="将乐高积木放入透明盒子" \
  --duration=60
```
</hfoption>
<hfoption id="哨兵模式（带记录）">
```bash
lerobot-rollout \
  --strategy.type=sentry \
  --strategy.upload_every_n_episodes=5 \
  --policy.path=${HF_USER}/my_policy \
  --robot.type=so100_follower \
  --robot.port=/dev/ttyACM1 \
  --robot.cameras="{ up: {type: opencv, index_or_path: /dev/video10, width: 640, height: 480, fps: 30}, side: {type: intelrealsense, serial_number_or_name: 233522074606, width: 640, height: 480, fps: 30}}" \
  --dataset.repo_id=${HF_USER}/eval_so100 \
  --dataset.single_task="将乐高积木放入透明盒子" \
  --duration=600
```
</hfoption>
</hfoptions>

`--strategy.type` 标志选择执行模式：

- `base`：自主部署，不记录数据（适用于快速评估）
- `sentry`：持续记录并自动上传（适用于大规模评估）
- `highlight`：环形缓冲区（ring buffer）记录，通过按键保存（适用于捕获感兴趣的事件）
- `dagger`：人在回路中的（human-in-the-loop）数据收集（参见 [HIL 数据收集](./hil_data_collection)）
- `episodic`：面向片段的策略记录，片段之间包含重置阶段

所有策略都支持 `--inference.type=rtc`，用于在较慢的 VLA 模型（Pi0、Pi0.5、SmolVLA）上实现平滑执行。
