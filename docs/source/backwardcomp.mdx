# 向后兼容性

## 策略归一化迁移（PR #1452）

**破坏性变更**：LeRobot 策略不再将归一化层内嵌到模型权重中。归一化现在由外部的 `PolicyProcessorPipeline` 组件处理。

### 变更内容

|                            | PR #1452 之前                                | PR #1452 之后                                                |
| -------------------------- | -------------------------------------------- | ------------------------------------------------------------ |
| **归一化位置**             | 嵌入在模型权重中（`normalize_inputs.*`）      | 外部 `PolicyProcessorPipeline` 组件                          |
| **模型状态字典**           | 包含归一化统计信息                            | **仅含干净权重** —— 不含归一化参数                            |
| **使用方式**               | `policy(batch)` 处理所有操作                  | `preprocessor(batch)` → `policy(...)` → `postprocessor(...)` |

### 对现有模型的影响

- PR #1452 **之前**训练的模型在其权重中嵌入了归一化
- 这些模型需要进行迁移才能适配新的 `PolicyProcessorPipeline` 系统
- 迁移过程会提取归一化统计信息并创建独立的处理器流水线

### 迁移旧模型

使用迁移脚本转换包含内嵌归一化的模型：

```shell
python src/lerobot/processor/migrate_policy_normalization.py \
    --pretrained-path lerobot/act_aloha_sim_transfer_cube_human \
    --push-to-hub \
    --branch migrated
```

该脚本：

1. **提取** 模型权重中的归一化统计信息
2. **创建** 外部预处理器和后处理器流水线
3. **移除** 模型权重中的归一化层
4. **保存** 干净的模型 + 处理器流水线
5. **推送** 到 Hub 并自动创建 PR

### 使用迁移后的模型

```python
# 迁移后的新使用模式
from lerobot.policies import make_policy, make_pre_post_processors

# 分别加载模型和处理器
policy = make_policy(config, ds_meta=dataset.meta)
preprocessor, postprocessor = make_pre_post_processors(
    policy_cfg=config,
    dataset_stats=dataset.meta.stats
)

# 通过流水线处理数据
processed_batch = preprocessor(raw_batch)
action = policy.select_action(processed_batch)
final_action = postprocessor(action)
```

## 硬件 API 重新设计

PR [#777](https://github.com/huggingface/lerobot/pull/777) 改进了 LeRobot 的标定，但**不向后兼容**。以下是变更内容的概述，以及如何继续使用此 PR 之前创建的数据集。

### 变更内容

|                                   | PR #777 之前                                | PR #777 之后                                                 |
| --------------------------------- | ------------------------------------------- | ------------------------------------------------------------ |
| **关节范围**                      | 度数 `-180...180°`                          | **归一化范围** 关节：`–100...100`，夹爪：`0...100`            |
| **零位（SO100 / SO101）**         | 手臂完全水平伸展                             | **每个关节在其范围中间**                                     |
| **边界处理**                      | 检测 ±180° 环绕的软件保护措施                | 因零位在范围中间，无需环绕逻辑                               |

---

### 对现有数据集的影响

- PR #777 **之前**录制的轨迹如果直接加载，将无法正确回放：
  - 关节角度存在偏移且归一化不正确。
- 任何直接在旧数据上微调或训练的模型都需要转换其输入和输出。

### 使用基于旧标定系统创建的数据集

我们在此提供了一个迁移示例脚本，用于回放使用旧标定录制的片段：`examples/backward_compatibility/replay.py`。
下面我们将逐步说明示例脚本中为使旧标定数据集正常工作所做的修改。

```diff
+   key = f"{name.removeprefix('main_')}.pos"
    action[key] = action_array[i].item()
+   action["shoulder_lift.pos"] = -(action["shoulder_lift.pos"] - 90)
+   action["elbow_flex.pos"] -= 90
```

让我们逐项分析。
新代码库使用 `.pos` 后缀表示位置观测值，并且我们移除了 `main_` 前缀：

<!-- prettier-ignore-start -->
```python
key = f"{name.removeprefix('main_')}.pos"
```
<!-- prettier-ignore-end -->

对于 `"shoulder_lift"`（ID = 2），相较于旧标定/代码，其零位偏移了 -90 度且方向相反。

<!-- prettier-ignore-start -->
```python
action["shoulder_lift.pos"] = -(action["shoulder_lift.pos"] - 90)
```
<!-- prettier-ignore-end -->

对于 `"elbow_flex"`（ID = 3），相较于旧标定/代码，其零位偏移了 -90 度。

<!-- prettier-ignore-start -->
```python
action["elbow_flex.pos"] -= 90
```
<!-- prettier-ignore-end -->

要使用角度制归一化，我们将 `--robot.use_degrees` 选项设置为 `true`。

```diff
python examples/backward_compatibility/replay.py \
    --robot.type=so101_follower \
    --robot.port=/dev/tty.usbmodem5A460814411 \
    --robot.id=blue \
+   --robot.use_degrees=true \
    --dataset.repo_id=my_dataset_id \
    --dataset.episode=0
```

### 使用基于旧标定系统训练的策略

策略输出的动作格式与数据集相同（`torch.Tensors`）。因此，应应用相同的变换。

要找出这些变换，我们建议首先尝试使用上述部分回放策略所训练数据集的一个片段。
然后，将这些相同的变换添加到你的推理脚本中（此处展示在 `record.py` 脚本中）：

```diff
action_values = predict_action(
    observation_frame,
    policy,
    get_safe_torch_device(policy.config.device),
    policy.config.use_amp,
    task=single_task,
    robot_type=robot.robot_type,
    )
    action = {key: action_values[i].item() for i, key in enumerate(robot.action_features)}

+   action["shoulder_lift.pos"] = -(action["shoulder_lift.pos"] - 90)
+   action["elbow_flex.pos"] -= 90
    robot.send_action(action)
```
