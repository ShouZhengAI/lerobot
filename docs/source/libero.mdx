# LIBERO

LIBERO 是一个专门研究**终身机器人学习（lifelong robot learning）**的基准测试平台——其核心理念是机器人不仅需要一次预训练，更需要随着与用户的交互持续学习和适应。它提供了一系列标准化的操作任务，重点关注**知识迁移**：机器人在新环境中能多好地应用已学知识。通过在 LIBERO 上进行评估，不同算法可以公平比较，研究人员也能在彼此的工作基础上继续推进。

- 论文：[Benchmarking Knowledge Transfer for Lifelong Robot Learning](https://arxiv.org/abs/2306.03310)
- GitHub：[Lifelong-Robot-Learning/LIBERO](https://github.com/Lifelong-Robot-Learning/LIBERO)
- 项目网站：[libero-project.github.io](https://libero-project.github.io)

![LIBERO 基准测试概览](https://libero-project.github.io/assets/img/libero/fig1.png)

## 可用任务

LIBERO 包含**五个任务套件**，涵盖**130 个任务**，从简单的物体操作到复杂的多步骤场景：

| 套件           | CLI 名称          | 任务数 | 描述                        |
| -------------- | ----------------- | ------ | --------------------------- |
| LIBERO-Spatial | `libero_spatial`  | 10     | 需要推理空间关系的任务      |
| LIBERO-Object  | `libero_object`   | 10     | 围绕操作不同物体的任务      |
| LIBERO-Goal    | `libero_goal`     | 10     | 目标条件任务，目标会变化    |
| LIBERO-90      | `libero_90`       | 90     | 来自 LIBERO-100 集合的短周期任务 |
| LIBERO-Long    | `libero_10`       | 10     | 来自 LIBERO-100 集合的长周期任务 |

## 安装

在完成 LeRobot 安装说明后：

```bash
pip install -e ".[libero]"
```

<Tip>
LIBERO 需要 Linux 环境（`sys_platform == 'linux'`）。LeRobot 使用 MuJoCo 进行仿真——请在训练或评估前设置渲染后端：

```bash
export MUJOCO_GL=egl  # 适用于无头服务器（HPC、云环境）
```

</Tip>

## 评估

### 默认评估（推荐）

跨四个标准套件进行评估（每个任务 10 个片段）：

```bash
lerobot-eval \
  --policy.path="your-policy-id" \
  --env.type=libero \
  --env.task=libero_spatial,libero_object,libero_goal,libero_10 \
  --eval.batch_size=1 \
  --eval.n_episodes=10 \
  --env.max_parallel_tasks=1
```

### 单套件评估

在单个 LIBERO 套件上评估：

```bash
lerobot-eval \
  --policy.path="your-policy-id" \
  --env.type=libero \
  --env.task=libero_object \
  --eval.batch_size=2 \
  --eval.n_episodes=3
```

- `--env.task` 选择套件（`libero_object`、`libero_spatial` 等）。
- `--env.task_ids` 限制为特定的任务索引（如 `[0]`、`[1,2,3]` 等）。省略则运行套件中的所有任务。
- `--eval.batch_size` 控制并行运行的环境数量。
- `--eval.n_episodes` 设置每个任务运行的片段数。

### 多套件评估

通过逗号分隔的列表一次性在多个套件上评估策略：

```bash
lerobot-eval \
  --policy.path="your-policy-id" \
  --env.type=libero \
  --env.task=libero_object,libero_spatial \
  --eval.batch_size=1 \
  --eval.n_episodes=2
```

### 控制模式

LIBERO 支持两种控制模式——`relative`（相对，默认）和 `absolute`（绝对）。不同的 VLA 检查点使用不同的动作参数化方式，因此请确保模式与你的策略匹配：

```bash
--env.control_mode=relative   # 或 "absolute"
```

### 策略输入和输出

**观测信息（Observations）：**

- `observation.state` — 8 维本体感受特征（末端执行器位置、轴角方向、夹爪位置）
- `observation.images.image` — 主摄像头视图（`agentview_image`），HWC uint8 格式
- `observation.images.image2` — 腕部摄像头视图（`robot0_eye_in_hand_image`），HWC uint8 格式

<Tip warning={true}>
  LeRobot 对视觉特征强制使用 `.images.*` 前缀。确保策略配置中的 `input_features` 使用相同的命名键名，并且数据集的元数据键名遵循此约定。如果你的数据包含不同的键名，则必须重命名观测信息以匹配策略的预期，因为命名键名已编码在归一化统计层中。
</Tip>

**动作（Actions）：**

- `Box(-1, 1, shape=(7,))` 中的连续控制——6 维末端执行器增量 + 1 维夹爪

### 推荐评估片段数

为了可重复的基准测试，建议在全部四个标准套件（Spatial、Object、Goal、Long）中每个任务运行 **10 个片段**。总计 400 个片段，与已发表结果使用的协议一致。

## 训练

### 数据集

我们提供了与 LeRobot 完全兼容的预处理 LIBERO 数据集：

- [HuggingFaceVLA/libero](https://huggingface.co/datasets/HuggingFaceVLA/libero)

作为参考，Physical Intelligence 发布的原始数据集：

- [physical-intelligence/libero](https://huggingface.co/datasets/physical-intelligence/libero)

### 训练命令示例

```bash
lerobot-train \
  --policy.type=smolvla \
  --policy.repo_id=${HF_USER}/libero-test \
  --policy.load_vlm_weights=true \
  --dataset.repo_id=HuggingFaceVLA/libero \
  --env.type=libero \
  --env.task=libero_10 \
  --output_dir=./outputs/ \
  --steps=100000 \
  --batch_size=4 \
  --eval.batch_size=1 \
  --eval.n_episodes=1 \
  --env_eval_freq=1000
```

## 复现已发表的结果

我们在 LIBERO 基准测试上复现了 Pi0.5 的结果。我们使用 Physical Intelligence 的 LIBERO 基础模型（`pi05_libero`），在 bfloat16 精度下额外微调 6000 步，批大小为 256，使用 8 张 H100 GPU 和 [HuggingFace LIBERO 数据集](https://huggingface.co/datasets/HuggingFaceVLA/libero)。

微调后的模型：[lerobot/pi05_libero_finetuned](https://huggingface.co/lerobot/pi05_libero_finetuned)

### 评估命令

```bash
lerobot-eval \
  --output_dir=./eval_logs/ \
  --env.type=libero \
  --env.task=libero_spatial,libero_object,libero_goal,libero_10 \
  --eval.batch_size=1 \
  --eval.n_episodes=10 \
  --policy.path=pi05_libero_finetuned \
  --policy.n_action_steps=10 \
  --env.max_parallel_tasks=1
```

我们将 `n_action_steps` 设为 10，与原始 OpenPI 实现一致。

### 结果

| 模型                | LIBERO Spatial | LIBERO Object | LIBERO Goal | LIBERO 10 | 平均值   |
| ------------------- | -------------- | ------------- | ----------- | --------- | -------- |
| **Pi0.5 (LeRobot)** | 97.0           | 99.0          | 98.0        | 96.0      | **97.5** |

这些结果与 Physical Intelligence 报告的[原始结果](https://github.com/Physical-Intelligence/openpi/tree/main/examples/libero#results)一致：

| 模型               | LIBERO Spatial | LIBERO Object | LIBERO Goal | LIBERO 10 | 平均值    |
| ------------------ | -------------- | ------------- | ----------- | --------- | --------- |
| **Pi0.5 (OpenPI)** | 98.8           | 98.2          | 98.0        | 92.4      | **96.85** |
