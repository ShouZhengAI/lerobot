# 摄像头

LeRobot 提供多种视频采集选项：

| 类（Class）       | 支持的摄像头                       |
| ----------------- | ----------------------------------- |
| `OpenCVCamera`    | 手机、笔记本内置摄像头、USB 网络摄像头 |
| `ZMQCamera`       | 网络连接的摄像头                     |
| `RealSenseCamera` | Intel RealSense（含深度）           |
| `Reachy2Camera`   | Reachy 2 机器人摄像头               |

> [!TIP]
> 有关 `OpenCVCamera` 兼容性的详细信息，请参阅 [Video I/O with OpenCV 概述](https://docs.opencv.org/4.x/d0/da7/videoio_overview.html)。

### 查找你的摄像头

每个摄像头都需要一个唯一标识符来实例化，以便区分多个连接的设备。

`OpenCVCamera` 和 `RealSenseCamera` 支持自动发现。运行以下命令列出可用设备及其标识符。请注意，根据你的操作系统，重启计算机或重新插拔摄像头后，这些标识符可能会发生变化。

```bash
lerobot-find-cameras opencv # 或者使用 realsense 以查找 Intel Realsense 摄像头
```

如果你连接了两个摄像头，输出将类似于：

```bash
--- 检测到的摄像头 ---
摄像头 #0:
  名称: OpenCV Camera @ 0
  类型: OpenCV
  ID: 0
  后端 API: AVFOUNDATION
  默认流配置:
    格式: 16.0
    宽度: 1920
    高度: 1080
    帧率: 15.0
--------------------
(更多摄像头...)
```

> [!WARNING]
> 在 `macOS` 上使用 Intel RealSense 摄像头时，可能会遇到此[错误](https://github.com/IntelRealSense/librealsense/issues/12307)：`Error finding RealSense cameras: failed to set power state`，可以通过使用 `sudo` 权限运行相同的命令来解决。请注意，在 `macOS` 上使用 RealSense 摄像头不稳定。

`ZMQCamera` 和 `Reachy2Camera` 不支持自动发现。必须通过提供它们的网络地址和端口或机器人 SDK 设置来手动配置。

## 使用摄像头

### 帧访问模式

所有摄像头类都实现了三种帧采集访问模式：

| 方法                      | 行为                                                                                                                               | 是否阻塞？    | 最适合用于                                 |
| ------------------------- | ---------------------------------------------------------------------------------------------------------------------------------- | ------------- | ------------------------------------------ |
| `read()`                  | 等待摄像头硬件返回一帧。根据摄像头和 SDK 不同，可能长时间阻塞。                                                                    | 是            | 简单脚本，顺序采集                         |
| `async_read(timeout_ms)`  | 从后台线程返回最新的未消费帧。仅在缓冲区为空时阻塞，最长阻塞 `timeout_ms`。如果没有帧到达则抛出 `TimeoutError`。                  | 有超时        | 与摄像头帧率同步的控制循环                 |
| `read_latest(max_age_ms)` | 查看缓冲区中的最新帧（可能有延迟）。如果帧的年龄超过 `max_age_ms` 则抛出 `TimeoutError`。                                         | 否            | UI 可视化，日志记录，监控                  |

### 使用示例

以下示例展示如何使用摄像头 API 配置和采集不同类型摄像头的帧。

- **阻塞和非阻塞帧采集** —— 使用基于 OpenCV 的摄像头
- **彩色和深度采集** —— 使用 Intel RealSense 摄像头

> [!WARNING]
> 未能正确断开摄像头连接可能导致资源泄漏。请使用上下文管理器协议确保自动清理：
>
> ```python
> with OpenCVCamera(config) as camera:
>     ...
> ```
>
> 你也可以手动调用 `connect()` 和 `disconnect()`，但对于后者请始终使用 `finally` 块。

<hfoption id="Open CV 摄像头">

<!-- prettier-ignore-start -->
```python
from lerobot.cameras.opencv import OpenCVCamera, OpenCVCameraConfig
from lerobot.cameras import ColorMode, Cv2Rotation

# 构建一个 `OpenCVCameraConfig`，设置所需的帧率、分辨率、色彩模式和旋转方向。
config = OpenCVCameraConfig(
    index_or_path=0,
    fps=15,
    width=1920,
    height=1080,
    color_mode=ColorMode.RGB,
    rotation=Cv2Rotation.NO_ROTATION
)

# 实例化并连接一个 `OpenCVCamera`，执行预热读取（默认行为）。
with OpenCVCamera(config) as camera:

    # 同步读取一帧 —— 阻塞直到硬件交付新帧
    frame = camera.read()
    print(f"read() 调用返回的帧形状:", frame.shape)

    # 带超时的异步读取 —— 返回最新的未消费帧，或等待 timeout_ms 等待新帧
    try:
        for i in range(10):
            frame = camera.async_read(timeout_ms=200)
            print(f"async_read 调用返回帧 {i}，形状:", frame.shape)
    except TimeoutError as e:
        print(f"超时内未收到帧: {e}")

    # 即时返回一帧 —— 返回摄像头捕获的最新帧
    try:
        initial_frame = camera.read_latest(max_age_ms=1000)
        for i in range(10):
            frame = camera.read_latest(max_age_ms=1000)
            print(f"read_latest 调用返回帧 {i}，形状:", frame.shape)
            print(f"摄像头是否有新帧到达？{not (initial_frame == frame).any()}")
    except TimeoutError as e:
        print(f"帧过旧: {e}")

```

```python
from lerobot.cameras.realsense import RealSenseCamera, RealSenseCameraConfig
from lerobot.cameras import ColorMode, Cv2Rotation

# 创建一个 `RealSenseCameraConfig`，指定摄像头的序列号并启用深度。
config = RealSenseCameraConfig(
    serial_number_or_name="233522074606",
    fps=15,
    width=640,
    height=480,
    color_mode=ColorMode.RGB,
    use_depth=True,
    rotation=Cv2Rotation.NO_ROTATION
)

# 实例化并连接一个 `RealSenseCamera`，带预热读取（默认行为）。
camera = RealSenseCamera(config)
camera.connect()

# 通过 `read()` 采集彩色帧，通过 `read_depth()` 采集深度图。
try:
    color_frame = camera.read()
    depth_map = camera.read_depth()
    print("彩色帧形状:", color_frame.shape)
    print("深度图形状:", depth_map.shape)
finally:
    camera.disconnect()
```

### 使用深度数据

Intel RealSense 和 Reachy 2 摄像头可以同步采集彩色和深度数据。调用 `read()` 返回形状为 `(H, W, 3)` 的 `uint8` **彩色**帧。调用 `read_depth()` 返回形状为 `(H, W, 1)` 的 `uint16` **深度图**，其中每个像素值表示传感器到物体的距离，以**毫米**为单位。像素值为 `0` 通常表示"无可用测量值"（超出范围、被遮挡或低置信度）。

在录制过程中，控制循环通过 `read_latest()`（彩色）和 `read_latest_depth()`（深度）以非阻塞方式获取缓冲区中最新的帧，将深度图作为同级特征添加（例如，`front` 旁边的 `front_depth`）。

关于深度流在录制数据集时如何存储和编码，请参阅视频编码指南中的[深度流](./video_encoding_parameters#深度流)部分。

## 使用手机摄像头

<hfoptions id="use phone">
<hfoption id="iPhone 与 macOS">

要在 macOS 上将 iPhone 用作摄像头，请启用连续互通相机（Continuity Camera）功能：

- 确保你的 Mac 运行 macOS 13 或更高版本，并且你的 iPhone 运行 iOS 16 或更高版本。
- 在两台设备上使用相同的 Apple ID 登录。
- 使用 USB 线缆连接设备，或开启 Wi-Fi 和蓝牙进行无线连接。

更多详情，请访问 [Apple 支持页面](https://support.apple.com/en-gb/guide/mac-help/mchl77879b8a/mac)。

</hfoption>
<hfoption id="OBS 虚拟摄像头">

如果你想使用 OBS 将手机用作摄像头，请按照以下步骤设置虚拟摄像头。

1. _（仅 Linux）安装 `v4l2loopback-dkms` 和 `v4l-utils`_。这些包用于创建虚拟摄像头设备并验证其设置。使用以下命令安装：

```bash
sudo apt install v4l2loopback-dkms v4l-utils
```

2. _在你的手机上安装 [DroidCam 应用](https://droidcam.app)_。该应用适用于 iOS 和 Android。
3. _下载并安装 [OBS Studio](https://obsproject.com)_。
4. _下载并安装 [DroidCam OBS 插件](https://droidcam.app/obs)_。
5. _启动 OBS Studio_。

6. _将你的手机添加为源_。按照[此处](https://droidcam.app/obs/usage)的说明操作。确保将分辨率设置为 `640x480` 以避免水印。
7. _调整分辨率设置_。在 OBS Studio 中，进入 `File > Settings > Video` 或 `OBS > Preferences... > Video`。手动输入将 `Base(Canvas) Resolution`（基础/画布分辨率）和 `Output(Scaled) Resolution`（输出/缩放分辨率）都改为 `640x480`。
8. _启动虚拟摄像头_。在 OBS Studio 中，按照[此处](https://obsproject.com/kb/virtual-camera-guide)的说明操作。
9. _验证虚拟摄像头设置和分辨率_。
   - **Linux**：使用 `v4l2-ctl` 列出设备并检查分辨率：
     ```bash
     v4l2-ctl --list-devices  # 找到 VirtualCam 并记下其 /dev/videoX 路径
     v4l2-ctl -d /dev/videoX --get-fmt-video  # 替换为你的 VirtualCam 路径
     ```
     你应该看到 `VirtualCam` 列出且分辨率为 `640x480`。
   - **macOS**：打开 Photo Booth 或 FaceTime，选择"OBS Virtual Camera"作为输入。
   - **Windows**：原生相机应用不支持虚拟摄像头。请使用视频会议应用（Zoom、Teams）或直接运行 `lerobot-find-cameras opencv` 进行验证。

<details>
<summary><strong>故障排除</strong></summary>

> 虚拟摄像头分辨率不正确。

删除虚拟摄像头源并重新创建。分辨率在创建后无法更改。

> 在 OpenCVCamera(X) 的后台线程中读取帧时出错：OpenCVCamera(X) 帧宽度=640 或高度=480 与配置的宽度=1920 或高度=1080 不匹配。

此错误是由 OBS 虚拟摄像头在重新缩放后仍然声明 `1920x1080` 分辨率导致的。目前的唯一解决方法是注释掉 `_postprocess_image()` 中的宽度和高度检查。

</details>
