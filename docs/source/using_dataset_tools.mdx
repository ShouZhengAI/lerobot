# 使用数据集工具

本指南介绍 LeRobot 中用于修改和编辑现有数据集的数据集工具实用程序。

## 概述

LeRobot 提供了多种操作数据集的实用工具：

1. **删除片段（Delete Episodes）** - 从数据集中删除特定片段
2. **拆分数据集（Split Dataset）** - 将数据集拆分为多个较小的数据集
3. **合并数据集（Merge Datasets）** - 将多个数据集合并为一个。数据集必须具有相同的特征，片段将按 `repo_ids` 中指定的顺序拼接
4. **添加特征（Add Features）** - 向数据集添加新特征
5. **移除特征（Remove Features）** - 从数据集中移除特征
6. **转换为视频（Convert to Video）** - 将基于图像的数据集转换为视频格式以实现高效存储（RGB 和深度相机使用独立的编码器进行编码）
7. **重新编码视频（Re-encode Videos）** - 使用新的编码器设置重新编码现有视频数据集的 RGB 和/或深度流
8. **显示数据集信息（Show the Info of Datasets）** - 显示数据集信息的摘要，例如片段数量等

核心实现位于 `lerobot.datasets.dataset_tools`。
有关如何使用工具 API 的示例脚本，请参见 `examples/dataset/use_dataset_tools.py`。

## 命令行工具：lerobot-edit-dataset

`lerobot-edit-dataset` 是一个用于编辑数据集的命令行脚本。可用于删除片段、拆分数据集、合并数据集、添加特征、移除特征以及将图像数据集转换为视频格式。

运行 `lerobot-edit-dataset --help` 获取有关每个操作配置的更多信息。

### 使用示例

#### 删除片段

从数据集中删除特定片段。这有助于过滤掉不需要的数据。

```bash
# 删除第 0、2 和 5 个片段（修改原始数据集）
lerobot-edit-dataset \
    --repo_id lerobot/pusht \
    --operation.type delete_episodes \
    --operation.episode_indices "[0, 2, 5]"

# 删除片段并保存到新数据集（保留原始数据集）
lerobot-edit-dataset \
    --repo_id lerobot/pusht \
    --new_repo_id lerobot/pusht_after_deletion \
    --operation.type delete_episodes \
    --operation.episode_indices "[0, 2, 5]"
```

#### 拆分数据集

将一个数据集拆分为多个子集。

```bash
# 按比例拆分（例如 80% 训练集、20% 测试集、20% 验证集）
lerobot-edit-dataset \
    --repo_id lerobot/pusht \
    --operation.type split \
    --operation.splits '{"train": 0.8, "test": 0.2, "val": 0.2}'

# 按特定片段索引拆分
lerobot-edit-dataset \
    --repo_id lerobot/pusht \
    --operation.type split \
    --operation.splits '{"task1": [0, 1, 2, 3], "task2": [4, 5]}'
```

拆分名称没有限制，可由用户自行决定。生成的数据集将以仓库 ID 附加拆分名称的方式保存，例如 `lerobot/pusht_train`、`lerobot/pusht_task1`、`lerobot/pusht_task2`。

#### 合并数据集

将多个数据集合并为一个数据集。

```bash
# 将训练集和验证集拆分合并回一个数据集
lerobot-edit-dataset \
    --repo_id lerobot/pusht_merged \
    --operation.type merge \
    --operation.repo_ids "['lerobot/pusht_train', 'lerobot/pusht_val']"
```

#### 移除特征

从数据集中移除特征。

```bash
# 移除一个相机特征
lerobot-edit-dataset \
    --repo_id lerobot/pusht \
    --operation.type remove_feature \
    --operation.feature_names "['observation.images.top']"
```

#### 转换为视频

将基于图像的数据集转换为视频格式，创建一个新的 LeRobotDataset，其中图像以视频形式存储。这有助于减少存储需求并提高数据加载性能。新数据集将具有与原始数据集完全相同的结构，但图像将编码为符合 LeRobot 格式的 MP4 视频。

```bash
# 仅本地：保存到自定义输出目录（不推送到 Hub）
lerobot-edit-dataset \
    --repo_id lerobot/pusht_image \
    --operation.type convert_image_to_video \
    --operation.output_dir /path/to/output/pusht_video

# 使用新的 repo_id 保存（本地存储）
lerobot-edit-dataset \
    --repo_id lerobot/pusht_image \
    --new_repo_id lerobot/pusht_video \
    --operation.type convert_image_to_video

# 转换并推送到 Hugging Face Hub
lerobot-edit-dataset \
    --repo_id lerobot/pusht_image \
    --new_repo_id lerobot/pusht_video \
    --operation.type convert_image_to_video \
    --push_to_hub true

# 使用自定义视频编解码器和质量设置进行转换
lerobot-edit-dataset \
    --repo_id lerobot/pusht_image \
    --operation.type convert_image_to_video \
    --operation.output_dir outputs/pusht_video \
    --operation.rgb_encoder.vcodec libsvtav1 \
    --operation.rgb_encoder.pix_fmt yuv420p \
    --operation.rgb_encoder.g 2 \
    --operation.rgb_encoder.crf 30

# 转换包含深度图的数据集，自定义深度编码器
lerobot-edit-dataset \
    --repo_id lerobot/pusht_image \
    --operation.type convert_image_to_video \
    --operation.output_dir outputs/pusht_video \
    --operation.depth_encoder.depth_min 0.01 \
    --operation.depth_encoder.depth_max 10.0 \
    --operation.depth_encoder.use_log true

# 仅转换特定片段
lerobot-edit-dataset \
    --repo_id lerobot/pusht_image \
    --operation.type convert_image_to_video \
    --operation.output_dir outputs/pusht_video \
    --operation.episode_indices "[0, 1, 2, 5, 10]"

# 使用多个工作进程进行并行处理
lerobot-edit-dataset \
    --repo_id lerobot/pusht_image \
    --operation.type convert_image_to_video \
    --operation.output_dir outputs/pusht_video \
    --operation.num_workers 8

# 对于内存受限的系统，用户可以指定限制：
lerobot-edit-dataset \
    --repo_id lerobot/pusht_image \
    --operation.type convert_to_video \
    --operation.max_episodes_per_batch 50 \
    --operation.max_frames_per_batch 10000
```

**参数：**

- `output_dir`：自定义输出目录（可选——默认使用 `new_repo_id` 或 `{repo_id}_video`）
- `rgb_encoder`：应用于 RGB 相机的视频编码器设置——所有子字段可通过 `--operation.rgb_encoder.<field>` 访问。更多详情请参见[视频编码参数](./video_encoding_parameters)。
- `depth_encoder`：应用于深度图相机（例如来自 Intel RealSense）的视频编码器设置。除了标准编码器字段外，还提供深度量化参数（`depth_min`、`depth_max`、`shift`、`use_log`），可通过 `--operation.depth_encoder.<field>` 访问。这些量化设置会持久化到数据集元数据中，以便在加载时可以将深度反量化回物理单位。详情请参见[深度流](./video_encoding_parameters#depth-streams)部分。
- `episode_indices`：要转换的特定片段列表（默认：所有片段）
- `num_workers`：并行处理的工作进程数量（默认：4）

**注意：** 生成的数据集将是标准的 LeRobotDataset，所有相机都以视频形式编码在 `videos/` 目录中，parquet 文件仅包含元数据（不含原始图像数据）。深度图相机将自动检测并路由到 `depth_encoder`，而 RGB 相机使用 `rgb_encoder`。所有片段、统计数据和任务均保留。

#### 重新编码视频

使用不同的编码器设置重新编码现有视频数据集的视频，无需回溯到原始帧。RGB 视频使用 `rgb_encoder`，深度视频使用 `depth_encoder`。仅提供你希望重新编码的编码器；另一种流类型保持不变。

```bash
# 使用新设置重新编码所有 RGB 视频（默认保存到 lerobot/pusht_reencoded）
lerobot-edit-dataset \
    --repo_id lerobot/pusht \
    --operation.type reencode_videos \
    --operation.rgb_encoder.vcodec h264 \
    --operation.rgb_encoder.pix_fmt yuv420p \
    --operation.rgb_encoder.crf 23

# 在包含深度图的数据集中同时重新编码 RGB 和深度视频
lerobot-edit-dataset \
    --repo_id lerobot/pusht_depth \
    --operation.type reencode_videos \
    --operation.rgb_encoder.vcodec h264 \
    --operation.depth_encoder.crf 50
```

**参数：**

- `rgb_encoder`：应用于每个 RGB 视频的编码器设置。省略则跳过重新编码 RGB 视频。
- `depth_encoder`：应用于每个深度视频的编码器设置。省略则跳过重新编码深度视频。
- `num_workers`：并行处理的工作进程数量。

> [!NOTE]
> 重新编码深度视频时，现有的深度量化参数（`depth_min`、`depth_max`、`shift`、`use_log`）和 `is_depth_map` 标志将**保留**——重新编码仅更改所存储流的编解码器/质量，不会改变加载时深度反量化的方式。

### 显示数据集信息

显示数据集的信息，如片段数量、帧数、文件大小等。
不会对数据集进行任何更改。

```bash

# 显示数据集信息，不包含特征详情
lerobot-edit-dataset \
    --repo_id lerobot/pusht_image \
    --operation.type info \

# 显示数据集信息，包含特征详情
lerobot-edit-dataset \
    --repo_id lerobot/pusht_image \
    --operation.type info \
    --operation.show_features true

```

**参数：**

- `parameters`：控制是否显示包含特征详情的数据集信息的标志（默认：false）

### 推送到 Hub

向任何命令添加 `--push_to_hub true` 标志，即可自动将生成的数据集上传到 Hugging Face Hub：

```bash
lerobot-edit-dataset \
    --repo_id lerobot/pusht \
    --new_repo_id lerobot/pusht_after_deletion \
    --operation.type delete_episodes \
    --operation.episode_indices "[0, 2, 5]" \
    --push_to_hub true
```

还有一个用于向数据集添加特征的工具，`lerobot-edit-dataset` 尚未涵盖该功能。

# 数据集可视化

## 在线可视化

当你使用 `lerobot` 录制数据集时，除非另有指定，否则它会自动上传到 Hugging Face Hub。要在线查看数据集，请使用我们的 **LeRobot Dataset Visualizer**，访问地址：
https://huggingface.co/spaces/lerobot/visualize_dataset

## 本地可视化

你也可以使用我们的命令行工具在本地可视化数据集中的片段。

**从 Hugging Face Hub：**

```bash
lerobot-dataset-viz \
    --repo-id lerobot/pusht \
    --episode-index 0
```

**从本地文件夹：**
添加 `--root` 选项并设置 `--mode local`。例如，要在 `./my_local_data_dir/lerobot/pusht` 中搜索：

```bash
lerobot-dataset-viz \
    --repo-id lerobot/pusht \
    --root ./my_local_data_dir \
    --mode local \
    --episode-index 0
```

执行后，该工具将打开 `rerun.io` 并显示所选片段的相机流、机器人状态和动作。

如果要使用 [Foxglove](https://foxglove.dev) 替代 Rerun，请安装额外扩展并添加 `--display-mode foxglove`。这将启动一个 WebSocket 服务器（将 Foxglove 应用连接到 `ws://127.0.0.1:8765`），以可查找的时间轴方式提供片段，你可以播放、暂停和拖动浏览。

如需了解高级用法——包括可视化存储在远程服务器上的数据集——请运行：

```bash
lerobot-dataset-viz --help
```
