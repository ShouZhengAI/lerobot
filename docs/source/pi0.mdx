# π₀ (Pi0)

π₀ 是一个来自 Physical Intelligence 的**视觉-语言-动作（Vision-Language-Action）模型，用于通用机器人控制**。LeRobot 的实现改编自其开源仓库 [OpenPI](https://github.com/Physical-Intelligence/openpi)。

## 模型概述

π₀ 代表了机器人领域的一项突破，是由 [Physical Intelligence](https://www.physicalintelligence.company/blog/pi0) 开发的第一个通用机器人基础模型（foundation model）。与专为重复运动编程的狭窄专业化传统机器人程序不同，π₀ 被设计为一个通用策略（generalist policy），能够理解视觉输入、解读自然语言指令，并在多种不同机器人上控制各种任务。

### 实体智能的愿景

正如 Physical Intelligence 所描述的那样，虽然人工智能在数字领域取得了显著成功——从下棋到药物发现——但人类智能在物理世界中仍然远远领先于 AI。借用 Moravec 悖论来说，下棋对 AI 来说是一个"简单"问题，而折叠衬衫或清理桌面则需要解决一些有史以来最困难的工程问题。π₀ 是开发人工实体智能（artificial physical intelligence）的第一步，使用户可以像使用大型语言模型一样，只需让机器人执行任意任务。

### 架构与方法

π₀ 结合了多项关键创新：

- **流匹配（Flow Matching）**：使用一种新颖的方法，通过流匹配（扩散模型的变体）为预训练的视觉语言模型（VLM）增强连续动作输出能力
- **跨形态训练（Cross-Embodiment Training）**：在来自 8 种不同机器人平台的数据上进行训练，包括 UR5e、双臂 UR5e、Franka、双臂 Trossen、双臂 ARX、移动 Trossen 和移动 Fibocom
- **互联网规模预训练**：继承自一个预训练的 3B 参数视觉语言模型（VLM）的语义知识
- **高频控制**：以最高 50Hz 的频率输出电机指令，实现实时灵巧操作

## 安装要求

1. 按照[安装指南](./installation)安装 LeRobot。
2. 安装 Pi0 依赖项，运行：

   ```bash
   pip install -e ".[pi]"
   ```

## 训练数据与能力

π₀ 在迄今为止最大的机器人交互数据集上进行训练，结合了三个关键数据源：

1. **互联网规模预训练**：来自网络的视觉语言数据，用于语义理解
2. **Open X-Embodiment 数据集**：开源机器人操作数据集
3. **Physical Intelligence 数据集**：涵盖 8 种不同机器人的大规模多样化灵巧任务数据集

## 使用方法

要在 LeRobot 中使用 π₀，将策略类型指定为：

```python
policy.type=pi0
```

## 训练

对于训练 π₀，可以使用标准的 LeRobot 训练脚本及适当的配置：

```bash
lerobot-train \
    --dataset.repo_id=your_dataset \
    --policy.type=pi0 \
    --output_dir=./outputs/pi0_training \
    --job_name=pi0_training \
    --policy.pretrained_path=lerobot/pi0_base \
    --policy.repo_id=your_repo_id \
    --policy.compile_model=true \
    --policy.gradient_checkpointing=true \
    --policy.dtype=bfloat16 \
    --policy.freeze_vision_encoder=false \
    --policy.train_expert_only=false \
    --steps=3000 \
    --policy.device=cuda \
    --batch_size=32
```

### 关键训练参数

- **`--policy.compile_model=true`**：启用模型编译以加快训练速度
- **`--policy.gradient_checkpointing=true`**：在训练中显著减少显存使用
- **`--policy.dtype=bfloat16`**：使用混合精度训练以提高效率
- **`--batch_size=32`**：训练的批量大小，根据你的 GPU 显存调整
- **`--policy.pretrained_path=lerobot/pi0_base`**：要进行微调的 π₀ 基础模型，可选参数：
  - [lerobot/pi0_base](https://huggingface.co/lerobot/pi0_base)
  - [lerobot/pi0_libero](https://huggingface.co/lerobot/pi0_libero)（针对 Libero 数据集专门训练）

### 训练参数说明

| 参数                    | 默认值    | 说明                             |
| ----------------------- | ------- | -------------------------------- |
| `freeze_vision_encoder` | `false` | 不冻结视觉编码器（vision encoder） |
| `train_expert_only`     | `false` | 不冻结 VLM，训练所有参数           |

**💡 提示**：设置 `train_expert_only=true` 会冻结 VLM，仅训练动作专家（action expert）和投影层，从而以更低的显存消耗进行微调。

## 相对动作

默认情况下，π₀ 预测绝对动作。你可以启用**相对动作**，使模型预测相对于当前机器人状态的偏移量。这可以提高某些配置下的训练稳定性。

要使用相对动作，首先通过 CLI 在相对空间下重新计算数据集统计信息：

```bash
lerobot-edit-dataset \
    --repo_id your_dataset \
    --operation.type recompute_stats \
    --operation.relative_action true \
    --operation.chunk_size 50 \
    --operation.relative_exclude_joints "['gripper']" \
    --push_to_hub true
```

或者在 Python 中等价实现：

```python
from lerobot.datasets import LeRobotDataset, recompute_stats

dataset = LeRobotDataset("your_dataset")
recompute_stats(dataset, relative_action=True, chunk_size=50, relative_exclude_joints=["gripper"])
dataset.push_to_hub()
```

`chunk_size` 应与你的策略的 `chunk_size` 一致（π₀ 默认为 50）。`relative_exclude_joints` 列出应保持为绝对空间的关节名称（例如夹爪指令）。使用 `--push_to_hub true` 将更新后的统计信息上传到 Hub。

然后启用相对动作进行训练：

```bash
lerobot-train \
    --dataset.repo_id=your_dataset \
    --policy.type=pi0 \
    --policy.use_relative_actions=true \
    --policy.relative_exclude_joints='["gripper"]' \
    ...
```
