## 设置开发环境

1. 按照我们的[安装指南](./installation)安装 LeRobot。
2. 运行以下命令安装 SmolVLA 依赖：

   ```bash
   pip install -e ".[smolvla]"
   ```

## 收集数据集

SmolVLA 是一个基础模型，因此需要在自己的数据上进行微调才能在你的设置中获得最佳性能。
我们建议开始时为你的任务录制约 50 个回合。请按照我们的指南开始操作：[录制数据集](./il_robots)

<Tip>

在你的数据集中，确保为每个你引入的变化（例如，如果是方块抓取放置任务，则确保方块在桌面上的不同位置）提供足够的演示。

我们建议查看下面链接的数据集作为参考，该数据集已在 [SmolVLA 论文](https://huggingface.co/papers/2506.01844)中使用：

🔗 [SVLA SO100 PickPlace](https://huggingface.co/spaces/lerobot/visualize_dataset?path=%2Flerobot%2Fsvla_so100_pickplace%2Fepisode_0)

在该数据集中，我们在 5 个不同的方块位置录制了 50 个回合。对于每个位置，我们收集了 10 个回合的抓取放置交互。这种对每个变化重复多次的结构帮助模型更好地泛化。我们尝试了只有 25 个回合的类似数据集，结果不够理想，导致性能不佳。因此，数据的质量和数量确实是一个关键因素。
当你将数据集上传到 Hub 后，就可以使用我们的微调脚本将 SmolVLA 适配到你的应用中。

</Tip>

## 在自有数据上微调 SmolVLA

使用我们预训练的 450M 模型 [`smolvla_base`](https://hf.co/lerobot/smolvla_base)，在你的数据上进行微调。
在单个 A100 GPU 上训练模型 20000 步大约需要 4 小时。你应该根据性能和具体用例来调整步数。

如果你没有 GPU 设备，可以使用我们的 Colab 笔记本进行训练：[![Google Colab](https://colab.research.google.com/assets/colab-badge.svg)](https://colab.research.google.com/github/huggingface/notebooks/blob/main/lerobot/training-smolvla.ipynb)

使用 `--dataset.repo_id` 将你的数据集传递给训练脚本。如果你想测试安装是否正常，可以运行以下命令，其中使用了我们为 [SmolVLA 论文](https://huggingface.co/papers/2506.01844)收集的一个数据集。

```bash
cd lerobot && lerobot-train \
  --policy.path=lerobot/smolvla_base \
  --dataset.repo_id=${HF_USER}/mydataset \
  --batch_size=64 \
  --steps=20000 \
  --output_dir=outputs/train/my_smolvla \
  --job_name=my_smolvla_training \
  --policy.device=cuda \
  --wandb.enable=true
```

<Tip>
  你可以从较小的批次大小开始，如果 GPU 允许且加载时间保持在
  可接受范围内，可以逐步增大。
</Tip>

微调是一门艺术。要了解微调选项的完整概览，请运行

```bash
lerobot-train --help
```

## 评估微调后的模型并实时运行

与录制回合时类似，建议你先登录 HuggingFace Hub。你可以按照相应步骤操作：[录制数据集](./il_robots)。
登录后，可通过以下命令在你的设备上运行推理：

```bash
lerobot-rollout \
  --strategy.type=base \
  --robot.type=so101_follower \
  --robot.port=/dev/ttyACM0 \ # <- 使用你的端口
  --robot.id=my_blue_follower_arm \ # <- 使用你的机器人 ID
  --robot.cameras="{ front: {type: opencv, index_or_path: 8, width: 640, height: 480, fps: 30}}" \ # <- 使用你的相机
  --task="Grasp a lego block and put it in the bin." \ # <- 使用与数据集中相同的任务描述
  # <- RTC 可选，在低功耗硬件上运行时使用 \
  # --inference.type=rtc \
  # --inference.rtc.execution_horizon=10 \
  # --inference.rtc.max_guidance_weight=10.0 \
  # <- 遥操作可选，如果你想在回合之间进行远程操作 \
  # --teleop.type=so100_leader \
  # --teleop.port=/dev/ttyACM0 \
  # --teleop.id=my_red_leader_arm \
  # --display_data=true # 可选，如果你想查看相机流 \
  --policy.path=HF_USER/FINETUNE_MODEL_NAME # <- 使用你微调后的模型
```

根据你的评估设置，你可以配置评估套件的持续时间和要录制的回合数。
