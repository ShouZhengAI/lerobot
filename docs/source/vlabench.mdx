# VLABench

[VLABench](https://github.com/OpenMOSS/VLABench) 是一个大规模基准测试平台，用于**基于语言条件的机器人操控与长时程推理**。该上游套件涵盖 100 个任务类别，涉及 2000 多个物体，评估机器人智能的六个维度：网格与纹理理解、空间推理、世界知识迁移、语义指令理解、物理定律理解和长时程规划。基于 MuJoCo / dm_control 构建，使用 Franka Panda 7 自由度机械臂。LeRobot 通过 `--env.task` 提供了其中的 **43 个任务**（21 个基础任务 + 22 个复合任务，见下方的[可用任务](#可用任务)）。

- 论文：[VLABench: A Large-Scale Benchmark for Language-Conditioned Robotics Manipulation with Long-Horizon Reasoning](https://arxiv.org/abs/2412.18194)
- GitHub：[OpenMOSS/VLABench](https://github.com/OpenMOSS/VLABench)
- 项目网站：[vlabench.github.io](https://vlabench.github.io)
- 预训练策略：[`lerobot/smolvla_vlabench`](https://huggingface.co/lerobot/smolvla_vlabench)

## 可用任务

VLABench 提供了两个任务套件，涵盖 LeRobot `--env.task` 接口中的 **43 个任务类别**：

| 套件     | CLI 名称      | 任务数 | 描述                                                 |
| -------- | ------------ | ------ | ---------------------------------------------------- |
| 基础任务 | `primitive`  | 21     | 单一/少量技能组合（选择、插入、物理问答）             |
| 复合任务 | `composite`  | 22     | 多步推理和长时程规划（烹饪、整理）                     |

**基础任务：** `select_fruit`、`select_toy`、`select_chemistry_tube`、`add_condiment`、`select_book`、`select_painting`、`select_drink`、`insert_flower`、`select_billiards`、`select_ingredient`、`select_mahjong`、`select_poker`，以及物理推理任务（`density_qa`、`friction_qa`、`magnetism_qa`、`reflection_qa`、`simple_cuestick_usage`、`simple_seesaw_usage`、`sound_speed_qa`、`thermal_expansion_qa`、`weight_qa`）。

**复合任务：** `cluster_billiards`、`cluster_book`、`cluster_drink`、`cluster_toy`、`cook_dishes`、`cool_drink`、`find_unseen_object`、`get_coffee`、`hammer_nail`、`heat_food`、`make_juice`、`play_mahjong`、`play_math_game`、`play_poker`、`play_snooker`、`rearrange_book`、`rearrange_chemistry_tube`、`set_dining_table`、`set_study_table`、`store_food`、`take_chemistry_experiment`、`use_seesaw_complex`。

`--env.task` 接受三种形式：

- 单个任务名称（`select_fruit`）
- 逗号分隔列表（`select_fruit,heat_food`）
- 套件快捷方式（`primitive`、`composite` 或 `primitive,composite`）

## 安装

VLABench **不在 PyPI 上**——其唯一的分发渠道是 [OpenMOSS/VLABench](https://github.com/OpenMOSS/VLABench) GitHub 仓库——因此 LeRobot 没有提供 `vlabench` 额外依赖。请手动以可编辑克隆方式安装，同时安装 VLABench 所需的 MuJoCo / dm_control 固定版本，然后获取网格资源：

```bash
# 在遵循标准 LeRobot 安装说明之后。

git clone https://github.com/OpenMOSS/VLABench.git ~/VLABench
git clone https://github.com/motion-planning/rrt-algorithms.git ~/rrt-algorithms
pip install -e ~/VLABench -e ~/rrt-algorithms
pip install "mujoco==3.2.2" "dm-control==1.0.22" \
            open3d colorlog scikit-learn openai gdown

python ~/VLABench/scripts/download_assets.py
```

<Tip>
VLABench 需要 Linux（`sys_platform == 'linux'`）和 Python 3.10+。运行前请设置 MuJoCo 渲染后端：

```bash
export MUJOCO_GL=egl  # 用于无头服务器（HPC、云端）
```

</Tip>

## 评估

以下所有评估代码片段与 CI 运行的命令一致（参见 `.github/workflows/benchmark_tests.yml`）。`--rename_map` 参数将 VLABench 的 `image` / `second_image` / `wrist_image` 相机键映射到已发布的 `smolvla_vlabench` 策略训练时使用的三相机（`camera1` / `camera2` / `camera3`）输入布局。

### 单任务评估（推荐用于快速迭代）

```bash
lerobot-eval \
  --policy.path=lerobot/smolvla_vlabench \
  --env.type=vlabench \
  --env.task=select_fruit \
  --eval.batch_size=1 \
  --eval.n_episodes=10 \
  --eval.use_async_envs=false \
  --policy.device=cuda \
  '--rename_map={"observation.images.image": "observation.images.camera1", "observation.images.second_image": "observation.images.camera2", "observation.images.wrist_image": "observation.images.camera3"}'
```

### 多任务评估

传递逗号分隔的任务列表：

```bash
lerobot-eval \
  --policy.path=lerobot/smolvla_vlabench \
  --env.type=vlabench \
  --env.task=select_fruit,select_toy,add_condiment,heat_food \
  --eval.batch_size=1 \
  --eval.n_episodes=10 \
  --eval.use_async_envs=false \
  --policy.device=cuda \
  '--rename_map={"observation.images.image": "observation.images.camera1", "observation.images.second_image": "observation.images.camera2", "observation.images.wrist_image": "observation.images.camera3"}'
```

### 全套件评估

运行整个套件（所有 21 个基础任务或所有 22 个复合任务）：

```bash
lerobot-eval \
  --policy.path=lerobot/smolvla_vlabench \
  --env.type=vlabench \
  --env.task=primitive \
  --eval.batch_size=1 \
  --eval.n_episodes=10 \
  --eval.use_async_envs=false \
  --policy.device=cuda \
  --env.max_parallel_tasks=1 \
  '--rename_map={"observation.images.image": "observation.images.camera1", "observation.images.second_image": "observation.images.camera2", "observation.images.wrist_image": "observation.images.camera3"}'
```

或者同时运行两个套件：

```bash
lerobot-eval \
  --policy.path=lerobot/smolvla_vlabench \
  --env.type=vlabench \
  --env.task=primitive,composite \
  --eval.batch_size=1 \
  --eval.n_episodes=10 \
  --eval.use_async_envs=false \
  --policy.device=cuda \
  --env.max_parallel_tasks=1 \
  '--rename_map={"observation.images.image": "observation.images.camera1", "observation.images.second_image": "observation.images.camera2", "observation.images.wrist_image": "observation.images.camera3"}'
```

### 推荐的评估片段数

**每个任务 10 个片段**，以实现可复现的基准测试（完整基础套件共 210 个，复合套件共 220 个）。与 VLABench 论文中的协议一致。

## 策略输入和输出

**观测值：**

- `observation.state` — 7 维末端执行器状态（位置 xyz + 欧拉角 xyz + 夹爪）
- `observation.images.image` — 前部相机，480×480 HWC uint8
- `observation.images.second_image` — 第二相机，480×480 HWC uint8
- `observation.images.wrist_image` — 腕部相机，480×480 HWC uint8

**动作：**

- 连续控制，空间为 `Box(-1, 1, shape=(7,))` — 3 维位置 + 3 维欧拉角朝向 + 1 维夹爪。

## 训练

### 数据集

Hub 上已收集好的 VLABench 数据集，格式为 LeRobot 格式：

- [`VLABench/vlabench_primitive_ft_lerobot_video`](https://huggingface.co/datasets/VLABench/vlabench_primitive_ft_lerobot_video) — 5,000 个片段，128 个任务，480×480 图像。
- [`VLABench/vlabench_composite_ft_lerobot_video`](https://huggingface.co/datasets/VLABench/vlabench_composite_ft_lerobot_video) — 5,977 个片段，167 个任务，224×224 图像。

### 示例训练命令

在基础套件上微调 SmolVLA 基础模型：

```bash
lerobot-train \
  --policy.type=smolvla \
  --policy.repo_id=${HF_USER}/smolvla_vlabench_primitive \
  --policy.load_vlm_weights=true \
  --policy.push_to_hub=true \
  --dataset.repo_id=VLABench/vlabench_primitive_ft_lerobot_video \
  --env.type=vlabench \
  --env.task=select_fruit \
  --output_dir=./outputs/smolvla_vlabench_primitive \
  --steps=100000 \
  --batch_size=4 \
  --env_eval_freq=5000 \
  --eval.batch_size=1 \
  --eval.n_episodes=1 \
  --save_freq=10000
```

## 复现已发布的结果

已发布的检查点 [`lerobot/smolvla_vlabench`](https://huggingface.co/lerobot/smolvla_vlabench) 在上述基础套件数据集上训练，并使用[单任务评估](#单任务评估推荐用于快速迭代) / [全套件评估](#全套件评估)命令进行评估。CI 在每次涉及该基准测试的 PR 上运行包含 10 个基础任务的冒烟测试评估（每个任务一个片段）。
