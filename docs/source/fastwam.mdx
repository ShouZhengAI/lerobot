# FastWAM

FastWAM 是一种用于机器人控制的世界动作模型（World Action Model）策略。LeRobot 集成通过标准策略 API 暴露 FastWAM，使其可以通过 `policy.type=fastwam` 配置、使用 `lerobot-train` 训练，并通过 LeRobot 预训练策略接口加载。

## 模型概述

FastWAM 在训练期间保留视频建模，但在推理时使用直接动作预测，而非迭代生成未来观测。此 LeRobot 策略封装了 FastWAM 动作模型，将 LeRobot 批次适配为 FastWAM 训练样本，并提供标准的处理器流水线用于归一化和动作后处理。

该实现默认从 `Wan-AI/Wan2.2-TI2V-5B` 初始化视觉世界模型组件，并预测形状为 `[batch, action_horizon, action_dim]` 的动作块。

### LeRobot 集成涵盖的内容

- 通过 LeRobot 标准化的 `policy.type=fastwam` 配置
- 图像、状态、动作和语言任务的批次适配
- 通过 `select_action` 和 `predict_action_chunk` 进行动作块推理
- 通过 LeRobot 策略 API 进行检查点保存/加载
- 可配置的 LIBERO 夹爪动作后处理

## 安装要求

从源码安装 LeRobot，然后安装 FastWAM 依赖项：

```bash
pip install -e ".[fastwam]"
```

这将安装 `pyproject.toml` 中的 FastWAM 策略额外依赖：`transformers`、`diffusers`、`ftfy` 和 `regex`，以及 LeRobot 的基础依赖项。

如需 LIBERO 评估，同时安装基准测试依赖项：

```bash
pip install -e ".[fastwam,libero]"
```

这将安装两个额外依赖。除上述 FastWAM 依赖项外，`libero` 额外依赖还安装 LeRobot 数据集依赖项、Linux 上的 `hf-libero` 和 `scipy`。

FastWAM 使用 Wan2.2 TI2V 骨干网络。默认模型 ID 为：

```python
policy.model_id=Wan-AI/Wan2.2-TI2V-5B
```

## 数据要求

FastWAM 期望的 LeRobot 数据集包含：

- 一个或多个视觉观测，其宽度拼接后等于 `policy.image_size[1]`
- 当 `policy.proprio_dim` 不为 `None` 时的 `observation.state`
- `action`
- 通过数据集任务字段的语言任务指令，或预计算的 `context` 和 `context_mask` 张量

默认视觉设置是一个名为 `observation.images.image` 的图像特征，形状为 `(3, 224, 448)`。如果数据集使用两个相机，配置 `policy.input_features` 使其高度匹配 `224`，宽度之和为 `448`。

## 使用方法

创建新的 FastWAM 策略：

```bash
lerobot-train \
  --dataset.repo_id=your-org/your-dataset \
  --policy.type=fastwam \
  --policy.action_dim=7 \
  --policy.proprio_dim=8 \
  --policy.action_horizon=32 \
  --policy.n_action_steps=10 \
  --policy.image_size='[224,448]' \
  --output_dir=./outputs/fastwam_training \
  --job_name=fastwam_training \
  --steps=300000 \
  --batch_size=8 \
  --policy.device=cuda
```

在 LIBERO-10 上评估现有的 LeRobot 格式检查点：

```bash
lerobot-eval \
  --policy.path=ZibinDong/fastwam_libero_uncond_2cam224 \
  --policy.device=cuda \
  --policy.torch_dtype=float32 \
  --policy.n_action_steps=10 \
  --env.type=libero \
  --env.task=libero_10 \
  --env.observation_height=224 \
  --env.observation_width=224 \
  --eval.batch_size=1 \
  --eval.n_episodes=50 \
  --seed=0 \
  --env.episode_length=600
```

对于 `libero_goal`、`libero_spatial` 和 `libero_object`，使用 `--env.episode_length=300`。

对于真实机器人执行，使用相同的检查点路径：

```bash
lerobot-rollout \
  --robot.type=so101_follower \
  --robot.port=/dev/ttyACM0 \
  --policy.path=your-org/fastwam-real-robot
```

## 配置说明

### 图像特征

`policy.image_size` 是拼接后 FastWAM 图像张量的大小，格式为 `(height, width)`。每个配置的图像特征必须具有形状 `(3, height, camera_width)`，且所有相机宽度之和必须等于配置的宽度。

### 动作分块

`policy.action_horizon` 控制训练期间监督和推理期间预测的未来动作数量。`policy.n_action_steps` 控制策略预测新块之前消费的动作数量。`policy.n_action_steps` 必须小于或等于 `policy.action_horizon`。

### Wan 组件

FastWAM 从配置的 Wan 模型目录或 Hugging Face Hub 模型 ID 加载 Wan VAE、视频 DiT、文本编码器和分词器。通过 `save_pretrained` 保存的 LeRobot 格式 FastWAM 检查点也会复制 `from_pretrained` 所需的本地 Wan 组件文件。

### 注意力后端

FastWAM 的 DiT 使用 PyTorch 的 `scaled_dot_product_attention`（SDPA）处理所有注意力计算。它**不**使用 FlashAttention：其混合变换器（Mixture-of-Transformers, MoT）路由需要任意的布尔型 `[query, key]` 注意力掩码，而 FlashAttention varlen API 无法表达这种掩码。因此安装 `flash-attn` 包对 FastWAM 路径无任何影响。（注意，SDPA 本身仍可能在内部选择 PyTorch 自己的 flash/memory-efficient/math 内核——这与 `flash-attn` 包无关。）

### LIBERO 动作切换

FastWAM LIBERO 检查点默认使用 `policy.toggle_action_dimensions=[-1]` 以匹配原始 FastWAM 评估流水线使用的夹爪动作约定：

```bash
--policy.toggle_action_dimensions='[-1]'
```

## 结果

使用 [`ZibinDong/fastwam_libero_uncond_2cam224`](https://huggingface.co/ZibinDong/fastwam_libero_uncond_2cam224) 在 LIBERO 上评估：

| 任务集           | 成功率     | 测试轮次 |
| --------------- | ---------: | -------: |
| libero_spatial  |      97.6% |      500 |
| libero_object   |      99.0% |      500 |
| libero_goal     |      95.0% |      500 |
| libero_10       |      94.0% |      500 |
| **平均**        |  **96.4%** |     2000 |

复现命令：`lerobot-eval --policy.path=ZibinDong/fastwam_libero_uncond_2cam224 --policy.device=cuda --policy.torch_dtype=float32 --policy.n_action_steps=10 --env.type=libero --env.task=libero_spatial --env.observation_height=256 --env.observation_width=256 --eval.batch_size=1 --eval.n_episodes=50 --seed=0 --env.episode_length=300`（1x H20 140 GB）。
