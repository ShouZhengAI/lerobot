# 重命名映射与空摄像头

在训练、评估或使用机器人策略记录数据时，你的**数据集**或**环境**会以一套键名提供观测数据（例如 `observation.images.front`、`observation.images.eagle`），而你的**策略**期待的却是另一套键名（例如 `observation.images.image`、`observation.images.image2`）。**重命名映射**可以在不修改策略或数据源的情况下弥合这一差异。

> **适用范围：** 重命名映射仅重命名**观测**键（图像和状态）。动作键不受影响。

## 为什么观测键名总是不一致

策略在其预训练配置中固定了一组**输入特征名称**。例如：

- [pi0fast-libero](https://huggingface.co/lerobot/pi0fast-libero) 期望 `observation.images.base_0_rgb` 和 `observation.images.left_wrist_0_rgb`。
- [xvla-base](https://huggingface.co/lerobot/xvla-base) 期望 `observation.images.image`、`observation.images.image2` 和 `observation.images.image3`。

你的数据集可能使用了完全不同的名称（例如 `observation.images.front`、`observation.images.eagle`、`observation.images.glove`），而你的评估环境可能又是另一套。与其修改策略配置文件或重命名数据集中的列，不如传入一个**重命名映射（rename map）**：一个 JSON 字典，将源键名映射到策略期望的键名。重命名在预处理器管道内部完成，因此策略始终能接收到它期望的键名。

## 使用重命名映射

在命令行中以 JSON 字符串形式传入映射。约定格式始终为：

```
--rename_map='{"source_key": "policy_key", ...}'
```

其中 **source_key** 是数据集或环境提供的键名，**policy_key** 是策略期望的键名。

仅列出的键名会被重命名；其他所有键名保持不变。映射条目的顺序无关紧要。

支持的策略：**PI0**、**PI05**、**PI0Fast**、**SmolVLA** 和 **XVLA**。

### 训练

假设你对 [lerobot/xvla-base](https://huggingface.co/lerobot/xvla-base) 进行微调，使用的数据集图像位于 `observation.images.front`、`observation.images.eagle` 和 `observation.images.glove` 下。XVLA 期望的是 `observation.images.image`、`observation.images.image2` 和 `observation.images.image3`：

```bash
lerobot-train \
  --dataset.repo_id=YOUR_DATASET \
  --output_dir=./outputs/xvla_training \
  --job_name=xvla_training \
  --policy.path="lerobot/xvla-base" \
  --policy.repo_id="HF_USER/xvla-your-robot" \
  --policy.dtype=bfloat16 \
  --policy.action_mode=auto \
  --steps=20000 \
  --policy.device=cuda \
  --policy.freeze_vision_encoder=false \
  --policy.freeze_language_encoder=false \
  --policy.train_policy_transformer=true \
  --policy.train_soft_prompts=true \
  --rename_map='{"observation.images.front": "observation.images.image", "observation.images.eagle": "observation.images.image2", "observation.images.glove": "observation.images.image3"}'
```

### 评估

一个策略期望 `observation.images.base_0_rgb` 和 `observation.images.left_wrist_0_rgb`（例如 [pi0fast-libero](https://huggingface.co/lerobot/pi0fast-libero)），但 LIBERO 环境返回的是 `observation.images.image` 和 `observation.images.image2`：

```bash
lerobot-eval \
  --policy.path=lerobot/pi0fast-libero \
  --env.type=libero \
  ... \
  --rename_map='{"observation.images.image": "observation.images.base_0_rgb", "observation.images.image2": "observation.images.left_wrist_0_rgb"}'
```

## 备选方案：直接修改策略配置

如果你始终使用相同的数据集或环境，可以直接**编辑策略的 `config.json`**，使其观测键名与你的数据源匹配。这样就不需要重命名映射了。

权衡之处：修改策略配置会将其绑定到单一数据源。而重命名映射则能让一个策略在多个数据集和环境间保持通用。

## 空摄像头：视图数量少于策略期望

某些策略设计为固定数量的图像输入。如果你的摄像头数量较少，可以在策略配置中设置 **`empty_cameras`**，而无需修改模型架构。

### 工作原理

将 `empty_cameras=N` 设置为 N，会向策略配置添加 N 个占位图像特征，命名为：

```
observation.images.empty_camera_0
observation.images.empty_camera_1
...
```

运行时，这些键名在批次中没有对应的数据。策略会用掩码虚拟张量填充它们（对于基于 SigLIP 的视觉编码器填充 `-1`，并赋予零注意力掩码），从而使额外的图像槽位在训练和推理期间被有效忽略。

### 示例

XVLA-base 默认有三个视觉输入，`empty_cameras=0`。你的数据集只有两个摄像头：

1. 设置 `--policy.empty_cameras=1`。
2. 配置会添加第三个键名：`observation.images.empty_camera_0`。
3. 按常规使用重命名映射处理两个真实摄像头。
4. 第三个槽位被掩码屏蔽——数据集中不需要伪图像。

## 快速参考

| 目标                                     | 操作方式                                                                 |
| ---------------------------------------- | ------------------------------------------------------------------------ |
| 数据集键名 ≠ 策略键名                     | `--rename_map='{"dataset_key": "policy_key", ...}'`                      |
| 环境键名 ≠ 策略键名（评估时）             | `--rename_map='{"env_key": "policy_key", ...}'`                          |
| 不同键名的推理部署                        | `--rename_map='{"source_key": "policy_key", ...}'`                       |
| 摄像头数量少于策略期望                    | `--policy.empty_cameras=N`（PI0、PI05、PI0Fast、SmolVLA、XVLA 均支持）    |
| 避免使用重命名映射                        | 编辑策略的 `config.json`，使其键名与你的数据源匹配                         |
