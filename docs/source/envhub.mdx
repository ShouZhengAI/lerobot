# 从 Hub 加载环境

**EnvHub** 功能允许你通过一行代码直接从 Hugging Face Hub 加载仿真环境。这开启了一种强大的新型协作模式：环境不再被锁定在单一的库中，任何人都可以发布自定义环境并与社区分享。

## 什么是 EnvHub？

EnvHub 让你可以使用自己的机器人模型和场景创建自定义的机器人仿真环境，并使任何人都能通过 LeRobot 框架轻松使用。

EnvHub 包存储在 Hugging Face Hub 上，可以通过 LeRobot 以一行代码无缝拉取和使用到你的 AI 机器人项目中。

借助 EnvHub，你可以：

1. **创建并发布环境**到 Hugging Face Hub 作为 Git 仓库，无需打包难题即可分发复杂的物理仿真
2. **动态加载环境**，无需将其安装为软件包
3. **版本管理和追踪**环境变更，使用 Git 语义
4. **发现**社区共享的新仿真任务

这种设计意味着你可以从在 Hub 上发现一个有趣的环境，到在数秒内运行实验；或者创建你自己的自定义机器人及环境，而无需担心依赖冲突或复杂的安装流程。

当你创建 EnvHub 包时，你可以在其中构建任何你想要的内容，并使用任何你喜欢的仿真工具：这是你自己的发挥空间。唯一的要求是该包必须包含一个 `env.py` 文件，用于定义环境并允许 LeRobot 加载和使用你的 EnvHub 包。

这个 `env.py` 文件需要暴露一个小的 API，以便 LeRobot 可以加载和运行它。具体来说，你必须提供一个 `make_env(n_envs: int = 1, use_async_envs: bool = False)` 或 `make_env(n_envs: int = 1, use_async_envs: bool = False, cfg: EnvConfig)` 函数，这是 LeRobot 的主要入口点。它应返回以下之一：

- 一个 `gym.vector.VectorEnv`（最常见）
- 单个 `gym.Env`（将自动被包装）
- 一个字典映射 `{suite_name: {task_id: VectorEnv}}`（用于多任务基准测试）

你还可以向 `make_env` 传递一个 `EnvConfig` 对象来配置环境（例如环境数量、任务、相机名称、初始状态、控制模式、回合长度等）。

最后，你的环境必须实现标准的 `gym.vector.VectorEnv` 接口，以便与 LeRobot 兼容，包括 `reset` 和 `step` 等方法。

## 快速开始

从 Hub 加载环境就像这样简单：

```python
from lerobot.envs import make_env

# 加载一个 Hub 环境（需要明确同意执行远程代码）
env = make_env("lerobot/cartpole-env", trust_remote_code=True)
```

<Tip warning={true}>
  **安全提示**：从 Hub 加载环境会执行第三方仓库中的 Python 代码。
  仅对您信任的仓库使用 `trust_remote_code=True`。为了可重现性和安全性，
  我们强烈建议固定到特定的提交哈希。
</Tip>

## 仓库结构

要使你的环境可以从 Hub 加载，你的仓库至少需要包含以下内容：

### 必需文件

**`env.py`**（或自定义 Python 文件）

- 必须暴露一个 `make_env(n_envs: int, use_async_envs: bool)` 函数
- 该函数应返回以下之一：
  - 一个 `gym.vector.VectorEnv`（最常见）
  - 单个 `gym.Env`（将自动被包装）
  - 一个字典映射 `{suite_name: {task_id: VectorEnv}}`（用于多任务基准测试）

### 可选文件

**`requirements.txt`**

- 列出你的环境所需的任何额外依赖
- 用户在加载环境前需要手动安装这些依赖

**`README.md`**

- 记录你的环境：实现了什么任务、观测/动作空间、奖励等
- 包含使用示例和任何特殊的设置说明

**`.gitignore`**

- 从你的仓库中排除不必要的文件

### 示例仓库结构

```
my-environment-repo/
├── env.py                 # 主要环境定义（必需）
├── requirements.txt       # 依赖项（可选）
├── README.md             # 文档（推荐）
├── assets/               # 图片、视频等（可选）
│   └── demo.gif
└── configs/              # 配置文件（可选）
    └── task_config.yaml
```

## 创建你的环境仓库

### 步骤 1：定义你的环境

创建一个包含 `make_env` 函数的 `env.py` 文件：

```python
# env.py
import gymnasium as gym

def make_env(n_envs: int = 1, use_async_envs: bool = False):
    """
    为你的自定义任务创建向量化环境。

    Args:
        n_envs: 并行环境的数量
        use_async_envs: 是否使用 AsyncVectorEnv 或 SyncVectorEnv

    Returns:
        gym.vector.VectorEnv 或字典映射套件名称到向量化环境
    """
    def _make_single_env():
        # 创建你的自定义环境
        return gym.make("CartPole-v1")

    # 选择向量环境类型
    env_cls = gym.vector.AsyncVectorEnv if use_async_envs else gym.vector.SyncVectorEnv

    # 创建向量化环境
    vec_env = env_cls([_make_single_env for _ in range(n_envs)])

    return vec_env
```

### 步骤 2：本地测试

在上传之前，先在本地测试你的环境：

```python
from lerobot.envs.utils import _load_module_from_path, _call_make_env, _normalize_hub_result

# 加载你的模块
module = _load_module_from_path("./env.py")

# 测试 make_env 函数
result = _call_make_env(module, n_envs=2, use_async_envs=False)
normalized = _normalize_hub_result(result)

# 验证其是否正常工作
suite_name = next(iter(normalized))
env = normalized[suite_name][0]
obs, info = env.reset()
print(f"观测值形状: {obs.shape if hasattr(obs, 'shape') else type(obs)}")
env.close()
```

### 步骤 3：上传到 Hub

将你的仓库上传到 Hugging Face：

```bash
# 如有需要，安装 huggingface_hub
pip install huggingface_hub

# 登录 Hugging Face
hf auth login

# 创建新仓库
hf repo create my-org/my-custom-env

# 初始化 git 并推送
git init
git add .
git commit -m "Initial environment implementation"
git remote add origin https://huggingface.co/my-org/my-custom-env
git push -u origin main
```

或者，使用 `huggingface_hub` Python API：

```python
from huggingface_hub import HfApi

api = HfApi()

# 创建仓库
api.create_repo("my-custom-env", repo_type="space")

# 上传文件
api.upload_folder(
    folder_path="./my-env-folder",
    repo_id="username/my-custom-env",
    repo_type="space",
)
```

## 从 Hub 加载环境

### 基本用法

```python
from lerobot.envs import make_env

# 从 Hub 加载
envs_dict = make_env(
    "username/my-custom-env",
    n_envs=4,
    trust_remote_code=True
)

# 访问环境
suite_name = next(iter(envs_dict))
env = envs_dict[suite_name][0]

# 像使用任何 gym 环境一样使用
obs, info = env.reset()
action = env.action_space.sample()
obs, reward, terminated, truncated, info = env.step(action)
```

### 进阶：固定到特定版本

为了可重现性和安全性，固定到特定的 Git 修订版：

```python
# 固定到特定分支
env = make_env("username/my-env@main", trust_remote_code=True)

# 固定到特定提交（建议用于论文/实验）
env = make_env("username/my-env@abc123def456", trust_remote_code=True)

# 固定到标签
env = make_env("username/my-env@v1.0.0", trust_remote_code=True)
```

### 自定义文件路径

如果你的环境定义不在 `env.py` 中：

```python
# 从自定义文件加载
env = make_env("username/my-env:custom_env.py", trust_remote_code=True)

# 结合版本固定
env = make_env("username/my-env@v1.0:envs/task_a.py", trust_remote_code=True)
```

### 异步环境

为了在多环境中获得更好的性能：

```python
envs_dict = make_env(
    "username/my-env",
    n_envs=8,
    use_async_envs=True,  # 使用 AsyncVectorEnv 进行并行执行
    trust_remote_code=True
)
```

## URL 格式参考

Hub URL 格式支持多种模式：

| 模式 | 描述 | 示例 |
| --- | --- | --- |
| `user/repo` | 从主分支加载 `env.py` | `make_env("lerobot/pusht-env")` |
| `user/repo@revision` | 从特定版本加载 | `make_env("lerobot/pusht-env@main")` |
| `user/repo:path` | 加载自定义文件 | `make_env("lerobot/envs:pusht.py")` |
| `user/repo@rev:path` | 版本 + 自定义文件 | `make_env("lerobot/envs@v1:pusht.py")` |

## 多任务环境

对于包含多个任务的基准测试（如 LIBERO），返回一个嵌套字典：

```python
def make_env(n_envs: int = 1, use_async_envs: bool = False):
    env_cls = gym.vector.AsyncVectorEnv if use_async_envs else gym.vector.SyncVectorEnv

    # 返回字典：{suite_name: {task_id: VectorEnv}}
    return {
        "suite_1": {
            0: env_cls([lambda: gym.make("Task1-v0") for _ in range(n_envs)]),
            1: env_cls([lambda: gym.make("Task2-v0") for _ in range(n_envs)]),
        },
        "suite_2": {
            0: env_cls([lambda: gym.make("Task3-v0") for _ in range(n_envs)]),
        }
    }
```

## 安全考虑

<Tip warning={true}>
  **重要提示**：需要 `trust_remote_code=True` 标志才能从 Hub 执行环境代码。
  这是出于安全考虑的设计。
</Tip>

从 Hub 加载环境时：

1. **先审查代码**：在加载前访问仓库并检查 `env.py`
2. **固定到提交**：使用特定的提交哈希以确保可重现性
3. **检查依赖项**：审查 `requirements.txt` 中是否有可疑的包
4. **使用可信源**：优先选择官方组织或知名研究人员
5. **必要时沙箱化**：在隔离环境（容器、虚拟机）中运行不受信任的代码

安全使用的示例：

```python
# ❌ 不好：不经检查就加载
env = make_env("random-user/untrusted-env", trust_remote_code=True)

# ✅ 好：审查代码，然后固定到特定提交
# 1. 访问 https://huggingface.co/trusted-org/verified-env
# 2. 审查 env.py 文件
# 3. 复制提交哈希
env = make_env("trusted-org/verified-env@a1b2c3d4", trust_remote_code=True)
```

## 示例：从 Hub 加载 CartPole

以下是使用参考 CartPole 环境的完整示例：

```python
from lerobot.envs import make_env
import numpy as np

# 加载环境
envs_dict = make_env("lerobot/cartpole-env", n_envs=4, trust_remote_code=True)

# 获取向量化环境
suite_name = next(iter(envs_dict))
env = envs_dict[suite_name][0]

# 运行一个简单的回合
obs, info = env.reset()
done = np.zeros(env.num_envs, dtype=bool)
total_reward = np.zeros(env.num_envs)

while not done.all():
    # 随机策略
    action = env.action_space.sample()
    obs, reward, terminated, truncated, info = env.step(action)
    total_reward += reward
    done = terminated | truncated

print(f"平均奖励: {total_reward.mean():.2f}")
env.close()
```

### 对于环境作者

- **轻松分发**：无需 PyPI 打包

- **快速迭代**：即时推送更新

### 对于研究人员

- **快速实验**：一行代码加载任何环境
- **可重现性**：固定到特定提交
- **发现**：浏览 Hub 上的环境
- **无冲突**：无需安装冲突的包

### 对于社区

- **不断增长的生态系统**：更多样化的仿真任务

## 故障排除

### "拒绝执行远程代码"

你必须显式传递 `trust_remote_code=True`：

```python
env = make_env("user/repo", trust_remote_code=True)
```

### "未找到模块 X"

Hub 环境有你需要安装的依赖项：

```bash
# 检查仓库的 requirements.txt 并安装依赖
pip install gymnasium numpy
```

### "在模块中未找到 make_env"

你的 `env.py` 必须暴露一个 `make_env` 函数：

```python
def make_env(n_envs: int, use_async_envs: bool):
    # 你的实现
    pass
```

### 环境返回了错误类型

`make_env` 函数必须返回：

- 一个 `gym.vector.VectorEnv`，或者
- 单个 `gym.Env`，或者
- 一个字典 `{suite_name: {task_id: VectorEnv}}`

## 最佳实践

1. **记录你的环境**：在 README 中包含观测/动作空间描述、奖励结构和终止条件
2. **添加 requirements.txt**：列出所有依赖项及版本
3. **充分测试**：在推送前验证你的环境在本地正常工作
4. **使用语义化版本**：用版本号标记发布版本
5. **添加示例**：在 README 中包含使用示例
6. **保持简洁**：尽可能减少依赖项
7. **许可你的作品**：添加 LICENSE 文件以明确使用条款
