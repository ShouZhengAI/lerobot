# 人在环路数据采集（Human-In-the-Loop Data Collection）

人在环路（Human-In-the-Loop，简称 HIL）数据采集让您可以通过在实际机器人上部署训练好的策略来改进它，同时由人类操作员监控并在需要时进行干预。干预数据（恢复动作和修正行为）会与自主运行段一起被记录，生成更丰富的训练数据集，教策略如何处理失败情况。

---

## 为什么需要人在环路？

标准的行为克隆（behavioral cloning）仅在成功演示上训练策略。在部署过程中，微小的误差可能累积，导致机器人进入训练期间从未见过的状态（分布偏移，distribution shift）。HIL 数据采集通过以下方式解决这一问题：

- 在实际机器人上运行训练好的策略
- 在机器人即将失败时由人类进行干预
- 记录人类的恢复和修正动作作为训练数据
- 在合并后的数据集上微调策略

这样生成的策略不仅知道如何执行任务，还知道在出错时如何恢复。

---

## 工作原理

在 HIL 会话期间，人类操作员在每个回合（episode）中遵循以下循环：

1. **观察**策略自主运行
2. **暂停**——当失败即将发生时暂停，机器人保持当前位置
3. **接管控制**——通过远程操控（teleoperation）将机器人恢复到良好状态（恢复，recovery），然后修正行为
4. **交还控制给策略**——策略从当前位置恢复自主执行
5. 在回合中根据需要重复步骤 2–4 多次
6. **结束回合**——任务完成时结束，保存并进入下一个 rollout

自主运行段和人工控制段都会被记录。策略和人类可以在单个回合中多次交替控制，每次交接后回合从当前状态继续（不因干预而重置）。这样在一个连续的轨迹中捕获了自主执行、恢复和修正过程。采集完成后，合并后的数据集（原始演示 + HIL 数据）用于微调策略。

这个过程可以迭代重复：部署、采集、微调、再重复。每轮针对当前策略的失败模式。

```
┌─────────────────────────────────────────────────────────────────────────┐
│  策略 v0（在演示数据上训练）                                               │
│       ↓                                                                 │
│  HIL 采集（针对当前失败模式）→ 微调 → 策略 v1                              │
│       ↓                                                                 │
│  HIL 采集（针对新的失败模式）→ 微调 → 策略 v2                              │
│       ↓                                                                 │
│  ...（重复直到达到满意的性能）                                             │
└─────────────────────────────────────────────────────────────────────────┘
```

## 硬件要求

### 远程操控设备要求

`lerobot-rollout --strategy.type=dagger` 模式需要**带有活动电机（active motors）的远程操控设备**，需要能够：

- 通过编程启用/禁用扭矩（torque）
- 移动到目标位置（以便在暂停时镜像机器人状态）

**兼容的远程操控设备：**

- `bi_openarm_mini` - 双臂 OpenArm Mini
- `so_leader` - SO100 / SO101 领导臂

> [!IMPORTANT]
> 提供的命令默认使用 `bi_openarm_follower` + `bi_openarm_mini`。
> `so_follower` + `so_leader` 配置也已注册，可以通过 CLI 参数使用。

---

## 脚本

使用 `lerobot-rollout` 配合 `--strategy.type=dagger` 进行 HIL 数据采集。使用 `--inference.type=sync|rtc` 选择推理后端：

| 模式 | 标志 | 模型 |
| ------------------------ | ---------------------- | --------------------- |
| 标准模式（默认） | _（无需标志）_ | ACT, Diffusion Policy |
| 实时分块（RTC） | `--inference.type=rtc` | Pi0, Pi0.5, SmolVLA |

## 分步指南

### 步骤 1：预训练基础策略

首先，在您的演示数据集上训练一个策略：

```bash
python src/lerobot/scripts/lerobot_train.py \
    --dataset.repo_id=your-username/demo-dataset \
    --policy.type=pi0 \
    --output_dir=outputs/pretrain \
    --batch_size=32 \
    --steps=50000
```

### 步骤 2：采集 HIL 数据

**标准推理（ACT, Diffusion Policy）：**

```bash
lerobot-rollout --strategy.type=dagger \
    --robot.type=bi_openarm_follower \
    --robot.left_arm_config.port=can1 \
    --robot.left_arm_config.side=left \
    --robot.right_arm_config.port=can0 \
    --robot.right_arm_config.side=right \
    --robot.cameras='{left_wrist: {type: opencv, index_or_path: "/dev/video0", width: 1280, height: 720, fps: 30}, right_wrist: {type: opencv, index_or_path: "/dev/video4", width: 1280, height: 720, fps: 30}, base: {type: opencv, index_or_path: "/dev/video2", width: 640, height: 480, fps: 30}}' \
    --teleop.type=bi_openarm_mini \
    --teleop.left_arm_config.port=/dev/ttyACM0 \
    --teleop.right_arm_config.port=/dev/ttyACM1 \
    --policy.path=outputs/pretrain/checkpoints/last/pretrained_model \
    --dataset.repo_id=your-username/rollout_hil_dataset \
    --dataset.single_task="Fold the T-shirt properly" \
    --dataset.fps=30 \
    --strategy.num_episodes=50 \
    --interpolation_multiplier=2
```

**使用 RTC 处理大型模型（Pi0, Pi0.5, SmolVLA）：**

对于推理延迟高的模型，启用 RTC 以实现流畅执行：

```bash
lerobot-rollout --strategy.type=dagger \
    --inference.type=rtc \
    --inference.rtc.execution_horizon=20 \
    --inference.rtc.max_guidance_weight=5.0 \
    --inference.rtc.prefix_attention_schedule=LINEAR \
    --robot.type=bi_openarm_follower \
    --robot.left_arm_config.port=can1 \
    --robot.left_arm_config.side=left \
    --robot.right_arm_config.port=can0 \
    --robot.right_arm_config.side=right \
    --robot.cameras='{left_wrist: {type: opencv, index_or_path: "/dev/video0", width: 1280, height: 720, fps: 30}, right_wrist: {type: opencv, index_or_path: "/dev/video4", width: 1280, height: 720, fps: 30}, base: {type: opencv, index_or_path: "/dev/video2", width: 640, height: 480, fps: 30}}' \
    --teleop.type=bi_openarm_mini \
    --teleop.left_arm_config.port=/dev/ttyACM0 \
    --teleop.right_arm_config.port=/dev/ttyACM1 \
    --policy.path=outputs/pretrain/checkpoints/last/pretrained_model \
    --dataset.repo_id=your-username/rollout_hil_rtc_dataset \
    --dataset.single_task="Fold the T-shirt properly" \
    --dataset.fps=30 \
    --strategy.num_episodes=50 \
    --interpolation_multiplier=3
```

**控制方式（概念说明）：**

交互模型如下：

- **暂停输入**：暂停自主策略执行
- **接管输入**：将控制权移交给人类操作员并记录干预数据
- **交还策略输入**：将控制权交还给策略，继续同一个回合
- **回合控制输入**：根据需要保存/重新录制/停止/重置

具体的按键/踏板绑定可能因脚本和硬件集成而异。请以每个脚本打印的控制说明作为您设备上具体映射的依据。

**HIL 协议：**

1. 观察策略自主运行（远程操控设备空闲/自由状态）
2. 当发现即将失败时，触发**暂停输入**
   - 策略停止
   - 远程操控设备移动到匹配机器人位置（扭矩启用）
   - 暂停期间不记录帧
3. 触发**接管输入**以获得控制权
   - 远程操控设备扭矩禁用，可以自由移动
   - **恢复**：远程操控机器人回到良好状态
   - **修正**：修正错误行为
   - 所有动作均被记录
4. 触发**交还策略输入**
   - 策略从当前状态恢复自主执行
   - 您可以随时再次干预（重复步骤 2–4）
5. 任务完成（或达到回合时间限制）时结束并保存回合
6. **重置**：远程操控设备移动到机器人位置，您可以移动机器人到起始位置
7. 开始下一个回合

**脚踏板设置（Linux）：**

如果使用 USB 脚踏板（PCsensor FootSwitch），请确保访问权限：

```bash
sudo setfacl -m u:$USER:rw /dev/input/by-id/usb-PCsensor_FootSwitch-event-kbd
```

### 步骤 3：微调策略

在**合并后的**数据集（`demo-dataset` + `hil-dataset` 合并在一起）上进行微调：

```bash
python src/lerobot/scripts/lerobot_train.py \
    --dataset.repo_id=your-username/hil-dataset \
    --policy.type=pi0 \
    --policy.pretrained_path=outputs/pretrain/checkpoints/last/pretrained_model \
    --output_dir=outputs/hil_finetune \
    --steps=20000
```

然后部署微调后的策略，并从步骤 2 开始重复，以针对其剩余的失败模式。

## 高效进行 HIL 采集的技巧

### 何时干预

当观察到以下情况时进行干预：

- 机器人即将犯下不可逆转的错误
- 机器人犹豫不决或表现出不确定的行为
- 机器人偏离预期轨迹

### 恢复：远程操控回到良好状态

在恢复过程中，将机器人远程操控回到以下状态：

- 机器人处于熟悉且符合分布（in-distribution）的配置
- 当前子任务仍然可以完成
- 恢复轨迹本身是富有信息量的训练数据

### 修正的质量

在修正过程中：

- 提供**自信、干净**的轨迹
- 完整完成当前子任务
- 不要过度修正或添加不必要的动作

## 相关工作

这种 HIL 数据采集方法建立在交互式模仿学习的思想之上：

- **DAgger**（Ross 等，2011）提出了核心思想：不仅在专家演示上训练，而是在_学习者_访问到的状态上向专家查询修正。这通过在迭代中收集策略内数据（on-policy data），打破了标准行为克隆的误差累积循环。

- **HG-DAgger**（Kelly 等，2019）使其在机器人领域变得实用：人类专家监控机器人，仅在需要时进行干预，而不是标注每个状态。自主控制与人工控制之间的切换，就是此处脚本中使用的"暂停 → 接管 → 交还策略"循环。

- **RaC**（Hu 等，2025）通过将干预明确分解为**恢复**（远程操控回到良好状态）和**修正**（从该状态开始演示正确的行为），将这个循环扩展到长时域任务。这种分解正是 `lerobot-rollout` 中 DAgger 策略所遵循的协议。

- **π0.6/RECAP**（Physical Intelligence，2025）在 VLA 模型规模上应用了相同的迭代采集-微调循环，表明即使是大型预训练策略，也能从针对其自身失败模式的人类修正中显著受益。π0.6 就是使用 RECAP 训练的。
