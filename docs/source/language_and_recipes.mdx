# 语言列（Language columns）和配方（Recipes）

大多数 LeRobot 数据集每个回合（episode）只附带一个 `task` 字符串——这对简短的单指令技能来说足够了，但对于该领域正在发展的长时域、多模态机器人策略（高层规划、记忆、插话、VQA、工具使用）来说则不够。为了支持这些策略而无需分支数据集格式，LeRobot 使用两个可选的语言列和一个轻量级配方层扩展了 `LeRobotDataset`，可在训练时将这些行动态转换为聊天风格的样本。

这一设计清晰地分为三个层次：

1. **数据集中的数据** —— 作为两个可选列（`language_persistent` 和 `language_events`）存储在 `data/chunk-*/file-*.parquet` 中帧旁边的语言注释。没有这些列的数据集保持原有行为。
2. **配方（Recipe）** —— 一个 YAML 文件，声明要绑定哪些注释行以及如何将它们布局为聊天轮次（`role`、`content`、可选图像、可选工具调用）。配方是纯配置；添加新配方无需编写 Python 代码。
3. **训练格式** —— 在样本采样时，`RenderMessagesStep` 根据每帧注释解析配方，并输出 HF 风格的 `messages` 加上 LeRobot 特定的侧数据（`message_streams`、`target_message_indices`），供策略处理器消费。

本页依次描述每个层次。

## 第一层 —— 数据集中的语言列

两个可选列位于 `data/chunk-*/file-*.parquet` 中的帧数据旁边：

- `language_persistent`：一个行列表，对回合中的每一帧广播，用于表示持续活跃的状态，例如 `subtask`、`plan` 和 `memory`。
- `language_events`：一个行列表，仅出现在事件被发出的精确帧上，例如 `interjection`、`vqa` 和语音工具调用。

两个列共享相同的行结构（事件行省略 `timestamp`，因为该行所在的帧已经提供了它）：

```text
role: string
content: string | null
style: string | null
timestamp: float32        # 仅持续行
camera: string | null     # observation.images.* 特征键，仅视角相关行
tool_calls: list[Json] | null
```

`camera` 字段标记其 `content` 与特定摄像头视角相关的行。视角相关风格（`vqa` 和 `trace`）的行**必须**将 `camera` 设置为匹配的 `observation.images.*` 特征键。所有其他风格的行——包括以关节/笛卡尔术语描述机器人框架基元的 `motion`——**必须**将 `camera` 保留为 `null`。流程编写者和验证器通过 `validate_camera_field(style, camera)` 强制执行此规则。

`meta/tasks.parquet` 仍然是任务（task）的规范来源。特殊的 `${task}` 配方绑定始终读取该任务字符串，不依赖于语言注释。

### 架构

语言栈本身有三个内部模块支持第一层：

1. `lerobot.datasets.language` 定义了模式、风格注册表和 `column_for_style`。
2. `lerobot.datasets.language_render` 解析行并渲染消息。
3. `RenderMessagesStep` 将数据集样本转换为 `messages`、`message_streams` 和 `target_message_indices`。

`LeRobotDataset` 保持与配方无关（recipe-agnostic）。当提供 `language_persistent` 和 `language_events` 时，它会透传它们；没有注释的数据集保持原有行为。

## 第二层 —— 配方结构

配方是由 `TrainingRecipe` 和 `MessageTurn` 支持的 YAML 文件。它们声明要提取哪些注释行（通过 `bindings`）以及如何将它们组合成聊天轮次（`messages`）。

```yaml
messages:
  - { role: user, content: "${task}", stream: high_level }
  - { role: assistant, content: "${subtask}", stream: low_level, target: true }
```

一个配方还可以分支成交替播的加权 **混合（blend）** 子配方。在样本采样时，根据样本索引确定性地选择一个分支，因此不同的帧训练不同的目标（例如内存更新 vs. 低层执行 vs. VQA），无需任何 Python 代码。

### 时间语义

持久风格在发出后保持活跃，直到被替换：

- `active_at(t, style=subtask)`
- `nth_prev(style=memory, offset=1)`
- `nth_next(style=subtask, offset=1)`

事件风格仅存在于其精确时间戳上：

- `emitted_at(t, style=interjection)`
- `emitted_at(t, style=vqa, role=user, camera=observation.images.top)`
- `emitted_at(t, role=assistant, tool_name=say)`

精确事件匹配没有容忍窗口，因此编写者必须使用 parquet 数据中的帧时间戳来标记事件行。

### 视角相关解析

对于视角相关的风格（`vqa` 和 `trace`），解析器获得一个并行于 `role=` 和 `tool_name=` 的 `camera=` 过滤器。具有多个摄像头的数据集通常在每个摄像头相同时间戳上发出一个（`vqa`，`user`）+（`vqa`，`assistant`）对；如果没有 `camera=`，这些解析器会看到两个匹配并引发歧义错误。配方通过自己的绑定加上匹配的图像块来消费每个摄像头，例如：

```yaml
ask_vqa_top:
  bindings:
    vqa_query: "emitted_at(t, style=vqa, role=user, camera=observation.images.top)"
    vqa: "emitted_at(t, style=vqa, role=assistant, camera=observation.images.top)"
  messages:
    - role: user
      stream: high_level
      if_present: vqa_query
      content:
        - { type: image, feature: observation.images.top }
        - { type: text, text: "${vqa_query}" }
    - {
        role: assistant,
        content: "${vqa}",
        stream: high_level,
        target: true,
        if_present: vqa,
      }
```

为数据集记录的每个摄像头添加一个这样的子配方。

## 第三层 —— 训练格式

渲染后的样本使用 HF 风格的聊天消息加上 LeRobot 侧数据：

```python
sample["messages"]
sample["message_streams"]
sample["target_message_indices"]
```

渲染器不应用分词器的聊天模板。策略处理器自行决定如何为其骨干网络序列化消息，这使得同一个数据集能够在 SmolVLA、Pi0.5 以及任何未来期望 OpenAI 风格聊天消息的 VLM 之间共用。

## 优雅的缺失处理

如果两个语言列都缺失、为 `None` 或为空，`RenderMessagesStep` 将不执行任何操作。
如果在没有所需事件行的帧上选择了事件作用域的分支，渲染返回 `None`，允许加载器重试另一个样本。
