# 接入你自己的硬件（Bring Your Own Hardware）

本教程将讲解如何将你自己的机器人设计集成到 LeRobot 生态系统中，并使其能够使用我们所有的工具（数据采集、控制流程、策略训练和推理）。

为此，我们在 LeRobot 中提供了 [`Robot`](https://github.com/huggingface/lerobot/blob/main/src/lerobot/robots/robot.py) 基类，它定义了物理机器人集成的标准接口。让我们来看看如何实现它。

## 前置条件

- 你自己的机器人，具备通信接口（例如串口、CAN、TCP）
- 能够通过编程方式读取传感器数据和发送电机指令，例如使用厂商的 SDK、API 或你自己的协议实现
- 环境中已安装 LeRobot。请参考我们的 [安装指南](./installation)

## 选择你的电机

如果你使用的是 Feetech 或 Dynamixel 电机，LeRobot 提供了内置的总线接口：

- [`FeetechMotorsBus`](https://github.com/huggingface/lerobot/blob/main/src/lerobot/motors/feetech/feetech.py) – 用于控制 Feetech 舵机
- [`DynamixelMotorsBus`](https://github.com/huggingface/lerobot/blob/main/src/lerobot/motors/dynamixel/dynamixel.py) – 用于控制 Dynamixel 舵机

请参考 [`MotorsBus`](https://github.com/huggingface/lerobot/blob/main/src/lerobot/motors/motors_bus.py) 抽象类了解其 API。
如需查看使用示例，可以参考我们自己的 [SO101 从端实现](https://github.com/huggingface/lerobot/blob/main/src/lerobot/robots/so_follower/so101_follower/so101_follower.py)

如果兼容，请直接使用这些接口。否则，你需要找到或编写一个 Python 接口（本教程不涉及）：

- 寻找已有的 Python SDK（或使用 C/C++ 绑定）
- 或实现一个基本的通信封装（例如通过 pyserial、socket 或 CANopen）

你并不孤单——许多社区贡献都使用了自定义板卡或固件！

对于 Feetech 和 Dynamixel，我们目前支持以下舵机： - Feetech： - STS 和 SMS 系列（协议 0）：`sts3215`、`sts3250`、`sm8512bl` - SCS 系列（协议 1）：`scs0009` - Dynamixel（仅协议 2.0）：`xl330-m077`、`xl330-m288`、`xl430-w250`、`xm430-w350`、`xm540-w270`、`xc430-w150`

如果你使用的 Feetech 或 Dynamixel 舵机不在这个列表中，可以将其添加到 [Feetech 表格](https://github.com/huggingface/lerobot/blob/main/src/lerobot/motors/feetech/tables.py)或 [Dynamixel 表格](https://github.com/huggingface/lerobot/blob/main/src/lerobot/motors/dynamixel/tables.py)中。根据具体型号，可能需要添加一些型号特定的信息。不过大多数情况下，需要添加的内容并不多。

在接下来的章节中，我们将使用 `FeetechMotorsBus` 作为示例的电机接口。如有必要，请替换为你自己的电机并进行相应调整。

## 第一步：继承 `Robot` 接口

首先需要指定配置类和机器人的字符串标识符（`name`）。如果你的机器人有特殊需求，希望便于修改，应该放在这里（例如端口/地址、波特率）。

这里，我们为机器人添加端口名称和一个默认摄像头：

<!-- prettier-ignore-start -->
```python
from dataclasses import dataclass, field

from lerobot.cameras import CameraConfig
from lerobot.cameras.opencv import OpenCVCameraConfig
from lerobot.robots import RobotConfig


@RobotConfig.register_subclass("my_cool_robot")
@dataclass
class MyCoolRobotConfig(RobotConfig):
    port: str
    cameras: dict[str, CameraConfig] = field(
        default_factory={
            "cam_1": OpenCVCameraConfig(
                index_or_path=2,
                fps=30,
                width=480,
                height=640,
            ),
        }
    )
```
<!-- prettier-ignore-end -->

[摄像头教程](./cameras)了解如何检测和添加摄像头。

接下来，我们创建实际的机器人类，继承自 `Robot`。这个抽象类定义了一个契约，你的机器人必须遵守才能与 LeRobot 的其他工具配合使用。

这里我们将创建一个带有一个摄像头的 5 自由度（5-DoF）机器人。它可以是一个简单的机械臂，但请注意 `Robot` 抽象类并未对你的机器人的外形做出任何假设。在设计新机器人时，你可以尽情发挥想象力！

<!-- prettier-ignore-start -->
```python
from lerobot.cameras import make_cameras_from_configs
from lerobot.motors import Motor, MotorNormMode
from lerobot.motors.feetech import FeetechMotorsBus
from lerobot.robots import Robot

class MyCoolRobot(Robot):
    config_class = MyCoolRobotConfig
    name = "my_cool_robot"

    def __init__(self, config: MyCoolRobotConfig):
        super().__init__(config)
        self.bus = FeetechMotorsBus(
            port=self.config.port,
            motors={
                "joint_1": Motor(1, "sts3250", MotorNormMode.RANGE_M100_100),
                "joint_2": Motor(2, "sts3215", MotorNormMode.RANGE_M100_100),
                "joint_3": Motor(3, "sts3215", MotorNormMode.RANGE_M100_100),
                "joint_4": Motor(4, "sts3215", MotorNormMode.RANGE_M100_100),
                "joint_5": Motor(5, "sts3215", MotorNormMode.RANGE_M100_100),
            },
            calibration=self.calibration,
        )
        self.cameras = make_cameras_from_configs(config.cameras)
```
<!-- prettier-ignore-end -->

## 第二步：定义观测（Observation）和动作（Action）特征

这两个属性定义了机器人与使用它的工具（如数据采集或学习流程）之间的**接口约定**。

> [!WARNING]
> 请注意，这些属性必须在机器人尚未连接时也能被调用，因此在定义它们时应避免依赖运行时的硬件状态。

### `observation_features`

该属性应返回一个字典，描述机器人传感器输出的结构。键与 `get_observation()` 返回的内容匹配，值描述形状（对于数组/图像）或类型（对于简单值）。

我们的 5 自由度机械臂带一个摄像头的示例如下：

<!-- prettier-ignore-start -->
```python
@property
def _motors_ft(self) -> dict[str, type]:
    return {
        "joint_1.pos": float,
        "joint_2.pos": float,
        "joint_3.pos": float,
        "joint_4.pos": float,
        "joint_5.pos": float,
    }

@property
def _cameras_ft(self) -> dict[str, tuple]:
    return {
        cam: (self.cameras[cam].height, self.cameras[cam].width, 3) for cam in self.cameras
    }

@property
def observation_features(self) -> dict:
    return {**self._motors_ft, **self._cameras_ft}
```
<!-- prettier-ignore-end -->

在这个例子中，观测（observation）由一个简单的字典组成，存储每个电机的位置和一张摄像头图像。

### `action_features`

该属性描述机器人通过 `send_action()` 期望接收的指令。同样，键必须与预期的输入格式匹配，值定义每个指令的形状/类型。

这里，我们使用与 `observation_features` 相同的关节本体感受特征（`self._motors_ft`）：发送的动作（action）就是每个电机的目标位置。

<!-- prettier-ignore-start -->
```python
def action_features(self) -> dict:
    return self._motors_ft
```
<!-- prettier-ignore-end -->

## 第三步：处理连接和断开连接

这些方法应处理与硬件通信的打开和关闭（例如串口、CAN 接口、USB 设备、摄像头）。

### `is_connected`

该属性应简单地反映与机器人硬件的通信是否已建立。当该属性为 `True` 时，应可以使用 `get_observation()` 和 `send_action()` 对硬件进行读写操作。

<!-- prettier-ignore-start -->
```python
@property
def is_connected(self) -> bool:
    return self.bus.is_connected and all(cam.is_connected for cam in self.cameras.values())
```
<!-- prettier-ignore-end -->

### `connect()`

该方法应建立与硬件的通信。此外，如果你的机器人需要标定且尚未标定，默认应启动标定流程。如果机器人需要某些特定配置，也应在该方法中进行。

<!-- prettier-ignore-start -->
```python
def connect(self, calibrate: bool = True) -> None:
    self.bus.connect()
    if not self.is_calibrated and calibrate:
        self.calibrate()

    for cam in self.cameras.values():
        cam.connect()

    self.configure()
```
<!-- prettier-ignore-end -->

### `disconnect()`

该方法应优雅地终止与硬件的通信：释放所有相关资源（线程或进程）、关闭端口等。

这里，我们的 `MotorsBus` 和 `Camera` 类已经处理了这些，所以我们只需要调用它们各自的 `disconnect()` 方法：

<!-- prettier-ignore-start -->
```python
def disconnect(self) -> None:
    self.bus.disconnect()
    for cam in self.cameras.values():
        cam.disconnect()
```
<!-- prettier-ignore-end -->

## 第四步：支持标定和配置

LeRobot 支持自动保存和加载标定数据。这对于关节偏移、零位或传感器对齐非常有用。

> 请注意，根据你的硬件情况，这可能不适用。如果是这样，你可以将这些方法留空：

<!-- prettier-ignore-start -->
```python
@property
def is_calibrated(self) -> bool:
    return True

def calibrate(self) -> None:
    pass
```
<!-- prettier-ignore-end -->

### `is_calibrated`

该属性应反映你的机器人是否已加载所需的标定数据。

<!-- prettier-ignore-start -->
```python
@property
def is_calibrated(self) -> bool:
    return self.bus.is_calibrated
```
<!-- prettier-ignore-end -->

### `calibrate()`

标定的目标有两个：

- 了解每个电机的物理运动范围，以便仅在此范围内发送指令。
- 将原始的电机位置归一化为合理的连续值（例如百分比、角度），而不是依赖于特定电机的任意离散值，这样在其他地方无法复现。

它应实现标定的逻辑（如果适用）并更新 `self.calibration` 字典。如果你使用的是 Feetech 或 Dynamixel 电机，我们的总线接口已经包含了辅助完成此操作的方法。

<!-- prettier-ignore-start -->
```python
def calibrate(self) -> None:
    self.bus.disable_torque()
    for motor in self.bus.motors:
        self.bus.write("Operating_Mode", motor, OperatingMode.POSITION.value)

    input(f"将 {self} 移动到其运动范围中间位置，然后按 ENTER....")
    homing_offsets = self.bus.set_half_turn_homings()

    print(
        "依次将每个关节移动至整个运动范围。\n"
        "记录位置。按 ENTER 停止..."
    )
    range_mins, range_maxes = self.bus.record_ranges_of_motion()

    self.calibration = {}
    for motor, m in self.bus.motors.items():
        self.calibration[motor] = MotorCalibration(
            id=m.id,
            drive_mode=0,
            homing_offset=homing_offsets[motor],
            range_min=range_mins[motor],
            range_max=range_maxes[motor],
        )

    self.bus.write_calibration(self.calibration)
    self._save_calibration()
    print("标定数据已保存至", self.calibration_fpath)
```
<!-- prettier-ignore-end -->

### `configure()`

使用此方法设置硬件的各种配置（舵机控制模式、控制器增益等）。通常应在连接时运行，并且应是幂等的。

<!-- prettier-ignore-start -->
```python
def configure(self) -> None:
    with self.bus.torque_disabled():
        self.bus.configure_motors()
        for motor in self.bus.motors:
            self.bus.write("Operating_Mode", motor, OperatingMode.POSITION.value)
            self.bus.write("P_Coefficient", motor, 16)
            self.bus.write("I_Coefficient", motor, 0)
            self.bus.write("D_Coefficient", motor, 32)
```
<!-- prettier-ignore-end -->

## 第五步：实现传感器读取和动作发送

这些是最重要的运行时函数：核心 I/O 循环。

### `get_observation()`

返回一个包含机器人传感器值的字典。这些通常包括电机状态、摄像头帧、各种传感器等。在 LeRobot 框架中，这些观测值（observation）将被输入策略（policy），用于预测要执行的动作（action）。字典的键和结构必须与 `observation_features` 一致。

<!-- prettier-ignore-start -->
```python
def get_observation(self) -> dict[str, Any]:
    if not self.is_connected:
        raise ConnectionError(f"{self} 未连接。")

    # 读取机械臂位置
    obs_dict = self.bus.sync_read("Present_Position")
    obs_dict = {f"{motor}.pos": val for motor, val in obs_dict.items()}

    # 从摄像头采集图像
    for cam_key, cam in self.cameras.items():
        obs_dict[cam_key] = cam.async_read()

    return obs_dict
```
<!-- prettier-ignore-end -->

### `send_action()`

接收一个与 `action_features` 匹配的字典，并将其发送到你的硬件。你可以添加安全限制（限幅、平滑处理）并返回实际发送的内容。

为简单起见，本例中我们不对动作做任何修改。

<!-- prettier-ignore-start -->
```python
def send_action(self, action: dict[str, Any]) -> dict[str, Any]:
    goal_pos = {key.removesuffix(".pos"): val for key, val in action.items()}

    # 向机械臂发送目标位置
    self.bus.sync_write("Goal_Position", goal_pos)

    return action
```
<!-- prettier-ignore-end -->

## 添加遥操作器（Teleoperator）

对于实现遥操作设备，我们还提供了一个 [`Teleoperator`](https://github.com/huggingface/lerobot/blob/main/src/lerobot/teleoperators/teleoperator.py) 基类。该类与 `Robot` 基类非常相似，同样不对外形做出任何假设。

主要区别在于 I/O 函数：遥操作器通过 `get_action` 产生动作（action），并通过 `send_feedback` 接收反馈动作（feedback）。反馈可以是遥操作设备上任何可控的内容，帮助操作人员理解所发送动作的后果。例如主端（leader）机械臂上的力/力矩反馈，或游戏手柄控制器上的震动。要实现一个遥操作器，你可以按照本教程并针对这两个方法进行调整。

## 使用你自己的 LeRobot 设备 🔌

你可以轻松地通过创建独立的、可安装的 Python 包来扩展 `lerobot`，以支持你自己的自定义硬件——无论是摄像头、机器人还是遥操作设备。只需遵循一些简单的约定，`lerobot` 的命令行工具（如 `lerobot-teleop` 和 `lerobot-record`）就会**自动发现并集成你的创作**，无需修改 `lerobot` 源代码。

本指南概述了你的插件必须遵循的约定。

### 4 个核心约定

为确保你的自定义设备可被发现，你必须遵守以下四条规则。

#### 1. 创建具有特定前缀的可安装包

你的项目必须是一个标准的、可安装的 Python 包。关键的是，包名（在 `pyproject.toml` 或 `setup.py` 中定义）必须以以下前缀之一开头：

- `lerobot_robot_` 用于机器人
- `lerobot_camera_` 用于摄像头
- `lerobot_teleoperator_` 用于遥操作设备

这个前缀系统是 `lerobot` 在 Python 环境中自动找到你插件的方式。

#### 2. 遵循 `SomethingConfig`/`Something` 命名模式

你的设备实现类必须以其配置类的名称命名，只需去掉 `Config` 后缀。

- **配置类：** `MyAwesomeTeleopConfig`
- **设备类：** `MyAwesomeTeleop`

#### 3. 将文件放置在可预测的结构中

设备类（`MyAwesomeTeleop`）必须位于相对于其配置类（`MyAwesomeTeleopConfig`）的可预测模块中。`lerobot` 将自动在这些位置搜索：

- 与配置类**同一模块**
- **以设备名称命名的子模块**（例如 `my_awesome_teleop.py`）

推荐且最简单的结构是将它们放在同一目录下各自独立、名称清晰的文件中。

#### 4. 在 `__init__.py` 中暴露类

你的包的 `__init__.py` 文件应导入并暴露配置类与设备类，使其易于访问。

### 完整示例

让我们创建一个名为 `my_awesome_teleop` 的新遥操作器。

#### 目录结构

以下是项目文件夹的样子。包名 `lerobot_teleoperator_my_awesome_teleop` 遵循**约定 #1**。

```
lerobot_teleoperator_my_awesome_teleop/
├── pyproject.toml  # 或 setup.py，将 lerobot 列为依赖项
└── lerobot_teleoperator_my_awesome_teleop/
    ├── __init__.py
    ├── config_my_awesome_teleop.py
    └── my_awesome_teleop.py
```

#### 文件内容

- **`config_my_awesome_teleop.py`**：定义配置类。注意 `Config` 后缀（**约定 #2**）。

  ```python
  from dataclasses import dataclass

  from lerobot.teleoperators.config import TeleoperatorConfig

  @TeleoperatorConfig.register_subclass("my_awesome_teleop")
  @dataclass
  class MyAwesomeTeleopConfig(TeleoperatorConfig):
      # 你的配置字段放在这里
      port: str = "192.168.1.1"
  ```

- **`my_awesome_teleop.py`**：实现设备。类名 `MyAwesomeTeleop` 与其配置类名称匹配（**约定 #2**）。此文件结构遵循**约定 #3**。

  ```python
  from lerobot.teleoperators.teleoperator import Teleoperator

  from .config_my_awesome_teleop import MyAwesomeTeleopConfig

  class MyAwesomeTeleop(Teleoperator):
      config_class = MyAwesomeTeleopConfig
      name = "my_awesome_teleop"

      def __init__(self, config: MyAwesomeTeleopConfig):
          super().__init__(config)
          self.config = config

      # 你的设备逻辑（例如连接）放在这里
  ```

- **`__init__.py`**：暴露关键类（**约定 #4**）。

  ```python
  from .config_my_awesome_teleop import MyAwesomeTeleopConfig
  from .my_awesome_teleop import MyAwesomeTeleop
  ```

### 安装和使用

1.  **在你的 Python 环境中安装新插件。**你可以使用 `pip` 的可编辑模式本地安装插件包，或者从 PyPi 安装。

    ```bash
    # 本地安装
    # 导航到插件根目录并安装
    cd lerobot_teleoperator_my_awesome_teleop
    pip install -e .

    # 从 PyPi 安装
    pip install lerobot_teleoperator_my_awesome_teleop
    ```

2.  **直接从命令行使用。**现在，你可以通过引用其类型来使用自定义设备。

    ```bash
    lerobot-teleoperate --teleop.type=my_awesome_teleop \
    # 其他参数
    ```

就这样！你的自定义设备现已完全集成。
