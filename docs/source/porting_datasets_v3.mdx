# 将大型数据集移植到 LeRobot Dataset v3.0

本教程介绍如何将大规模机器人数据集移植到 LeRobot Dataset v3.0 格式。我们以 **DROID 1.0.1** 数据集为主要示例，演示如何在 SLURM 集群上处理包含数千个分片（shard）的数 TB 级数据集。

## 文件组织方式：v2.1 与 v3.0 对比

Dataset v3.0 从根本上改变了数据的组织和存储方式：

**v2.1 结构（基于片段）**：

```
dataset/
├── data/chunk-000/episode_000000.parquet
├── data/chunk-000/episode_000001.parquet
├── videos/chunk-000/camera/episode_000000.mp4
└── meta/episodes.jsonl
```

**v3.0 结构（基于文件）**：

```
dataset/
├── data/chunk-000/file-000.parquet        # 多个片段合并在一个文件中
├── videos/camera/chunk-000/file-000.mp4   # 合并的视频块
└── meta/episodes/chunk-000/file-000.parquet  # 结构化元数据
```

从单个片段文件过渡到基于文件的块，极大地提升了性能并减少了存储开销。

## Dataset v3.0 的新特性

Dataset v3.0 引入了处理大规模数据集的重要改进：

### 🏗️ **增强的文件组织**

- **基于文件的结构**：片段现在分组到分块文件中，不再使用单个片段文件
- **可配置的文件大小**：可配置数据文件和视频文件的大小
- **更高的存储效率**：更好的压缩和更低的开销

### 📊 **现代化的元数据管理**

- **基于 Parquet 的元数据**：用高效的 parquet 格式取代了 JSON Lines
- **结构化片段访问**：通过 `dataset.meta.episodes` 直接以 pandas DataFrame 形式访问
- **逐片段统计**：在片段级别提供增强的统计信息追踪

### 🚀 **性能提升**

- **内存映射访问**：通过 PyArrow 内存映射改善内存使用
- **更快的加载速度**：显著减少数据集初始化时间
- **更好的可扩展性**：专为数百万片段的规模设计

## 前提条件

在移植大型数据集之前，请确保您已准备好以下条件：

- **已安装支持 v3.0 的 LeRobot**。请按照我们的[安装指南](./installation)操作。
- **充足的存储空间**：原始数据集可能非常大（例如，DROID 需要 2TB）
- **集群访问权限**（大型数据集推荐）：SLURM 或类似的作业调度器
- **数据集特定依赖**：对于 DROID，需要 TensorFlow Dataset 工具

## 了解 DROID 数据集

[DROID 1.0.1](https://droid-dataset.github.io/droid/the-droid-dataset) 是大规模机器人数据集的优秀示例：

- **大小**：1.7TB（RLDS 格式），8.7TB（原始数据）
- **结构**：2048 个预定义的 TensorFlow 数据集分片
- **内容**：来自 Franka Emika Panda 机器人的 76,000 多个操作轨迹
- **范围**：涵盖多种环境和物体的真实世界操作任务
- **格式**：原始格式为 TensorFlow Records/RLDS，需转换为 LeRobot 格式
- **托管**：Google Cloud Storage，可通过 `gsutil` 公开访问

该数据集包含多样化的操作演示内容：

- 多视角摄像头（腕部摄像头、外部摄像头）
- 自然语言任务描述
- 机器人本体感知状态和动作
- 成功/失败标注

### DROID 特征模式

```python
DROID_FEATURES = {
    # 片段标记
    "is_first": {"dtype": "bool", "shape": (1,)},
    "is_last": {"dtype": "bool", "shape": (1,)},
    "is_terminal": {"dtype": "bool", "shape": (1,)},

    # 语言指令
    "language_instruction": {"dtype": "string", "shape": (1,)},
    "language_instruction_2": {"dtype": "string", "shape": (1,)},
    "language_instruction_3": {"dtype": "string", "shape": (1,)},

    # 机器人状态
    "observation.state.gripper_position": {"dtype": "float32", "shape": (1,)},
    "observation.state.cartesian_position": {"dtype": "float32", "shape": (6,)},
    "observation.state.joint_position": {"dtype": "float32", "shape": (7,)},

    # 摄像头观察
    "observation.images.wrist_left": {"dtype": "image"},
    "observation.images.exterior_1_left": {"dtype": "image"},
    "observation.images.exterior_2_left": {"dtype": "image"},

    # 动作
    "action.gripper_position": {"dtype": "float32", "shape": (1,)},
    "action.cartesian_position": {"dtype": "float32", "shape": (6,)},
    "action.joint_position": {"dtype": "float32", "shape": (7,)},

    # 标准 LeRobot 格式
    "observation.state": {"dtype": "float32", "shape": (8,)},  # 关节 + 夹爪
    "action": {"dtype": "float32", "shape": (8,)},  # 关节 + 夹爪
}
```

## 方法一：单机移植

### 步骤 1：安装依赖

针对 DROID 的特定安装：

```bash
pip install tensorflow
pip install tensorflow_datasets
```

对于其他数据集，请根据源格式安装相应的读取库。

### 步骤 2：下载原始数据

使用 `gsutil` 从 Google Cloud Storage 下载 DROID：

```bash
# 如果尚未安装 Google Cloud SDK，请先安装
# https://cloud.google.com/sdk/docs/install

# 下载完整的 RLDS 数据集（1.7TB）
gsutil -m cp -r gs://gresearch/robotics/droid/1.0.1 /your/data/

# 或仅下载 100 个片段的样本（2GB）用于测试
gsutil -m cp -r gs://gresearch/robotics/droid_100 /your/data/
```

> [!WARNING]
> 大型数据集需要大量时间和存储空间：
>
> - **完整 DROID（1.7TB）**：根据带宽可能需要数天下载
> - **处理时间**：本地移植完整数据集需要 7 天以上
> - **上传时间**：推送到 Hugging Face Hub 需要 3 天以上
> - **本地存储**：处理后的 LeRobot 格式约需 400GB

### 步骤 3：移植数据集

```bash
python examples/port_datasets/port_droid.py \
    --raw-dir /your/data/droid/1.0.1 \
    --repo-id your_id/droid_1.0.1 \
    --push-to-hub
```

### 开发和测试

开发期间可以仅移植单个分片：

```bash
python examples/port_datasets/port_droid.py \
    --raw-dir /your/data/droid/1.0.1 \
    --repo-id your_id/droid_1.0.1_test \
    --num-shards 2048 \
    --shard-index 0
```

这种方法适用于较小的数据集或测试场景，但大型数据集需要集群计算。

## 方法二：SLURM 集群移植（推荐）

对于像 DROID 这样的大型数据集，跨多个节点的并行处理可大幅缩短处理时间。

### 步骤 1：安装集群依赖

```bash
pip install datatrove  # Hugging Face 分布式处理库
```

### 步骤 2：配置 SLURM 环境

查看您的分区信息：

```bash
sinfo --format="%R"  # 列出可用分区
sinfo -N -p your_partition -h -o "%N cpus=%c mem=%m"  # 查看资源信息
```

选择 **CPU 分区**——数据集移植不需要 GPU。

### 步骤 3：启动并行移植作业

```bash
python examples/port_datasets/slurm_port_shards.py \
    --raw-dir /your/data/droid/1.0.1 \
    --repo-id your_id/droid_1.0.1 \
    --logs-dir /your/logs \
    --job-name port_droid \
    --partition your_partition \
    --workers 2048 \
    --cpus-per-task 8 \
    --mem-per-cpu 1950M
```

#### 参数指南

- **`--workers`**：并行作业数量（DROID 的分片数最多为 2048）
- **`--cpus-per-task`**：建议 8 个 CPU，用于帧编码并行化
- **`--mem-per-cpu`**：加载原始帧约需 16GB 总内存（8×1950M）

> [!TIP]
> 在启动数千个作业之前，先用较少的 workers（例如 100 个）测试您的集群配置。

### 步骤 4：监控进度

查看运行中的作业：

```bash
squeue -u $USER
```

监控整体进度：

```bash
jobs_status /your/logs
```

查看单个作业日志：

```bash
less /your/logs/port_droid/slurm_jobs/JOB_ID_WORKER_ID.out
```

调试失败的作业：

```bash
failed_logs /your/logs/port_droid
```

### 步骤 5：聚合分片

所有移植作业完成后：

```bash
python examples/port_datasets/slurm_aggregate_shards.py \
    --repo-id your_id/droid_1.0.1 \
    --logs-dir /your/logs \
    --job-name aggr_droid \
    --partition your_partition \
    --workers 2048 \
    --cpus-per-task 8 \
    --mem-per-cpu 1950M
```

### 步骤 6：上传到 Hub

```bash
python examples/port_datasets/slurm_upload.py \
    --repo-id your_id/droid_1.0.1 \
    --logs-dir /your/logs \
    --job-name upload_droid \
    --partition your_partition \
    --workers 50 \
    --cpus-per-task 4 \
    --mem-per-cpu 1950M
```

> [!NOTE]
> 上传使用较少的 workers（50），因为它是网络密集型而非计算密集型。

## Dataset v3.0 文件结构

移植完成后的数据集将采用这种现代化结构：

```
dataset/
├── meta/
│   ├── episodes/
│   │   └── chunk-000/
│   │       └── file-000.parquet    # 片段元数据
│   ├── tasks.parquet               # 任务定义
│   ├── stats.json                  # 汇总统计
│   └── info.json                   # 数据集信息
├── data/
│   └── chunk-000/
│       └── file-000.parquet        # 合并的片段数据
└── videos/
    └── camera_key/
        └── chunk-000/
            └── file-000.mp4        # 合并的视频文件
```

这取代了旧的每个片段一个文件的模式，代之以高效且大小最优的块。

## 从 Dataset v2.1 迁移

如果您已有 v2.1 格式的数据集，请使用迁移工具：

```bash
python src/lerobot/scripts/convert_dataset_v21_to_v30.py \
    --repo-id your_id/existing_dataset
```

该工具会自动：

- 将文件结构转换为 v3.0 格式
- 将元数据从 JSON Lines 迁移到 parquet
- 聚合统计信息并创建逐片段统计
- 更新版本信息

## 性能优势

Dataset v3.0 为大型数据集带来了显著改进：

- **更快的加载速度**：初始化时间减少 3-5 倍
- **内存效率**：通过内存映射更好地利用内存
- **可扩展处理**：高效处理数百万个片段
- **存储优化**：减少文件数量并改进压缩
